## wall-vault гэж юу вэ?

**wall-vault = OpenClaw-д зориулсан AI төлөөлөгч (прокси) + API түлхүүрийн санг**

AI үйлчилгээ ашиглахад **API түлхүүр** (дижитал нэвтрэх үнэмлэх) шаардлагатай. API түлхүүр гэдэг нь "энэ хүн энэ үйлчилгээг ашиглах эрхтэй" гэдгийг баталгаажуулдаг **дижитал нэвтрэх үнэмлэх** юм. Гэвч энэ үнэмлэх өдөрт ашиглах тоо хязгаарлагдмал бөгөөд буруу удирдвал задрах аюул байдаг.

wall-vault эдгээр үнэмлэхүүдийг аюулгүй санд хадгалж, OpenClaw болон AI үйлчилгээний хооронд **төлөөлөгч (прокси)** үүрэг гүйцэтгэнэ. Товчоор хэлбэл, OpenClaw зөвхөн wall-vault-д холбогдох хэрэгтэй бөгөөд үлдсэн нарийн ажлыг wall-vault өөрөө зохицуулна.

wall-vault шийдэж өгдөг асуудлууд:

- **API түлхүүрийн автомат ээлж**: Нэг түлхүүрийн хязгаар дүүрэх эсвэл түр хаагдахад (хөргөлтийн хугацаа) дараагийн түлхүүрт чимээгүйхэн шилждэг. OpenClaw тасралтгүй ажиллана.
- **Үйлчилгээний автомат солих (fallback)**: Google хариу өгөхгүй бол OpenRouter-руу, тэр ч ажиллахгүй бол өөрийн компьютерт суулгасан Ollama (локал AI) руу автоматаар шилждэг. Сесс тасрахгүй.
- **Бодит цагийн синхрончлол (SSE)**: Сангийн хяналтын самбараас загварыг өөрчлөхөд 1–3 секундийн дотор OpenClaw дэлгэцэнд тусгалдана. SSE (Server-Sent Events) гэдэг нь серверийн өөрчлөлтийг бодит цагаар клиент рүү дамжуулдаг технологи юм.
- **Бодит цагийн мэдэгдэл**: Түлхүүр дуусах эсвэл үйлчилгээний саатал зэрэг үйл явдлууд OpenClaw TUI (терминалын дэлгэц) доод хэсэгт шууд харагдана.

> 💡 **Claude Code, Cursor, VS Code** ч гэсэн холбож ашиглаж болох боловч wall-vault-ийн үндсэн зориулалт нь OpenClaw-тай хамтарч ажиллах явдал юм.

```
OpenClaw (TUI терминалын дэлгэц)
        │
        ▼
  wall-vault прокси (:56244)   ← түлхүүр удирдлага, чиглүүлэлт, fallback, үйл явдал
        │
        ├─ Google Gemini API
        ├─ OpenRouter API (340+ загвар)
        └─ Ollama (өөрийн компьютер, сүүлчийн нөөц)
```

## Суулгах

### Linux / macOS

Терминал нээгээд доорх командыг бүтнээр нь хуулж оруулна уу.

```bash
# Linux (энгийн PC, сервер — amd64)
curl -L https://github.com/sookmook/wall-vault/releases/latest/download/wall-vault-linux-amd64 \
  -o ~/.local/bin/wall-vault && chmod +x ~/.local/bin/wall-vault

# macOS Apple Silicon (M1/M2/M3 Mac)
curl -L https://github.com/sookmook/wall-vault/releases/latest/download/wall-vault-darwin-arm64 \
  -o /usr/local/bin/wall-vault && chmod +x /usr/local/bin/wall-vault
```

- `curl -L ...` — файлыг интернетээс татаж авна.
- `chmod +x` — татаж авсан файлыг "ажиллуулах боломжтой" болгоно. Энэ алхмыг орхивол "зөвшөөрөл байхгүй" гэсэн алдаа гарна.

### Windows

PowerShell (администраторын эрхтэй) нээгээд доорх командыг ажиллуулна уу.

```powershell
# Татаж авах
Invoke-WebRequest -Uri `
  "https://github.com/sookmook/wall-vault/releases/latest/download/wall-vault-windows-amd64.exe" `
  -OutFile "$env:LOCALAPPDATA\Programs\wall-vault\wall-vault.exe"

# PATH-д нэмэх (PowerShell дахин эхлүүлсний дараа хэрэгжинэ)
$env:PATH += ";$env:LOCALAPPDATA\Programs\wall-vault"
```

> 💡 **PATH гэж юу вэ?** Компьютер командыг хайж үздэг хавтасны жагсаалт юм. PATH-д нэмснээр аль ч хавтсаас `wall-vault` гэж бичиж ажиллуулах боломжтой болно.

### Эх кодоос бие даан бүтээх (хөгжүүлэгчдэд)

Зөвхөн Go хэлний хөгжүүлэлтийн орчин суулгасан тохиолдолд хамаарна.

```bash
git clone https://github.com/sookmook/wall-vault
cd wall-vault
make build       # bin/wall-vault (хувилбар: v0.1.6.YYYYMMDD.HHmmss)
make install     # ~/.local/bin/wall-vault
```

> 💡 **Бүтээлтийн цагийн тэмдэглэлт хувилбар**: `make build`-ээр бүтээхэд хувилбар нь `v0.1.6.20260314.231308` гэх мэт огноо·цаг агуулсан хэлбэрээр автоматаар үүснэ. `go build ./...`-аар шууд бүтээвэл хувилбар нь зөвхөн `"dev"` гэж харагдана.

## Эхний тохиргоо

### setup шидтэн ажиллуулах

Суулгасны дараа эхний удаа заавал доорх командаар **тохиргооны шидтэн** ажиллуулна уу. Шидтэн шаардлагатай зүйлсийг нэг нэгээр асуун дагуулна.

```bash
wall-vault setup
```

Шидтэний явагдах алхмууд:

```
1. Хэл сонгох (монгол болон 10 хэл)
2. Сэдэв сонгох (light / dark / gold / cherry / ocean)
3. Ажиллах горим — ганцаараа (standalone) эсвэл олон компьютерт хамтдаа (distributed) сонгох
4. Ботын нэр оруулах — хяналтын самбарт харагдах нэр
5. Порт тохиргоо — үндсэн утга: прокси 56244, санг 56243 (өөрчлөх шаардлагагүй бол Enter дарна)
6. AI үйлчилгээ сонгох — Google / OpenRouter / Ollama дотроос ашиглах үйлчилгээ
7. Хэрэгслийн аюулгүй байдлын шүүлтүүр тохиргоо
8. Администраторын токен тохиргоо — хяналтын самбарын удирдлагын функцийг түгжих нууц үг. Автоматаар үүсгэх боломжтой
9. API түлхүүрийг шифрлэх нууц үг тохиргоо — түлхүүрийг илүү аюулгүй хадгалахыг хүсвэл (сонголтоор)
10. Тохиргооны файл хадгалах зам
```

> ⚠️ **Администраторын токеныг заавал санаж байгаарай.** Хяналтын самбараас түлхүүр нэмэх эсвэл тохиргоо өөрчлөхөд шаардлагатай. Мартвал тохиргооны файлыг шууд засах хэрэгтэй болно.

Шидтэнийг дуусгасны дараа `wall-vault.yaml` тохиргооны файл автоматаар үүснэ.

### Эхлүүлэх

```bash
wall-vault start
```

Доорх хоёр сервер нэгэн зэрэг эхэлнэ:

- **Прокси** (`http://localhost:56244`) — OpenClaw болон AI үйлчилгээний хооронд холбодог төлөөлөгч
- **Түлхүүрийн санг** (`http://localhost:56243`) — API түлхүүр удирдлага болон вэб хяналтын самбар

Браузерт `http://localhost:56243`-г нээвэл хяналтын самбарыг шууд харах боломжтой.

## API түлхүүр бүртгэх

API түлхүүр бүртгэх дөрвөн арга байна. **Эхлэн ашиглагчдад 1-р арга (орчны хувьсагч) санал болгоно.**

### 1-р арга: Орчны хувьсагч (санал болгосон — хамгийн энгийн)

Орчны хувьсагч гэдэг нь програм эхлэхдээ унших **урьдчилан тохируулсан утга** юм. Терминалд доорх байдлаар оруулна:

```bash
# Google Gemini түлхүүр бүртгэх
export WV_KEY_GOOGLE=AIzaSy...

# OpenRouter түлхүүр бүртгэх
export WV_KEY_OPENROUTER=sk-or-v1-...

# Бүртгэсний дараа эхлүүлэх
wall-vault start
```

Хэд хэдэн түлхүүр байвал таслалаар (,) холбоно уу. wall-vault түлхүүрүүдийг ротацилан автоматаар ашиглана (round robin):

```bash
export WV_KEY_GOOGLE=AIzaSy...,AIzaSy...,AIzaSy...
```

> 💡 **Зөвлөгөө**: `export` команд зөвхөн одоогийн терминалын сессид хэрэгжинэ. Компьютер дахин ачаалсны дараа ч хадгалагдахын тулд `~/.bashrc` эсвэл `~/.zshrc` файлд дээрх мөрийг нэмнэ үү.

### 2-р арга: Хяналтын самбарын UI (хулганаар товшиж)

1. Браузерт `http://localhost:56243` нэвтрэх
2. Дээд хэсэгт байх **🔑 API түлхүүр** картаас `[+ Нэмэх]` товчийг дарна
3. Үйлчилгээний төрөл, түлхүүрийн утга, шошго (тайлбар нэр), өдрийн хязгаар оруулаад хадгална

### 3-р арга: REST API (автоматжуулалт·скриптэд)

REST API гэдэг нь програмууд HTTP ашиглан өгөгдөл солилцох арга юм. Скриптээр автоматаар бүртгэхэд тохиромжтой.

```bash
curl -X POST http://localhost:56243/admin/keys \
  -H "Authorization: Bearer администраторын-токен" \
  -H "Content-Type: application/json" \
  -d '{
    "service": "google",
    "key": "AIzaSy...",
    "label": "Үндсэн түлхүүр",
    "daily_limit": 1000
  }'
```

### 4-р арга: proxy тугний параметр (түр тест хийхэд)

Албан ёсны бүртгэлгүйгээр түр түлхүүр оруулж тест хийх үед ашиглана. Програм зогсоход устна.

```bash
wall-vault proxy --key-google=AIzaSy... --key-openrouter=sk-or-...
```

## Прокси ашиглах

### OpenClaw-д ашиглах (үндсэн зориулалт)

OpenClaw нь wall-vault-ийг дамжуулан AI үйлчилгээтэй холбогдохоор тохируулах арга юм.

`~/.openclaw/openclaw.json` файлыг нээгээд доорх агуулгыг нэмнэ үү:

```json5
// ~/.openclaw/openclaw.json
{
  models: {
    providers: {
      "wall-vault": {
        baseUrl: "http://localhost:56244/v1",
        apiKey: "your-agent-token",   // vault агентын токен
        api: "openai-completions",
        models: [
          { id: "wall-vault/gemini-2.5-flash" },
          { id: "wall-vault/gemini-2.5-pro" },
          { id: "wall-vault/hunter-alpha" },    // үнэгүй 1M контекст
          { id: "wall-vault/claude-opus-4-6" }
        ]
      }
    }
  }
}
```

> 💡 **Илүү хялбар арга**: Хяналтын самбарын агентын картаас **🦞 OpenClaw тохиргоо хуулах** товчийг дарвал токен болон хаяг аль хэдийн бөглөгдсөн snippet клипборд руу хуулагдана. Зүгээр буулгахад л болно.

**Загварын нэрний өмнөх `wall-vault/` ямар үйлчилгээтэй холбогдох вэ?**

Загварын нэрийг харан wall-vault ямар AI үйлчилгээ рүү хүсэлт илгээхийг автоматаар тодорхойлно:

| Загварын хэлбэр | Холбогдох үйлчилгээ |
|----------------|-------------------|
| `wall-vault/gemini-*` | Google Gemini шууд холболт |
| `wall-vault/gpt-*`, `wall-vault/o3`, `wall-vault/o4*` | OpenAI шууд холболт |
| `wall-vault/claude-*` | OpenRouter дамжуулан Anthropic холболт |
| `wall-vault/hunter-alpha`, `wall-vault/healer-alpha` | OpenRouter (үнэгүй 1 сая токен контекст) |
| `wall-vault/kimi-*`, `wall-vault/glm-*`, `wall-vault/deepseek-*` | OpenRouter холболт |
| `google/загварнэр`, `openai/загварнэр`, `anthropic/загварнэр` г.м. | Тухайн үйлчилгээтэй шууд холболт |
| `custom/google/загварнэр`, `custom/openai/загварнэр` г.м. | `custom/` хэсгийг хасаад дахин чиглүүлэлт |
| `загварнэр:cloud` | `:cloud` хэсгийг хасаад OpenRouter холболт |

> 💡 **Контекст гэж юу вэ?** AI нэг удаад санаж чадах харилцааны хэмжээ юм. 1M (нэг сая токен) гэдэг нь маш урт яриа эсвэл урт баримт бичгийг нэг дор боловсруулах боломжтой гэсэн үг.

### Gemini API хэлбэрээр шууд холболт (одоо байгаа хэрэгслэлтэй нийцтэй)

Google Gemini API-г шууд ашиглаж байсан хэрэгсэл байвал зөвхөн хаягийг wall-vault болгон өөрчилнө үү:

```bash
export ANTHROPIC_BASE_URL=http://localhost:56244/google
```

Эсвэл URL-г шууд зааж болдог хэрэгсэлд:

```
http://localhost:56244/google/v1beta/models/gemini-2.5-flash:generateContent
```

### OpenAI SDK (Python)-д ашиглах

Python дээр AI ашиглах кодод wall-vault-г холбож болно. Зөвхөн `base_url`-г өөрчилнө:

```python
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:56244/v1",
    api_key="not-needed"  # API түлхүүрийг wall-vault өөрөө удирдана
)

response = client.chat.completions.create(
    model="google/gemini-2.5-flash",   # provider/model хэлбэрээр оруулна
    messages=[{"role": "user", "content": "Сайн байна уу"}]
)
```

### Ажиллаж байх үед загвар солих

wall-vault аль хэдийн ажиллаж байх үед ашиглах AI загварыг өөрчлөхийн тулд:

```bash
# Проксид шууд хүсэлт илгээж загвар өөрчлөх
curl -X PUT http://localhost:56244/api/config/model \
  -H "Content-Type: application/json" \
  -d '{"service": "openrouter", "model": "anthropic/claude-3.5-sonnet"}'

# Тархсан горим (олон бот)-д санг серверээс өөрчлөх → SSE-ээр шууд тусгалдана
curl -X PUT http://localhost:56243/admin/clients/миний-бот-id \
  -H "Authorization: Bearer администраторын-токен" \
  -H "Content-Type: application/json" \
  -d '{"default_service": "google", "default_model": "gemini-2.5-pro"}'
```

### Ашиглах боломжтой загваруудын жагсаалт харах

```bash
# Бүх жагсаалт харах
curl http://localhost:56244/api/models | python3 -m json.tool

# Зөвхөн Google загварууд харах
curl "http://localhost:56244/api/models?service=google"

# Нэрээр хайх (жишээ: "claude" агуулсан загварууд)
curl "http://localhost:56244/api/models?q=claude"
```

**Үйлчилгээ тус бүрийн үндсэн загваруудын хураангуй:**

| Үйлчилгээ | Үндсэн загварууд |
|-----------|----------------|
| Google | gemini-2.5-pro, gemini-2.5-flash, gemini-2.5-flash-8b, gemini-2.0-flash |
| OpenAI | gpt-4o, gpt-4o-mini, o3, o1, o1-mini |
| OpenRouter | 346 гаруй (Hunter Alpha 1M контекст үнэгүй, DeepSeek R1/V3, Qwen 2.5 г.м.) |
| Ollama | Өөрийн компьютерт суулгасан локал сервер автоматаар илрүүлэгдэнэ |

## Түлхүүрийн санги хяналтын самбар

Браузерт `http://localhost:56243`-г нэвтэрвэл хяналтын самбар харагдана.

**Дэлгэцийн бүтэц:**
- **Дээд тогтмол мөр (topbar)**: лого, хэл·сэдэв сонгогч, SSE холболтын статус харуулагч
- **Картын сүлжээ**: агент·үйлчилгээ·API түлхүүрийн карт тайлбай хэлбэрээр байрлана

### API түлхүүрийн карт

Бүртгэгдсэн API түлхүүрүүдийг нэг дор удирдах карт.

- Үйлчилгээ тус бүрд ангилан түлхүүрийн жагсаалт харуулна.
- `today_usage`: өнөөдөр амжилттай боловсруулсан токен (AI уншиж бичсэн тэмдэгтийн тоо) тоо
- `today_attempts`: өнөөдрийн нийт дуудлагын тоо (амжилттай + амжилтгүй оролдлого)
- `[+ Нэмэх]` товчоор шинэ түлхүүр бүртгэж, `✕`-ээр түлхүүр устгана.

> 💡 **Токен гэж юу вэ?** AI текст боловсруулахад ашиглах нэгж юм. Ойролцоогоор англи хэлний нэг үг, эсвэл монгол 1–2 тэмдэгтэд тохирно. API-ийн үнэ ихэвчлэн энэ токены тоогоор тооцогдоно.

### Агентын карт

wall-vault проксид холбогдсон ботуудын (агентуудын) статусыг харуулдаг карт.

**Холболтын статус 4 шатаар харагдана:**

| Харагдах байдал | Статус | Утга |
|----------------|--------|------|
| 🟢 | Ажиллаж байна | Прокси хэвийн ажиллаж байна |
| 🟡 | Саатал | Хариу ирж байгаа боловч удаан |
| 🔴 | Офлайн | Прокси хариу өгхгүй байна |
| ⚫ | Холбоогүй·идэвхгүй | Прокси санд холбогдож байгаагүй эсвэл идэвхгүй болгогдсон |

**Агентын картын доод хэсгийн товчнуудын тайлбар:**

Агентыг бүртгэхдээ **агентын төрлийг** зааж өгвөл тухайн төрөлд тохирсон тав тухтай товчнууд автоматаар гарч ирнэ.

---

#### 🔘 Тохиргоо хуулах товч — холболтын тохиргоог автоматаар үүсгэнэ

Товчийг дарвал тухайн агентын токен, проксийн хаяг, загварын мэдээлэл аль хэдийн бөглөгдсөн тохиргооны snippet клипборд руу хуулагдана. Хуулсан агуулгыг доорх хүснэгтийн байрлалд буулгахад холболтын тохиргоо бэлэн болно.

| Товч | Агентын төрөл | Буулгах байрлал |
|------|--------------|----------------|
| 🦞 OpenClaw тохиргоо хуулах | `openclaw` | `~/.openclaw/openclaw.json` |
| 🦀 NanoClaw тохиргоо хуулах | `nanoclaw` | `~/.openclaw/openclaw.json` |
| 🟠 Claude Code тохиргоо хуулах | `claude-code` | `~/.claude/settings.json` |
| ⌨ Cursor тохиргоо хуулах | `cursor` | Cursor → Settings → AI |
| 💻 VSCode тохиргоо хуулах | `vscode` | `~/.continue/config.json` |

**Жишээ — Claude Code төрөл бол иймэрхүү агуулга хуулагдана:**

```json
// ~/.claude/settings.json
{
  "apiProvider": "openai",
  "baseUrl": "http://192.168.1.20:56244/v1",
  "apiKey": "энэ-агентын-токен"
}
```

**Жишээ — VSCode (Continue) төрөл бол:**

```json
// ~/.continue/config.json
{
  "models": [{
    "title": "wall-vault proxy",
    "provider": "openai",
    "model": "gemini-2.5-flash",
    "apiBase": "http://192.168.1.20:56244/v1",
    "apiKey": "энэ-агентын-токен"
  }]
}
```

**Жишээ — Cursor төрөл бол:**

```
Base URL : http://192.168.1.20:56244/v1
API Key  : энэ-агентын-токен

// Эсвэл орчны хувьсагчаар:
OPENAI_BASE_URL=http://192.168.1.20:56244/v1
OPENAI_API_KEY=энэ-агентын-токен
```

> ⚠️ **Клипборд руу хуулагдахгүй үед**: Браузерийн аюулгүй байдлын бодлогоос хуулах нь хаагддаг тохиолдол байдаг. Текст хайрцаг бүхий попап нээгдвэл Ctrl+A дарж бүгдийг сонгоод Ctrl+C-ээр хуулна уу.

---

#### 🟣 Байршуулах командыг хуулах товч — шинэ компьютерт суулгахад ашиглана

Шинэ компьютерт wall-vault прокси анх суулгаж санд холбоход ашиглана. Товчийг дарвал суулгах скрипт бүхэлдээ хуулагдана. Шинэ компьютерийн терминалд буулгаж ажиллуулбал дараах зүйлс нэг дор хийгдэнэ:

1. wall-vault binary суулгах (аль хэдийн суулгасан бол алгасна)
2. systemd хэрэглэгчийн үйлчилгээ автоматаар бүртгэх
3. Үйлчилгээ эхлүүлэх болон санд автоматаар холбогдох

> 💡 Скриптийн дотор энэ агентын токен болон санг серверийн хаяг аль хэдийн бөглөгдсөн байгаа тул буулгасны дараа нэмэлт засвар хийлгүй шууд ажиллуулж болно.

---

### Үйлчилгээний карт

Ашиглах AI үйлчилгээг асааж унтрааж эсвэл тохируулах карт.

- Үйлчилгээ тус бүрийг идэвхжүүлэх·идэвхгүй болгох toggle switch
- Локал AI сервер (өөрийн компьютерт ажиллуулж байгаа Ollama, LM Studio, vLLM г.м.)-ийн хаягийг оруулбал ашиглах боломжтой загваруудыг автоматаар олж ирнэ.
- **Локал үйлчилгээний холболтын статус**: Үйлчилгээний нэрний хажуу дахь ● цэг **ногоон** бол холбогдсон, **саарал** бол холбогдоогүй
- **Checkbox автомат синхрончлол**: Хуудас нээгдэх үед локал үйлчилгээ (Ollama г.м.) ажиллаж байвал автоматаар checkbox-н идэвхжүүлсэн байдалд орно.

> 💡 **Локал үйлчилгээ өөр компьютерт ажиллаж байвал**: Үйлчилгээний URL талбарт тухайн компьютерийн IP-г оруулна уу. Жишээ: `http://192.168.1.20:11434` (Ollama), `http://192.168.1.20:1234` (LM Studio)

### Администраторын токен оруулах

Хяналтын самбараас түлхүүр нэмэх·устгах зэрэг чухал функц ашиглахад администраторын токен оруулах попап гарч ирнэ. setup шидтэнд тохируулсан токеноо оруулна уу. Нэг удаа оруулбал браузер хаахаас өмнө хадгалагдсан хэвээр байна.

> ⚠️ **15 минутын дотор 10 удаа баталгаажуулалт амжилтгүй болбол тухайн IP түр хаагдана.** Токеноо мартсан бол `wall-vault.yaml` файлын `admin_token` хэсгийг шалгана уу.

## Тархсан горим (олон бот)

Хэд хэдэн компьютерт OpenClaw нэгэн зэрэг ажиллуулахад **нэг түлхүүрийн сангийг хуваалцдаг** тохиргоо юм. Түлхүүр удирдлагыг нэг газраас л хийвэл хангалттай тул тохиромжтой.

### Тохиргооны жишээ

```
[Түлхүүрийн санг сервер]
  wall-vault vault    (түлхүүрийн санг :56243, хяналтын самбар)

[WSL Альфа]           [Raspberry Pi Гамма]   [Mac Mini Локал]
  wall-vault proxy      wall-vault proxy        wall-vault proxy
  openclaw TUI          openclaw TUI            openclaw TUI
  ↕ SSE синхрончлол     ↕ SSE синхрончлол       ↕ SSE синхрончлол
```

Бүх бот дунд байрлах санг серверийг харж байгаа тул санг дээр загвар солих эсвэл түлхүүр нэмэхэд бүх ботод шууд тусгалдана.

### 1-р алхам: Түлхүүрийн санг сервер эхлүүлэх

Санг сервер болгох компьютерт ажиллуулна:

```bash
wall-vault vault
```

### 2-р алхам: Бот (клиент) бүр бүртгэх

Санг серверт холбогдох бот бүрийн мэдээллийг урьдчилан бүртгэнэ:

```bash
curl -X POST http://localhost:56243/admin/clients \
  -H "Authorization: Bearer администраторын-токен" \
  -H "Content-Type: application/json" \
  -d '{
    "id": "botA",
    "name": "БотА",
    "token": "bota-secret",
    "default_service": "google",
    "default_model": "gemini-2.5-flash"
  }'
```

### 3-р алхам: Бот бүрийн компьютерт проксийг эхлүүлэх

Бот суулгасан бүр компьютерт санг серверийн хаяг болон токеныг зааж проксийг ажиллуулна:

```bash
WV_VAULT_URL=http://192.168.x.x:56243 \
WV_VAULT_TOKEN=bota-secret \
WV_VAULT_CLIENT_ID=botA \
wall-vault proxy
```

> 💡 **`192.168.x.x`** хэсгийг санг серверийн компьютерийн бодит дотоод IP хаягаар солино уу. Чиглүүлэгчийн тохиргоо эсвэл `ip addr` командаар шалгаж болно.

## Автомат эхлүүлэх тохиргоо

Компьютер дахин ачаалах бүрт wall-vault-г гараар асаах нь залхмаар бол системийн үйлчилгээ болгон бүртгэнэ үү. Нэг удаа бүртгэсний дараа ачаалах үед автоматаар эхэлнэ.

### Linux — systemd (ихэнх Linux)

systemd гэдэг нь Linux дээр програмуудыг автоматаар эхлүүлж удирддаг систем юм:

```bash
wall-vault doctor deploy
systemctl --user daemon-reload
systemctl --user enable --now wall-vault
```

Лог шалгах:

```bash
journalctl --user -u wall-vault -f
```

### macOS — launchd

macOS дээр програмын автомат ажиллуулалтыг хариуцдаг систем:

```bash
wall-vault doctor deploy launchd
launchctl load ~/Library/LaunchAgents/com.wall-vault.plist
```

### Windows — NSSM

1. [nssm.cc](https://nssm.cc/download)-ээс NSSM-ийг татаж авч PATH-д нэмнэ.
2. Администраторын эрхтэй PowerShell-д:

```powershell
wall-vault doctor deploy windows
```

## Doctor оношлогч

`doctor` команд нь wall-vault зөв тохируулагдсан эсэхийг **өөрөө оношлож засах хэрэгсэл** юм.

```bash
wall-vault doctor check   # Одоогийн байдлыг оношлох (зөвхөн уншина, юу ч өөрчлөхгүй)
wall-vault doctor fix     # Асуудлыг автоматаар засах
wall-vault doctor all     # Оношлох + автомат засах нэг дор
```

> 💡 Ямар нэг алдаатай байх шиг санагдвал эхлээд `wall-vault doctor all`-г ажиллуулж үзнэ үү. Олон асуудлыг автоматаар илрүүлж засна.

## Орчны хувьсагч лавлах

Орчны хувьсагч гэдэг нь програмд тохиргооны утгыг дамжуулах арга юм. `export хувьсагчийн_нэр=утга` хэлбэрээр терминалд оруулах эсвэл автомат эхлүүлэх үйлчилгээний файлд хийвэл үргэлж хэрэгжинэ.

| Хувьсагч | Тайлбар | Жишээ утга |
|---------|---------|-----------|
| `WV_LANG` | Хяналтын самбарын хэл | `ko`, `en`, `ja` |
| `WV_THEME` | Хяналтын самбарын сэдэв | `light`, `dark`, `gold` |
| `WV_KEY_GOOGLE` | Google API түлхүүр (таслалаар олон түлхүүр) | `AIza...,AIza...` |
| `WV_KEY_OPENROUTER` | OpenRouter API түлхүүр | `sk-or-v1-...` |
| `WV_VAULT_URL` | Тархсан горим дахь санг серверийн хаяг | `http://192.168.x.x:56243` |
| `WV_VAULT_TOKEN` | Клиент (бот)-ийн баталгаажуулах токен | `my-secret-token` |
| `WV_ADMIN_TOKEN` | Администраторын токен | `admin-token-here` |
| `WV_MASTER_PASS` | API түлхүүр шифрлэх нууц үг | `my-password` |
| `WV_AVATAR` | Аватар зургийн файлын зам (`~/.openclaw/`-аас харьцангуй зам) | `workspace/avatars/avatar.png` |
| `OLLAMA_URL` | Ollama локал серверийн хаяг | `http://192.168.x.x:11434` |

## Алдаа засах

### Прокси эхлэхгүй байх үед

Порт аль хэдийн өөр програм ашиглаж байгаа тохиолдол олонтаа гардаг.

```bash
ss -tlnp | grep 56244   # 56244 портыг хэн ашиглаж байгааг шалгах
wall-vault proxy --port 8080   # Өөр порт дугаараар эхлүүлэх
```

### API түлхүүрийн алдаа гарах үед (429, 402, 401, 403, 582)

| Алдааны код | Утга | Шийдэх арга |
|------------|------|------------|
| **429** | Хүсэлт хэт олон (хэрэглээний хязгаар давсан) | Хэсэг хүлээх эсвэл өөр түлхүүр нэмэх |
| **402** | Төлбөр шаардлагатай эсвэл кредит дутмаг | Тухайн үйлчилгээнд кредит нэмэх |
| **401 / 403** | Түлхүүр буруу эсвэл зөвшөөрөл байхгүй | Түлхүүрийн утгыг дахин шалгаад дахин бүртгэх |
| **582** | Гарцийн хэт ачаалал (5 минут хөргөлт) | 5 минутын дараа автоматаар нээгдэнэ |

```bash
# Бүртгэгдсэн түлхүүрийн жагсаалт болон статус шалгах
curl -H "Authorization: Bearer администраторын-токен" http://localhost:56243/admin/keys

# Түлхүүрийн хэрэглээний тоолуур дахин тохируулах
curl -X POST -H "Authorization: Bearer администраторын-токен" http://localhost:56243/admin/keys/reset
```

### Агент "холбоогүй" гэж харагдах үед

"Холбоогүй" гэдэг нь прокси процесс санд дохио (heartbeat) илгээхгүй байгаа байдал юм. **Энэ нь тохиргоо хадгалагдаагүй гэсэн утга биш юм.** Прокси санг серверийн хаяг болон токеныг мэдэж ажиллах ёстой бөгөөд тэгж байж холбогдсон байдалд шилжинэ.

```bash
# Санг серверийн хаяг, токен, клиентийн ID зааж проксийг эхлүүлэх
WV_VAULT_URL=http://санг-серверийн-хаяг:56243 \
WV_VAULT_TOKEN=клиентийн-токен \
WV_VAULT_CLIENT_ID=клиентийн-ID \
wall-vault proxy
```

Амжилттай холбогдвол ойролцоогоор 20 секундийн дотор хяналтын самбарт 🟢 Ажиллаж байна гэж өөрчлөгдөнө.

### Ollama холбогдохгүй байх үед

Ollama гэдэг нь өөрийн компьютерт шууд AI ажиллуулах програм юм. Эхлээд Ollama асаалттай эсэхийг шалгана уу.

```bash
curl http://localhost:11434/api/tags   # Загваруудын жагсаалт гарвал хэвийн
export OLLAMA_URL=http://192.168.x.x:11434   # Өөр компьютерт ажиллаж байгаа тохиолдолд
```

> ⚠️ Ollama хариу өгхгүй бол `ollama serve` командаар эхлээд Ollama-г эхлүүлнэ үү.

> ⚠️ **Том загварууд удаан хариулдаг**: `qwen3.5:35b`, `deepseek-r1` зэрэг том загварууд хариу үүсгэхэд хэдэн минут зарцуулдаг. Хариу байхгүй мэт харагдсан ч хэвийн боловсруулж байж болох тул хүлээнэ үү.
