## wall-vault гэж юу вэ?

**wall-vault = OpenClaw-д зориулсан AI төлөөлөгч (прокси) + API түлхүүрийн санг**

AI үйлчилгээ ашиглахад **API түлхүүр** шаардлагатай. API түлхүүр гэдэг нь "энэ хүн энэ үйлчилгээг ашиглах эрхтэй" гэдгийг баталгаажуулдаг **дижитал нэвтрэх үнэмлэх** юм. Гэвч энэ үнэмлэх өдөрт ашиглах тоо хязгаарлагдмал бөгөөд буруу удирдвал задрах аюул байдаг.

wall-vault эдгээр үнэмлэхүүдийг аюулгүй санд хадгалж, OpenClaw болон AI үйлчилгээний хооронд **төлөөлөгч (прокси)** үүрэг гүйцэтгэнэ. Товчоор хэлбэл, OpenClaw зөвхөн wall-vault-д холбогдох хэрэгтэй бөгөөд үлдсэн нарийн ажлыг wall-vault өөрөө зохицуулна.

wall-vault шийдэж өгдөг асуудлууд:

- **API түлхүүрийн автомат ээлж**: Нэг түлхүүрийн хязгаар дүүрэх эсвэл түр хаагдахад (хөргөлтийн хугацаа) дараагийн түлхүүрт чимээгүйхэн шилждэг. OpenClaw тасралтгүй ажиллана.
- **Үйлчилгээний автомат солих (fallback)**: Google хариу өгөхгүй бол OpenRouter-руу, тэр ч ажиллахгүй бол өөрийн компьютерт суулгасан Ollama·LM Studio·vLLM (локал AI) руу автоматаар шилждэг. Сесс тасрахгүй. Анхны үйлчилгээ сэргэх үед дараагийн хүсэлтээс эхлэн автоматаар буцаж шилждэг (v0.1.18+, LM Studio/vLLM: v0.1.21+).
- **Бодит цагийн синхрончлол (SSE)**: Сангийн хяналтын самбараас загварыг өөрчлөхөд 1–3 секундийн дотор OpenClaw дэлгэцэнд тусгалдана. SSE (Server-Sent Events) гэдэг нь серверийн өөрчлөлтийг бодит цагаар клиент рүү дамжуулдаг технологи юм.
- **Бодит цагийн мэдэгдэл**: Түлхүүр дуусах эсвэл үйлчилгээний саатал зэрэг үйл явдлууд OpenClaw TUI (терминалын дэлгэц) доод хэсэгт шууд харагдана.

> 💡 **Claude Code, Cursor, VS Code** ч гэсэн холбож ашиглаж болох боловч wall-vault-ийн үндсэн зориулалт нь OpenClaw-тай хамтарч ажиллах явдал юм.

```
OpenClaw (TUI терминалын дэлгэц)
        │
        ▼
  wall-vault прокси (:56244)   ← түлхүүр удирдлага, чиглүүлэлт, fallback, үйл явдал
        │
        ├─ Google Gemini API
        ├─ OpenRouter API (340+ загвар)
        ├─ Ollama / LM Studio / vLLM (өөрийн компьютер, сүүлчийн нөөц)
        └─ OpenAI / Anthropic API
```

## Суулгах

### Linux / macOS

Терминал нээж доорх командуудыг байгаа чигтээ нь хуулна уу.

```bash
# Linux (ердийн PC, сервер — amd64)
curl -L https://github.com/sookmook/wall-vault/releases/latest/download/wall-vault-linux-amd64 \
  -o ~/.local/bin/wall-vault && chmod +x ~/.local/bin/wall-vault

# macOS Apple Silicon (M1/M2/M3 Mac)
curl -L https://github.com/sookmook/wall-vault/releases/latest/download/wall-vault-darwin-arm64 \
  -o /usr/local/bin/wall-vault && chmod +x /usr/local/bin/wall-vault
```

- `curl -L ...` — Интернэтээс файл татаж авна.
- `chmod +x` — Татсан файлыг "ажиллуулах боломжтой" болгоно. Энэ алхамыг алгасвал "зөвшөөрөл байхгүй" алдаа гарна.

### Windows

PowerShell (администраторын эрхээр) нээж доорх командуудыг ажиллуулна уу.

```powershell
# Татах
Invoke-WebRequest -Uri `
  "https://github.com/sookmook/wall-vault/releases/latest/download/wall-vault-windows-amd64.exe" `
  -OutFile "$env:LOCALAPPDATA\Programs\wall-vault\wall-vault.exe"

# PATH нэмэх (PowerShell дахин нээсний дараа хүчинтэй болно)
$env:PATH += ";$env:LOCALAPPDATA\Programs\wall-vault"
```

> 💡 **PATH гэж юу вэ?** Компьютер команд хайдаг хавтаснуудын жагсаалт юм. PATH-д нэмвэл аль ч хавтаснаас `wall-vault` гэж бичиж ажиллуулж болно.

### Эх кодноос шууд бүтээх (хөгжүүлэгчдэд)

Go хэлний хөгжүүлэлтийн орчин суулгасан тохиолдолд л хамаарна.

```bash
git clone https://github.com/sookmook/wall-vault
cd wall-vault
make build       # bin/wall-vault (хувилбар: v0.1.23.YYYYMMDD.HHmmss)
make install     # ~/.local/bin/wall-vault
```

> 💡 **Бүтээх цагийн тэмдэг хувилбар**: `make build`-ээр бүтээхэд хувилбар `v0.1.23.20260406.211004` шиг огноо·цаг агуулсан формаатаар автоматаар үүсгэгдэнэ. `go build ./...`-ээр шууд бүтээхэд хувилбар `"dev"` гэжээ харагдана.

## Эхний тохиргоо

### setup шидтэн ажиллуулах

Суулгасны дараа заавал доорх командаар **тохиргооны шидтэн** ажиллуулна уу. Шидтэн шаардлагатай зүйлсийг нэг нэгээр нь асууж чиглүүлнэ.

```bash
wall-vault setup
```

Шидтэнгийн явагдах алхамууд:

```
1. Хэлний сонголт (Монгол хэл зэрэг 10 хэл)
2. Загварын сонголт (light / dark / gold / cherry / ocean)
3. Ажиллагааны горим — ганцаараа (standalone), эсвэл олон төхөөрөмжөөр хамт (distributed)
4. Ботын нэр — хяналтын самбарт харуулах нэр
5. Порт тохиргоо — анхдагч: прокси 56244, санг 56243 (өөрчлөх шаардлагагүй бол Enter дарна)
6. AI үйлчилгээний сонголт — Google / OpenRouter / Ollama / LM Studio / vLLM
7. Хэрэгслийн аюулгүйн шүүлтүүрийн тохиргоо
8. Администраторын токен тохиргоо — хяналтын самбарын удирдлагын функцийг түгжих нууц үг. Автомат үүсгэх боломжтой
9. API түлхүүрийн шифрлэлтийн нууц үг — түлхүүрийг илүү найдвартай хадгалахад (сонголттой)
10. Тохиргооны файл хадгалах байршил
```

> ⚠️ **Администраторын токеныг заавал санана уу.** Дараа нь хяналтын самбарт түлхүүр нэмэх эсвэл тохиргоо өөрчлөхөд хэрэгтэй. Мартвал тохиргооны файлыг шууд засварлах шаардлагатай.

Шидтэн дуусахад `wall-vault.yaml` тохиргооны файл автоматаар үүсгэгдэнэ.

### Ажиллуулах

```bash
wall-vault start
```

Доорх хоёр сервер нэгэн зэрэг эхэлнэ:

- **Прокси** (`http://localhost:56244`) — OpenClaw болон AI үйлчилгээний хооронд холбож өгөх төлөөлөгч
- **Түлхүүрийн санг** (`http://localhost:56243`) — API түлхүүр удирдлага ба вэб хяналтын самбар

Хөтөчөөр `http://localhost:56243` руу орж хяналтын самбарыг шууд харж болно.

## API түлхүүр бүртгэх

API түлхүүр бүртгэх дөрвөн арга бий. **Анх удаа ашиглаж байгаа хүмүүст арга 1 (орчны хувьсагч) санал болгоно**.

### Арга 1: Орчны хувьсагч (Санал болгох — Хамгийн хялбар)

Орчны хувьсагч гэдэг нь програм эхлэхэд уншдаг **урьдчилж тохируулсан утга** юм. Терминалд ингэж бичнэ:

```bash
# Google Gemini түлхүүр бүртгэх
export WV_KEY_GOOGLE=AIzaSy...

# OpenRouter түлхүүр бүртгэх
export WV_KEY_OPENROUTER=sk-or-v1-...

# Бүртгэсний дараа ажиллуулах
wall-vault start
```

Олон түлхүүртэй бол таслалаар (,) холбоно. wall-vault түлхүүрүүдийг ээлжлэн автоматаар ашиглана (round robin):

```bash
export WV_KEY_GOOGLE=AIzaSy...,AIzaSy...,AIzaSy...
```

> 💡 **Зөвлөгөө**: `export` команд зөвхөн одоогийн терминал сешнд л хүчинтэй. Компьютер дахин асаасны дараа ч хадгалахыг хүсвэл `~/.bashrc` эсвэл `~/.zshrc` файлд дээрх мөрийг нэмнэ үү.

### Арга 2: Хяналтын самбарын UI (хулганаар товших)

1. Хөтөч дээр `http://localhost:56243` руу орно
2. Дээд талд байгаа **🔑 API түлхүүр** картын `[+ Нэмэх]` товчийг дарна
3. Үйлчилгээний төрөл, түлхүүрийн утга, шошго (санах нэр), өдрийн хязгаарыг оруулж хадгална

### Арга 3: REST API (автоматжуулалт, скрипт)

REST API гэдэг нь программууд HTTP-ээр өгөгдөл солилцох арга юм. Скриптээр автомат бүртгэхэд тохиромжтой.

```bash
curl -X POST http://localhost:56243/admin/keys \
  -H "Authorization: Bearer администраторын-токен" \
  -H "Content-Type: application/json" \
  -d '{
    "service": "google",
    "key": "AIzaSy...",
    "label": "Үндсэн түлхүүр",
    "daily_limit": 1000
  }'
```

### Арга 4: proxy флаг (түр туршилтанд)

Албан ёсны бүртгэлгүйгээр түр зуур түлхүүр оруулж туршихад ашиглана. Програм зогсоход алга болно.

```bash
wall-vault proxy --key-google=AIzaSy... --key-openrouter=sk-or-...
```

## Прокси ашиглах

### OpenClaw-тай ашиглах (Үндсэн зорилго)

OpenClaw-ийг wall-vault-аар дамжуулан AI үйлчилгээтэй холбох тохиргоо.

`~/.openclaw/openclaw.json` файлыг нээж доорх агуулгыг нэмнэ:

```json5
// ~/.openclaw/openclaw.json
{
  models: {
    providers: {
      "wall-vault": {
        baseUrl: "http://localhost:56244/v1",
        apiKey: "your-agent-token",   // vault агентын токен
        api: "openai-completions",
        models: [
          { id: "wall-vault/gemini-2.5-flash" },
          { id: "wall-vault/gemini-2.5-pro" },
          { id: "wall-vault/hunter-alpha" },    // үнэгүй 1M context
          { id: "wall-vault/claude-opus-4-6" }
        ]
      }
    }
  }
}
```

> 💡 **Илүү хялбар арга**: Хяналтын самбарын агент картын **🦞 OpenClaw тохиргоо хуулах** товчийг дарна уу. Токен, хаяг аль хэдийн бөглөгдсөн хэсэг clipboard-д хуулагдана. Зүгээр буулгана.

**Загварын нэрний өмнөх `wall-vault/` хаашаа холбогдох вэ?**

Загварын нэрийг харж wall-vault ямар AI үйлчилгээ рүү хүсэлт илгээхээ автоматаар шийднэ:

| Загварын формат | Холбогдох үйлчилгээ |
|----------|--------------|
| `wall-vault/gemini-*` | Google Gemini шууд холболт |
| `wall-vault/gpt-*`, `wall-vault/o3`, `wall-vault/o4*` | OpenAI шууд холболт |
| `wall-vault/claude-*` | OpenRouter-ээр дамжуулан Anthropic холболт |
| `wall-vault/hunter-alpha`, `wall-vault/healer-alpha` | OpenRouter (1 сая токен context үнэгүй) |
| `wall-vault/kimi-*`, `wall-vault/glm-*`, `wall-vault/deepseek-*` | OpenRouter холболт |
| `google/загварын-нэр`, `openai/загварын-нэр`, `anthropic/загварын-нэр` г.м. | Тухайн үйлчилгээ шууд холболт |
| `custom/google/загварын-нэр`, `custom/openai/загварын-нэр` г.м. | `custom/` хэсгийг хасч дахин чиглүүлэх |
| `загварын-нэр:cloud` | `:cloud` хэсгийг хасч OpenRouter-ээр холбох |

> 💡 **Context гэж юу вэ?** AI-ийн нэг дор санаж чадах яриаын хэмжээ юм. 1M (1 сая токен) гэдэг нь маш урт яриа эсвэл урт бичиг баримтыг нэг дор боловсруулж чадна гэсэн үг.

### Gemini API форматаар шууд холбох (одоо байгаа хэрэгслүүдтэй нийцэх)

Google Gemini API шууд ашигладаг хэрэгсэл байгаа бол хаягийг wall-vault руу солино:

```bash
export ANTHROPIC_BASE_URL=http://localhost:56244/google
```

Эсвэл URL шууд заадаг хэрэгсэлд:

```
http://localhost:56244/google/v1beta/models/gemini-2.5-flash:generateContent
```

### OpenAI SDK (Python)-д ашиглах

Python дээр AI ашигладаг кодонд ч wall-vault холбож болно. `base_url`-ийг л солино:

```python
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:56244/v1",
    api_key="not-needed"  # API түлхүүрийг wall-vault өөрөө удирдана
)

response = client.chat.completions.create(
    model="google/gemini-2.5-flash",   # provider/model форматаар оруулна
    messages=[{"role": "user", "content": "Сайн байна уу"}]
)
```

### Ажиллаж байх үед загвар солих

wall-vault аль хэдийн ажиллаж байгаа үед AI загварыг солихын тулд:

```bash
# Прокси руу шууд хүсэлт илгээж загвар өөрчлөх
curl -X PUT http://localhost:56244/api/config/model \
  -H "Content-Type: application/json" \
  -d '{"service": "openrouter", "model": "anthropic/claude-3.5-sonnet"}'

# Тархсан горимд (олон бот) сангийн серверээс өөрчлөх → SSE-ээр шууд тусгалдана
curl -X PUT http://localhost:56243/admin/clients/миний-бот-id \
  -H "Authorization: Bearer администраторын-токен" \
  -H "Content-Type: application/json" \
  -d '{"default_service": "google", "default_model": "gemini-2.5-pro"}'
```

### Боломжтой загварын жагсаалт харах

```bash
# Бүх жагсаалт харах
curl http://localhost:56244/api/models | python3 -m json.tool

# Зөвхөн Google загварууд
curl "http://localhost:56244/api/models?service=google"

# Нэрээр хайх (жишээ: "claude" агуулсан загварууд)
curl "http://localhost:56244/api/models?q=claude"
```

**Үйлчилгээ тус бүрийн гол загваруудын хураангуй:**

| Үйлчилгээ | Гол загварууд |
|--------|----------|
| Google | gemini-2.5-pro, gemini-2.5-flash, gemini-2.5-flash-8b, gemini-2.0-flash |
| OpenAI | gpt-4o, gpt-4o-mini, o3, o1, o1-mini |
| OpenRouter | 346-аас дээш (Hunter Alpha 1M context үнэгүй, DeepSeek R1/V3, Qwen 2.5 г.м.) |
| Ollama | Компьютерт суулгасан локал серверийн загваруудыг автомат олох |
| LM Studio | Компьютерийн локал сервер (порт 1234) |
| vLLM | Компьютерийн локал сервер (порт 8000) |

## Түлхүүрийн санги хяналтын самбар

Хөтөч дээр `http://localhost:56243` руу орж хяналтын самбарыг харж болно.

**Дэлгэцийн бүтэц:**
- **Дээд тогтмол мөр (topbar)**: Лого, хэл·загварын сонголт, SSE холболтын төлөв
- **Карт грид**: Агент·үйлчилгээ·API түлхүүр картууд хавтан хэлбэрээр байрласан

### API түлхүүрийн карт

Бүртгэлтэй API түлхүүрүүдийг нэг харцаар удирдах карт.

- Үйлчилгээ тус бүрээр ялгаж түлхүүрийн жагсаалтыг харуулна.
- `today_usage`: Өнөөдөр амжилттай боловсруулсан токены тоо (AI-ийн уншсан, бичсэн тэмдэгтийн тоо)
- `today_attempts`: Өнөөдрийн нийт хүсэлтийн тоо (амжилттай + амжилтгүй)
- `[+ Нэмэх]` товчоор шинэ түлхүүр бүртгэж, `✕`-ээр түлхүүр устгана.

> 💡 **Токен (token) гэж юу вэ?** AI текст боловсруулахад ашигладаг нэгж юм. Ойролцоогоор англи хэлний нэг үг, эсвэл бусад хэлний 1–2 тэмдэгттэй тэнцэнэ. API-ийн төлбөр ихэвчлэн энэ токены тоогоор тооцогддог.

### Агент карт

wall-vault прокситэй холбогдсон бот (агент)-уудын төлвийг харуулах карт.

**Холболтын төлөв 4 түвшнээр харуулна:**

| Тэмдэг | Төлөв | Утга |
|------|------|------|
| 🟢 | Ажиллаж буй | Прокси хэвийн ажиллаж байна |
| 🟡 | Удааширсан | Хариу ирж байгаа боловч удаан |
| 🔴 | Офлайн | Прокси хариу өгөхгүй байна |
| ⚫ | Холбогдоогүй·Идэвхгүй | Прокси сангтай хэзээ ч холбогдоогүй эсвэл идэвхгүй болгосон |

**Агент картын доод хэсгийн товчнуудын тайлбар:**

Агент бүртгэхэд **агентын төрөл** заасан тохиолдолд тухайн төрөлд тохирох хялбаршуулах товчнууд автоматаар гарч ирнэ.

#### 🔘 Тохиргоо хуулах товч — Холболтын тохиргоог автоматаар хийж өгнө

Товчийг дарахад тухайн агентын токен, прокси хаяг, загварын мэдээлэл аль хэдийн бөглөгдсөн тохиргооны хэсэг clipboard-д хуулагдана. Хуулсан агуулгыг доорх хүснэгтийн байршилд буулгавал холболтын тохиргоо дуусна.

| Товч | Агентын төрөл | Буулгах байршил |
|------|-------------|-------------|
| 🦞 OpenClaw тохиргоо хуулах | `openclaw` | `~/.openclaw/openclaw.json` |
| 🦀 NanoClaw тохиргоо хуулах | `nanoclaw` | `~/.openclaw/openclaw.json` |
| 🟠 Claude Code тохиргоо хуулах | `claude-code` | `~/.claude/settings.json` |
| ⌨ Cursor тохиргоо хуулах | `cursor` | Cursor → Settings → AI |
| 💻 VSCode тохиргоо хуулах | `vscode` | `~/.continue/config.json` |

**Жишээ — Claude Code төрөл бол ийм зүйл хуулагдна:**

```json
// ~/.claude/settings.json
{
  "apiProvider": "openai",
  "baseUrl": "http://192.168.0.6:56244/v1",
  "apiKey": "энэ-агентын-токен"
}
```

**Жишээ — VSCode (Continue) төрөл бол:**

```yaml
# ~/.continue/config.yaml  ← config.json биш config.yaml-д буулгана
name: My Config
version: 0.0.1
schema: v1

models:
  - name: wall-vault proxy
    provider: openai
    model: gemini-2.5-flash
    apiBase: http://192.168.0.6:56244/v1
    apiKey: энэ-агентын-токен
    roles:
      - chat
      - edit
      - apply
```

> ⚠️ **Continue-ийн хамгийн сүүлийн хувилбар `config.yaml` ашигладаг.** `config.yaml` байгаа бол `config.json`-ийг бүрэн үл тоомсорлоно. Заавал `config.yaml`-д буулгана уу.

**Жишээ — Cursor төрөл бол:**

```
Base URL : http://192.168.0.6:56244/v1
API Key  : энэ-агентын-токен

// Эсвэл орчны хувьсагч:
OPENAI_BASE_URL=http://192.168.0.6:56244/v1
OPENAI_API_KEY=энэ-агентын-токен
```

> ⚠️ **Clipboard хуулалт ажиллахгүй бол**: Хөтчийн аюулгүйн бодлого хуулахыг хориглож байгаа байж болно. Popup текст хайрцаг нээгдвэл Ctrl+A-аар бүгдийг сонгоод Ctrl+C-ээр хуулна уу.

#### ⚡ Автомат хэрэглэх товч — Нэг удаа дарахад тохиргоо дуусна

Агентын төрөл `cline`, `claude-code`, `openclaw`, эсвэл `nanoclaw` бол агент картад **⚡ Тохиргоо хэрэглэх** товч харагдана. Энэ товчийг дарахад тухайн агентын локал тохиргооны файл автоматаар шинэчлэгдэнэ.

| Товч | Агентын төрөл | Хэрэглэгдэх файл |
|------|-------------|-------------|
| ⚡ Cline тохиргоо хэрэглэх | `cline` | `~/.cline/data/globalState.json` + `secrets.json` |
| ⚡ Claude Code тохиргоо хэрэглэх | `claude-code` | `~/.claude/settings.json` |
| ⚡ OpenClaw тохиргоо хэрэглэх | `openclaw` | `~/.openclaw/openclaw.json` |
| ⚡ NanoClaw тохиргоо хэрэглэх | `nanoclaw` | `~/.openclaw/openclaw.json` |

> ⚠️ Энэ товч **localhost:56244** (локал прокси) руу хүсэлт илгээнэ. Тухайн машин дээр прокси ажиллаж байх ёстой.

#### 🔄 Хоёр чиглэлийн загвар синхрончлол (v0.1.16)

Сангийн хяналтын самбараас агентын загвар солиход тухайн агентын локал тохиргоо автоматаар шинэчлэгдэнэ.

**Cline-ийн хувьд:**
- Сангаас загвар өөрчлөх → SSE үйл явдал → прокси `globalState.json` дахь загварын талбарыг шинэчлэх
- Шинэчлэгдэх талбарууд: `actModeOpenAiModelId`, `planModeOpenAiModelId`, `openAiModelId`
- `openAiBaseUrl` болон API түлхүүр хөндөхгүй
- **VS Code дахин ачаалах (`Ctrl+Alt+R` эсвэл `Ctrl+Shift+P` → `Developer: Reload Window`) шаардлагатай**
  - Cline ажиллаж байх үедээ тохиргооны файлыг дахин уншдаггүй

**Claude Code-ийн хувьд:**
- Сангаас загвар өөрчлөх → SSE үйл явдал → прокси `settings.json` дахь `model` талбарыг шинэчлэх
- WSL болон Windows хоёр замыг автоматаар хайна (`~/.claude/`, `/mnt/c/Users/*/.claude/`)

**Эсрэг чиглэл (агент → санг):**
- Агент (Cline, Claude Code г.м.) прокси руу хүсэлт илгээхэд прокси зүрхний цохилтод тухайн клиентийн үйлчилгээ·загвар мэдээллийг оруулна
- Сангийн хяналтын самбарын агент картад одоо ашиглаж байгаа үйлчилгээ/загвар бодит цагаар харагдана

> 💡 **Гол зүйл**: Прокси хүсэлтийн Authorization токеноор агентыг таньж, сангт тохируулсан үйлчилгээ/загварт автоматаар чиглүүлнэ. Cline эсвэл Claude Code өөр загварын нэр илгээсэн ч прокси сангийн тохиргоогоор орлуулна.

### VS Code дээр Cline ашиглах — Дэлгэрэнгүй зааварчилгаа

#### Алхам 1: Cline суулгах

VS Code өргөтгөлийн зах зээлээс **Cline** (ID: `saoudrizwan.claude-dev`)-ийг суулгана.

#### Алхам 2: Сангд агент бүртгэх

1. Сангийн хяналтын самбар (`http://сангийн-IP:56243`) нээнэ
2. **Агент** хэсэгт **+ Нэмэх** дарна
3. Дараах байдлаар оруулна:

| Талбар | Утга | Тайлбар |
|------|----|------|
| ID | `миний_cline` | Давтагдашгүй танигч (англи үсэг, зай байхгүй) |
| Нэр | `Миний Cline` | Хяналтын самбарт харуулах нэр |
| Агентын төрөл | `cline` | ← заавал `cline` сонгоно |
| Үйлчилгээ | Ашиглах үйлчилгээ сонгоно (жишээ: `google`) | |
| Загвар | Ашиглах загвар оруулна (жишээ: `gemini-2.5-flash`) | |

4. **Хадгалах** дарахад токен автоматаар үүсгэгдэнэ

#### Алхам 3: Cline холбох

**Арга А — Автомат хэрэглэх (санал болгох)**

1. Тухайн машин дээр wall-vault **прокси** ажиллаж байгаа эсэхийг шалгана (`localhost:56244`)
2. Хяналтын самбарын агент картад байгаа **⚡ Cline тохиргоо хэрэглэх** товчийг дарна
3. "Тохиргоо хэрэглэсэн!" мэдэгдэл гарч ирвэл амжилттай
4. VS Code-ийг дахин ачаална (`Ctrl+Alt+R`)

**Арга Б — Гараар тохируулах**

Cline-ийн хажуу мөрний тохиргоо (⚙️)-г нээнэ:
- **API Provider**: `OpenAI Compatible`
- **Base URL**: `http://прокси-хаяг:56244/v1`
  - Ижил машинд: `http://localhost:56244/v1`
  - Өөр машинд (жишээ: Mini сервер): `http://192.168.0.6:56244/v1`
- **API Key**: Сангаас олгосон токен (агент картаас хуулна)
- **Model ID**: Сангт тохируулсан загвар (жишээ: `gemini-2.5-flash`)

#### Алхам 4: Шалгах

Cline-ийн чат хайрцагт ямар нэг мессеж илгээнэ. Хэвийн бол:
- Сангийн хяналтын самбарын тухайн агент картад **ногоон цэг (● Ажиллаж буй)** харагдана
- Картад одоогийн үйлчилгээ/загвар харагдана (жишээ: `google / gemini-2.5-flash`)

#### Загвар солих

Cline-ийн загварыг солихыг хүсвэл **сангийн хяналтын самбар** дээрээс өөрчилнө:

1. Агент картын үйлчилгээ/загвар дусалт цэсийг өөрчилнө
2. **Хэрэглэх** дарна
3. VS Code дахин ачаалах (`Ctrl+Alt+R`) — Cline-ийн доод хэсгийн загварын нэр шинэчлэгдэнэ
4. Дараагийн хүсэлтээс шинэ загвар ашиглагдана

> 💡 Бодит байдал дээр прокси Cline-ийн хүсэлтийг токеноор таньж сангийн тохиргооны загварт чиглүүлнэ. VS Code дахин ачаалахгүй ч **бодитоор ашиглагдаж буй загвар шууд солигдоно** — дахин ачаалах нь Cline UI-ийн загварын нэрийг шинэчлэхэд л зориулагдсан.

#### Холболт тасрахыг мэдрэх

VS Code-ийг хаахад сангийн хяналтын самбар дахь агент карт ойролцоогоор **90 секундын** дараа шар (удааширсан), **3 минутын** дараа улаан (офлайн) болно. (v0.1.18-аас 15 секундын давтамжтай төлөвийн шалгалтаар офлайн мэдрэх хурд нэмэгдсэн.)

#### Алдаа засах

| Шинж тэмдэг | Шалтгаан | Шийдэл |
|------|------|------|
| Cline дээр "холбогдох амжилтгүй" алдаа | Прокси ажиллахгүй эсвэл хаяг буруу | `curl http://localhost:56244/health`-аар прокси шалгах |
| Сангт ногоон цэг харагдахгүй | API түлхүүр (токен) тохируулаагүй | **⚡ Cline тохиргоо хэрэглэх** товчийг дахин дарна |

### Үйлчилгээний карт

Ашиглах AI үйлчилгээг асааж унтраах эсвэл тохируулах карт.

- Үйлчилгээ тус бүрийн идэвхжүүлэх·идэвхгүй болгох товч
- Локал AI серверийн (компьютерт ажиллаж буй Ollama, LM Studio, vLLM г.м.) хаягийг оруулахад ашиглах боломжтой загваруудыг автоматаар олно.
- **Локал үйлчилгээний холболтын төлөв**: Үйлчилгээний нэрний хажууд ● цэг **ногоон** бол холбогдсон, **саарал** бол холбогдоогүй
- **Локал үйлчилгээний автомат гэрэл дохио** (v0.1.23+): Локал үйлчилгээнүүд (Ollama, LM Studio, vLLM) холбогдох боломжтой эсэхээс хамаарч автоматаар идэвхжинэ/идэвхгүй болно. Үйлчилгээ асаахад 15 секундийн дотор ● ногоон болж чеклэгч хайрцаг асана; үйлчилгээ унтраахад автоматаар унтарна. Клоуд үйлчилгээ (Google, OpenRouter г.м.) API түлхүүр байгаа/үгүйгээс хамаарч автомат шилжих аргатай адил.

> 💡 **Локал үйлчилгээ өөр компьютер дээр ажиллаж байгаа бол**: Үйлчилгээний URL оруулах хэсэгт тухайн компьютерын IP-г оруулна. Жишээ: `http://192.168.0.6:11434` (Ollama), `http://192.168.0.6:1234` (LM Studio). Үйлчилгээ `0.0.0.0` биш `127.0.0.1` дээр бинд хийгдсэн бол гадна IP-ээр хандах боломжгүй учир үйлчилгээний тохиргоонд бинд хаягийг шалгана уу.

## Тархсан горим (олон бот)

Олон компьютер дээр OpenClaw-ийг зэрэг ажиллуулахад **нэг түлхүүрийн сангийг хуваалцах** бүтэц. Түлхүүрийн удирдлагыг нэг дороос хийхэд тав тух юм.

### Бүтцийн жишээ

```
[Түлхүүрийн сангийн сервер]
  wall-vault vault    (түлхүүрийн санг :56243, хяналтын самбар)

[WSL Альфа]            [Raspberry Pi Гамма]    [Mac Mini Локал]
  wall-vault proxy      wall-vault proxy        wall-vault proxy
  openclaw TUI          openclaw TUI            openclaw TUI
  ↕ SSE синхрончлол     ↕ SSE синхрончлол        ↕ SSE синхрончлол
```

Бүх бот дундын сангийн сервер рүү харж байгаа учир сангаас загвар солих эсвэл түлхүүр нэмэхэд бүх ботод шууд тусгалдана.

### Алхам 1: Түлхүүрийн сангийн сервер эхлүүлэх

Сангийн серверийн үүрэг гүйцэтгэх компьютер дээр ажиллуулна:

```bash
wall-vault vault
```

### Алхам 2: Бот тус бүрийг (клиент) бүртгэх

Сангийн серверт холбогдох бот тус бүрийн мэдээллийг урьдчилж бүртгэнэ:

```bash
curl -X POST http://localhost:56243/admin/clients \
  -H "Authorization: Bearer администраторын-токен" \
  -H "Content-Type: application/json" \
  -d '{
    "id": "botA",
    "name": "БотА",
    "token": "bota-secret",
    "default_service": "google",
    "default_model": "gemini-2.5-flash"
  }'
```

### Алхам 3: Бот тус бүрийн компьютер дээр прокси эхлүүлэх

Бот суулгасан компьютер тус бүр дээр сангийн серверийн хаяг, токен заагаж прокси ажиллуулна:

```bash
WV_VAULT_URL=http://192.168.x.x:56243 \
WV_VAULT_TOKEN=bota-secret \
WV_VAULT_CLIENT_ID=botA \
wall-vault proxy
```

> 💡 **`192.168.x.x`** хэсгийг сангийн сервер компьютерын бодит дотоод IP хаягаар солино. Чиглүүлэгчийн тохиргоо эсвэл `ip addr` командаар шалгаж болно.

## Автомат эхлүүлэх тохиргоо

Компьютер дахин асаах бүрт wall-vault-ийг гараар асаах нь төвөгтэй бол системийн үйлчилгээ болгон бүртгэнэ. Нэг бүртгэвэл boot үед автоматаар эхэлнэ.

### Linux — systemd (ихэнх Linux)

systemd нь Linux дээр програмыг автоматаар эхлүүлж удирдах систем:

```bash
wall-vault doctor deploy
systemctl --user daemon-reload
systemctl --user enable --now wall-vault
```

Лог шалгах:

```bash
journalctl --user -u wall-vault -f
```

### macOS — launchd

macOS дээр програмыг автоматаар ажиллуулах систем:

```bash
wall-vault doctor deploy launchd
launchctl load ~/Library/LaunchAgents/com.wall-vault.plist
```

### Windows — NSSM

1. [nssm.cc](https://nssm.cc/download)-ээс NSSM татаж PATH-д нэмнэ.
2. Администраторын PowerShell-д:

```powershell
wall-vault doctor deploy windows
```

## Doctor оношлогч

`doctor` команд нь wall-vault-ийн **өөрийгөө оношлож засах хэрэгсэл** юм.

```bash
wall-vault doctor check   # Одоогийн төлвийг оношлох (зөвхөн унших, юу ч өөрчлөхгүй)
wall-vault doctor fix     # Асуудлыг автоматаар засах
wall-vault doctor all     # Оношилгоо + автомат засвар нэг дор
```

> 💡 Ямар нэг зүйл буруу юм шиг санагдвал `wall-vault doctor all`-ийг эхлээд ажиллуулна уу. Олон асуудлыг автоматаар барьж засна.

## Орчны хувьсагч лавлах

Орчны хувьсагч нь програмд тохиргооны утга дамжуулах арга юм. Терминалд `export хувьсагчийн-нэр=утга` хэлбэрээр оруулна, эсвэл автомат эхлүүлэх үйлчилгээний файлд оруулж байнга хэрэглэж болно.

| Хувьсагч | Тайлбар | Жишээ утга |
|------|------|---------|
| `WV_LANG` | Хяналтын самбарын хэл | `ko`, `en`, `ja` |
| `WV_THEME` | Хяналтын самбарын загвар | `light`, `dark`, `gold` |
| `WV_KEY_GOOGLE` | Google API түлхүүр (таслалаар олон) | `AIza...,AIza...` |
| `WV_KEY_OPENROUTER` | OpenRouter API түлхүүр | `sk-or-v1-...` |
| `WV_VAULT_URL` | Тархсан горимд сангийн серверийн хаяг | `http://192.168.x.x:56243` |
| `WV_VAULT_TOKEN` | Клиент (бот) баталгаажуулах токен | `my-secret-token` |
| `WV_ADMIN_TOKEN` | Администраторын токен | `admin-token-here` |
| `WV_MASTER_PASS` | API түлхүүр шифрлэлтийн нууц үг | `my-password` |
| `WV_AVATAR` | Аватар зургийн файлын зам (`~/.openclaw/`-аас харьцангуй зам) | `workspace/avatars/avatar.png` |
| `OLLAMA_URL` | Ollama локал серверийн хаяг | `http://192.168.x.x:11434` |

## Алдаа засах

### Прокси эхлэхгүй байна

Порт аль хэдийн өөр програм ашиглаж байгаа тохиолдол нийтлэг.

```bash
ss -tlnp | grep 56244   # 56244 портыг хэн ашиглаж байгааг шалгах
wall-vault proxy --port 8080   # Өөр портын дугаараар эхлүүлэх
```

### API түлхүүрийн алдаа (429, 402, 401, 403, 582)

| Алдааны код | Утга | Шийдэл |
|----------|------|----------|
| **429** | Хүсэлт хэт олон (ашиглалтын хязгаар хэтэрсэн) | Түр хүлээх эсвэл өөр түлхүүр нэмэх |
| **402** | Төлбөр шаардлагатай эсвэл кредит дутмаг | Тухайн үйлчилгээнд кредит цэнэглэх |
| **401 / 403** | Түлхүүр буруу эсвэл зөвшөөрөл байхгүй | Түлхүүрийн утгыг дахин шалгаж дахин бүртгэх |
| **582** | Gateway хэт ачаалал (хөргөлт 5 минут) | 5 минутын дараа автоматаар нээгдэнэ |

```bash
# Бүртгэлтэй түлхүүрийн жагсаалт ба төлвийг шалгах
curl -H "Authorization: Bearer администраторын-токен" http://localhost:56243/admin/keys

# Түлхүүрийн ашиглалтын тоолуур дахин тохируулах
curl -X POST -H "Authorization: Bearer администраторын-токен" http://localhost:56243/admin/keys/reset
```

### Агент "Холбогдоогүй" гэж харагдаж байна

"Холбогдоогүй" гэдэг нь прокси процесс сангт дохио (heartbeat) илгээхгүй байгаа төлөв юм. **Тохиргоо хадгалагдаагүй гэсэн утга биш.** Прокси нь сангийн серверийн хаяг, токен мэддэг байж ажиллаж байх ёстой, тэгвэл холбогдсон төлөвт шилжинэ.

```bash
# Сангийн серверийн хаяг, токен, клиент ID заагаж прокси эхлүүлэх
WV_VAULT_URL=http://сангийн-серверийн-хаяг:56243 \
WV_VAULT_TOKEN=клиентийн-токен \
WV_VAULT_CLIENT_ID=клиентийн-ID \
wall-vault proxy
```

Холболт амжилттай болвол ойролцоогоор 20 секундийн дотор хяналтын самбарт 🟢 Ажиллаж буй болж хувирна.

### Ollama холбогдохгүй байна

Ollama нь компьютер дээрээ шууд AI ажиллуулах програм юм. Эхлээд Ollama ажиллаж байгаа эсэхийг шалгана.

```bash
curl http://localhost:11434/api/tags   # Загварын жагсаалт гарч ирвэл хэвийн
export OLLAMA_URL=http://192.168.x.x:11434   # Өөр компьютер дээр ажиллаж байвал
```

> ⚠️ Ollama хариу өгөхгүй бол `ollama serve` командаар эхлээд Ollama-г ажиллуулна.

> ⚠️ **Том загварууд удаан хариулна**: `qwen3.5:35b`, `deepseek-r1` зэрэг том загварууд хариу үүсгэхэд хэдэн минут зарцуулж болно. Хариу байхгүй юм шиг харагдаж байсан ч хэвийн боловсруулж байж болзошгүй учир хүлээнэ үү.
