## ما هو wall-vault؟

**wall-vault = وكيل ذكاء اصطناعي (Proxy) + خزينة مفاتيح API — مُصمَّم لـ OpenClaw**

لاستخدام خدمات الذكاء الاصطناعي، تحتاج إلى **مفتاح API** (تصريح دخول رقمي — وثيقة تُثبت أن لك الحق في استخدام الخدمة). لهذا المفتاح حدٌّ أقصى لعدد الاستخدامات اليومية، وإن لم يُحفظ بشكل صحيح فقد يتعرض للاختراق.

يحفظ wall-vault هذه المفاتيح في خزينة آمنة، ويعمل **وكيلاً (proxy)** بين OpenClaw وخدمات الذكاء الاصطناعي. ببساطة: يتصل OpenClaw بـ wall-vault فقط، ويتولى wall-vault كل شيء آخر تلقائياً.

المشكلات التي يحلّها wall-vault:

- **التبديل التلقائي بين المفاتيح**: إذا بلغ مفتاحٌ ما حدَّه أو جُمِّد مؤقتاً، ينتقل wall-vault بصمت إلى المفتاح التالي دون انقطاع في عمل OpenClaw.
- **التحويل التلقائي بين الخدمات (Fallback)**: إذا لم يستجب Google، يتحوَّل إلى OpenRouter، وإن تعذَّر ذلك يتحوَّل إلى Ollama (الذكاء الاصطناعي المحلي على جهازك). لا تنقطع الجلسة أبداً. عندما تتعافى الخدمة الأصلية، يُعاد التحويل إليها تلقائياً بدءاً من الطلب التالي (v0.1.18+).
- **المزامنة الفورية (SSE)**: تغيير النموذج في لوحة التحكم يظهر على شاشة OpenClaw خلال 1–3 ثوانٍ. SSE (Server-Sent Events) هي تقنية تُرسِل بها الخادم التحديثات فورياً إلى المستخدم.
- **التنبيهات الفورية**: تظهر أحداث مثل نفاد مفتاح أو عطل في الخدمة مباشرةً في الشريط السفلي لواجهة OpenClaw النصية (TUI).

> 💡 يمكنك أيضاً توصيل **Claude Code و Cursor و VS Code** بـ wall-vault، غير أن هدفه الأساسي هو العمل مع OpenClaw.

```
OpenClaw (واجهة TUI النصية)
        │
        ▼
  wall-vault proxy (:56244)   ← إدارة المفاتيح، التوجيه، الفولباك، الأحداث
        │
        ├─ Google Gemini API
        ├─ OpenRouter API (أكثر من 340 نموذجاً)
        └─ Ollama (على جهازك، الخيار الأخير)
```

## التثبيت

### Linux / macOS

افتح الطرفية (Terminal) والصق الأوامر التالية كما هي:

```bash
# Linux (حاسب عادي أو خادم — amd64)
curl -L https://github.com/sookmook/wall-vault/releases/latest/download/wall-vault-linux-amd64 \
  -o ~/.local/bin/wall-vault && chmod +x ~/.local/bin/wall-vault

# macOS Apple Silicon (أجهزة M1/M2/M3)
curl -L https://github.com/sookmook/wall-vault/releases/latest/download/wall-vault-darwin-arm64 \
  -o /usr/local/bin/wall-vault && chmod +x /usr/local/bin/wall-vault
```

- `curl -L ...` — تنزيل الملف من الإنترنت.
- `chmod +x` — منح الملف صلاحية التنفيذ. إن أغفلتَ هذه الخطوة، ستظهر رسالة "Permission denied".

### Windows

افتح PowerShell بصلاحيات المسؤول (Administrator) ونفِّذ الأوامر التالية:

```powershell
# تنزيل الملف
Invoke-WebRequest -Uri `
  "https://github.com/sookmook/wall-vault/releases/latest/download/wall-vault-windows-amd64.exe" `
  -OutFile "$env:LOCALAPPDATA\Programs\wall-vault\wall-vault.exe"

# إضافة المسار إلى PATH (يُطبَّق بعد إعادة تشغيل PowerShell)
$env:PATH += ";$env:LOCALAPPDATA\Programs\wall-vault"
```

> 💡 **ما هو PATH؟** هو قائمة المجلدات التي يبحث فيها الحاسب عن البرامج. عند إضافة wall-vault إلى PATH، يمكنك كتابة `wall-vault` في أي مكان وتشغيله مباشرةً.

### البناء من الكود المصدري (للمطورين)

ينطبق هذا فقط إذا كانت بيئة تطوير Go مثبَّتة لديك:

```bash
git clone https://github.com/sookmook/wall-vault
cd wall-vault
make build       # bin/wall-vault (الإصدار: v0.1.6.YYYYMMDD.HHmmss)
make install     # ~/.local/bin/wall-vault
```

> 💡 **إصدار مختوم بالتاريخ**: عند البناء بـ `make build`، يُولَّد الإصدار تلقائياً بصيغة تتضمن التاريخ والوقت مثل `v0.1.6.20260314.231308`. أما البناء المباشر بـ `go build ./...` فيُظهر الإصدار `"dev"` فقط.

## البدء الأول

### تشغيل معالج الإعداد (setup)

بعد التثبيت مباشرةً، شغِّل **معالج الإعداد** بالأمر أدناه؛ سيرشدك خطوةً بخطوة:

```bash
wall-vault setup
```

خطوات المعالج:

```
1. اختيار اللغة (10 لغات من بينها العربية)
2. اختيار المظهر (light / dark / gold / cherry / ocean)
3. وضع التشغيل — مستقل (standalone) أو موزع (distributed) مع أجهزة متعددة
4. اسم البوت — الاسم الذي يظهر في لوحة التحكم
5. إعداد المنافذ — الافتراضي: Proxy على 56244، الخزينة على 56243 (اضغط Enter للقبول)
6. اختيار خدمات الذكاء الاصطناعي — Google / OpenRouter / Ollama
7. إعداد فلتر أمان الأدوات
8. رمز المدير (Admin Token) — كلمة سر تحمي وظائف الإدارة، يمكن توليدها تلقائياً
9. كلمة مرور تشفير مفاتيح API — لتأمين إضافي اختياري
10. مسار حفظ ملف الإعداد
```

> ⚠️ **احتفظ برمز المدير في مكان آمن.** ستحتاجه لاحقاً عند إضافة مفاتيح أو تعديل الإعدادات من لوحة التحكم. إن نسيتَه، ستضطر إلى تعديل ملف الإعداد يدوياً.

عند اكتمال المعالج، يُنشأ ملف الإعداد `wall-vault.yaml` تلقائياً.

### التشغيل

```bash
wall-vault start
```

يبدأ خادمان في آنٍ واحد:

- **الوكيل (Proxy)** (`http://localhost:56244`) — يربط OpenClaw بخدمات الذكاء الاصطناعي
- **خزينة المفاتيح (Vault)** (`http://localhost:56243`) — إدارة مفاتيح API ولوحة التحكم

افتح `http://localhost:56243` في المتصفح لعرض لوحة التحكم فوراً.

## تسجيل مفاتيح API

هناك أربع طرق لتسجيل مفاتيح API. **للمبتدئين نوصي بالطريقة الأولى (متغيرات البيئة).**

### الطريقة 1: متغيرات البيئة (موصى بها — الأبسط)

متغيرات البيئة (Environment Variables) هي قيم مُعدَّة مسبقاً يقرأها البرنامج عند التشغيل. اكتبها في الطرفية هكذا:

```bash
# تسجيل مفتاح Google Gemini
export WV_KEY_GOOGLE=AIzaSy...

# تسجيل مفتاح OpenRouter
export WV_KEY_OPENROUTER=sk-or-v1-...

# التشغيل بعد التسجيل
wall-vault start
```

إن كان لديك عدة مفاتيح، اجمعها بفاصلة (,). سيستخدمها wall-vault بالتناوب تلقائياً (Round Robin):

```bash
export WV_KEY_GOOGLE=AIzaSy...,AIzaSy...,AIzaSy...
```

> 💡 **تلميح**: أمر `export` ينطبق على جلسة الطرفية الحالية فقط. لجعله دائماً بعد إعادة التشغيل، أضف السطر إلى ملف `~/.bashrc` أو `~/.zshrc`.

### الطريقة 2: واجهة لوحة التحكم (بالنقر)

1. افتح `http://localhost:56243` في المتصفح
2. في بطاقة **🔑 API Keys** بالأعلى، انقر `[+ إضافة]`
3. أدخل نوع الخدمة، قيمة المفتاح، التسمية (ملاحظة)، والحد اليومي، ثم احفظ

### الطريقة 3: REST API (للأتمتة والسكريبتات)

REST API هي طريقة تتبادل بها البرامج البيانات عبر HTTP. مفيدة للتسجيل الآلي بالسكريبتات:

```bash
curl -X POST http://localhost:56243/admin/keys \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "service": "google",
    "key": "AIzaSy...",
    "label": "المفتاح الرئيسي",
    "daily_limit": 1000
  }'
```

### الطريقة 4: علامة proxy (للاختبار السريع)

للاختبار المؤقت دون تسجيل رسمي. تختفي البيانات عند إيقاف البرنامج:

```bash
wall-vault proxy --key-google=AIzaSy... --key-openrouter=sk-or-...
```

## استخدام الوكيل (Proxy)

### الاستخدام من OpenClaw (الغرض الأساسي)

لإعداد OpenClaw بحيث يتصل بخدمات الذكاء الاصطناعي عبر wall-vault، افتح الملف `~/.openclaw/openclaw.json` وأضف ما يلي:

```json5
// ~/.openclaw/openclaw.json
{
  models: {
    providers: {
      "wall-vault": {
        baseUrl: "http://localhost:56244/v1",
        apiKey: "your-agent-token",   // رمز الوكيل في الخزينة
        api: "openai-completions",
        models: [
          { id: "wall-vault/gemini-2.5-flash" },
          { id: "wall-vault/gemini-2.5-pro" },
          { id: "wall-vault/hunter-alpha" },    // سياق مجاني 1M رمز
          { id: "wall-vault/claude-opus-4-6" }
        ]
      }
    }
  }
}
```

> 💡 **طريقة أسهل**: في لوحة التحكم، انقر زر **🦞 نسخ إعداد OpenClaw** في بطاقة الوكيل — يُنسخ إلى الحافظة مقتطف الإعداد مع الرمز والعنوان معبَّأَيْن مسبقاً. فقط الصقه.

**إلى أين يتوجه كل نموذج؟**

يحدد wall-vault الخدمة المستهدفة من اسم النموذج تلقائياً:

| صيغة النموذج | الخدمة المستهدفة |
|----------|--------------|
| `wall-vault/gemini-*` | Google Gemini مباشرةً |
| `wall-vault/gpt-*`، `wall-vault/o3`، `wall-vault/o4*` | OpenAI مباشرةً |
| `wall-vault/claude-*` | Anthropic عبر OpenRouter |
| `wall-vault/hunter-alpha`، `wall-vault/healer-alpha` | OpenRouter (سياق مجاني مليون رمز) |
| `wall-vault/kimi-*`، `wall-vault/glm-*`، `wall-vault/deepseek-*` | OpenRouter |
| `google/اسم-النموذج`، `openai/اسم-النموذج`، `anthropic/اسم-النموذج` | الخدمة المقابلة مباشرةً |
| `custom/google/اسم-النموذج`، `custom/openai/اسم-النموذج` | يُزيل `custom/` ويُعيد التوجيه |
| `اسم-النموذج:cloud` | يُزيل `:cloud` ويُوجّه عبر OpenRouter |

> 💡 **ما هو السياق (Context)؟** هو حجم المحادثة الذي يستطيع الذكاء الاصطناعي تذكُّره في المرة الواحدة. سياق 1M (مليون رمز) يعني أنه يمكن معالجة محادثات ووثائق طويلة جداً في طلب واحد.

### الاتصال المباشر بصيغة Gemini API (للأدوات الموجودة)

إن كانت لديك أداة تستخدم Gemini API مباشرةً، غيِّر العنوان فقط ليشير إلى wall-vault:

```bash
export ANTHROPIC_BASE_URL=http://localhost:56244/google
```

أو إن كانت الأداة تقبل تحديد URL مباشرةً:

```
http://localhost:56244/google/v1beta/models/gemini-2.5-flash:generateContent
```

### الاستخدام من OpenAI SDK (Python)

يمكن توصيل wall-vault في أكواد Python أيضاً — غيِّر `base_url` فقط:

```python
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:56244/v1",
    api_key="not-needed"  # wall-vault يتولى إدارة المفاتيح تلقائياً
)

response = client.chat.completions.create(
    model="google/gemini-2.5-flash",   # بصيغة provider/model
    messages=[{"role": "user", "content": "مرحباً"}]
)
```

### تغيير النموذج أثناء التشغيل

لتغيير نموذج الذكاء الاصطناعي وwall-vault يعمل:

```bash
# تغيير النموذج بطلب مباشر للوكيل
curl -X PUT http://localhost:56244/api/config/model \
  -H "Content-Type: application/json" \
  -d '{"service": "openrouter", "model": "anthropic/claude-3.5-sonnet"}'

# في الوضع الموزع: التغيير من الخزينة → يُطبَّق فوراً عبر SSE
curl -X PUT http://localhost:56243/admin/clients/YOUR-BOT-ID \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"default_service": "google", "default_model": "gemini-2.5-pro"}'
```

### عرض قائمة النماذج المتاحة

```bash
# عرض القائمة الكاملة
curl http://localhost:56244/api/models | python3 -m json.tool

# نماذج Google فقط
curl "http://localhost:56244/api/models?service=google"

# البحث بالاسم (مثال: نماذج تحتوي "claude")
curl "http://localhost:56244/api/models?q=claude"
```

**ملخص النماذج الرئيسية لكل خدمة:**

| الخدمة | النماذج الرئيسية |
|--------|----------|
| Google | gemini-2.5-pro, gemini-2.5-flash, gemini-2.5-flash-8b, gemini-2.0-flash |
| OpenAI | gpt-4o, gpt-4o-mini, o3, o1, o1-mini |
| OpenRouter | أكثر من 346 نموذجاً (Hunter Alpha 1M سياق مجاني، DeepSeek R1/V3، Qwen 2.5 وغيرها) |
| Ollama | اكتشاف تلقائي للخادم المحلي المثبَّت على جهازك |

## لوحة تحكم خزينة المفاتيح

افتح `http://localhost:56243` في المتصفح لعرض لوحة التحكم.

**تصميم الشاشة:**
- **الشريط العلوي الثابت (Topbar)**: الشعار، منتقي اللغة والمظهر، مؤشر حالة اتصال SSE
- **شبكة البطاقات**: بطاقات الوكلاء والخدمات ومفاتيح API مرتَّبة على شكل بلاط

### بطاقة مفاتيح API

بطاقة تُتيح إدارة جميع مفاتيح API المسجَّلة بنظرة واحدة:

- تعرض قائمة المفاتيح مُجمَّعةً حسب الخدمة.
- `today_usage`: عدد الرموز (توكنات) المُعالَجة بنجاح اليوم (الأحرف التي قرأها وكتبها الذكاء الاصطناعي)
- `today_attempts`: إجمالي عدد الطلبات اليوم (ناجحة + فاشلة)
- زر `[+ إضافة]` لتسجيل مفتاح جديد، و`✕` لحذف مفتاح.

> 💡 **ما هو الرمز (Token)؟** وحدة يستخدمها الذكاء الاصطناعي لمعالجة النصوص. يعادل تقريباً كلمة إنجليزية واحدة، أو حرفاً إلى حرفين عربيَّيْن. تُحتسب رسوم API عادةً بناءً على عدد الرموز.

### بطاقة الوكلاء (Agents)

بطاقة تعرض حالة البوتات (الوكلاء) المتصلة بوكيل wall-vault:

**تُعرَض حالة الاتصال بأربع درجات:**

| المؤشر | الحالة | المعنى |
|------|------|------|
| 🟢 | يعمل | الوكيل يعمل بشكل طبيعي |
| 🟡 | تأخر | يستجيب لكنه بطيء |
| 🔴 | غير متصل | الوكيل لا يستجيب |
| ⚫ | غير مُوصَّل / معطَّل | الوكيل لم يتصل بالخزينة قط، أو معطَّل |

**أزرار بطاقة الوكيل:**

عند تسجيل وكيل بتحديد **نوعه**، تظهر أزرار مساعدة مناسبة تلقائياً.

#### 🔘 زر نسخ الإعداد — ينشئ إعداد الاتصال تلقائياً

انقر الزر لنسخ مقتطف إعداد كامل إلى الحافظة، معبَّأً مسبقاً برمز الوكيل وعنوان الوكيل ومعلومات النموذج. فقط الصقه في الموقع المناسب حسب الجدول:

| الزر | نوع الوكيل | مكان اللصق |
|------|-------------|-------------|
| 🦞 نسخ إعداد OpenClaw | `openclaw` | `~/.openclaw/openclaw.json` |
| 🦀 نسخ إعداد NanoClaw | `nanoclaw` | `~/.openclaw/openclaw.json` |
| 🟠 نسخ إعداد Claude Code | `claude-code` | `~/.claude/settings.json` |
| ⌨ نسخ إعداد Cursor | `cursor` | Cursor → Settings → AI |
| 💻 نسخ إعداد VSCode | `vscode` | `~/.continue/config.json` |

**مثال — نوع Claude Code، المحتوى المنسوخ:**

```json
// ~/.claude/settings.json
{
  "apiProvider": "openai",
  "baseUrl": "http://192.168.1.20:56244/v1",
  "apiKey": "YOUR-AGENT-TOKEN"
}
```

**مثال — نوع VSCode (Continue):**

```json
// ~/.continue/config.json
{
  "models": [{
    "title": "wall-vault proxy",
    "provider": "openai",
    "model": "gemini-2.5-flash",
    "apiBase": "http://192.168.1.20:56244/v1",
    "apiKey": "YOUR-AGENT-TOKEN"
  }]
}
```

**مثال — نوع Cursor:**

```
Base URL : http://192.168.1.20:56244/v1
API Key  : YOUR-AGENT-TOKEN

// أو عبر متغيرات البيئة:
OPENAI_BASE_URL=http://192.168.1.20:56244/v1
OPENAI_API_KEY=YOUR-AGENT-TOKEN
```

> ⚠️ **إذا لم تنجح عملية النسخ**: قد تحجب سياسة أمان المتصفح النسخ أحياناً. إن ظهر مربع نص في نافذة منبثقة، اضغط Ctrl+A لتحديد الكل ثم Ctrl+C للنسخ.

#### ⚡ زر التطبيق التلقائي — اضغط مرة واحدة وينتهي الإعداد

إذا كان نوع الوكيل `cline` أو `claude-code` أو `openclaw` أو `nanoclaw`، يظهر زر **⚡ تطبيق الإعدادات** في بطاقة الوكيل. بالضغط عليه يُحدَّث ملف إعدادات الوكيل المحلي تلقائياً.

| الزر | نوع الوكيل | الملف المُعدَّل |
|------|-----------|----------------|
| ⚡ تطبيق إعدادات Cline | `cline` | `~/.cline/data/globalState.json` + `secrets.json` |
| ⚡ تطبيق إعدادات Claude Code | `claude-code` | `~/.claude/settings.json` |
| ⚡ تطبيق إعدادات OpenClaw | `openclaw` | `~/.openclaw/openclaw.json` |
| ⚡ تطبيق إعدادات NanoClaw | `nanoclaw` | `~/.openclaw/openclaw.json` |

> ⚠️ هذا الزر يرسل طلباً إلى **localhost:56244** (الوكيل المحلي). يجب أن يكون الوكيل قيد التشغيل على هذا الجهاز حتى يعمل.

#### 🔀 سحب وإسقاط لترتيب البطاقات (v0.1.17)

يمكنك **سحب** بطاقات الوكلاء في لوحة التحكم وإعادة ترتيبها كما تريد.

1. أمسك بطاقة الوكيل بالماوس واسحبها
2. أفلتها فوق البطاقة في الموضع المطلوب لتبديل الترتيب
3. يُحفظ الترتيب الجديد **فوراً على الخادم** ويبقى كما هو حتى بعد تحديث الصفحة

> 💡 الأجهزة التي تعمل باللمس (الهواتف/الأجهزة اللوحية) غير مدعومة بعد. يُرجى استخدام متصفح سطح المكتب.

#### 🔄 مزامنة النماذج ثنائية الاتجاه (v0.1.16)

عند تغيير نموذج الوكيل من لوحة تحكم الخزينة، تُحدَّث الإعدادات المحلية لذلك الوكيل تلقائياً.

**في حالة Cline:**
- تغيير النموذج في الخزينة → حدث SSE → يُحدِّث الوكيل حقل النموذج في `globalState.json`
- الحقول المُحدَّثة: `actModeOpenAiModelId`، `planModeOpenAiModelId`، `openAiModelId`
- لا يُعدَّل `openAiBaseUrl` ومفتاح API
- **يلزم إعادة تحميل VS Code (`Ctrl+Alt+R` أو `Ctrl+Shift+P` → `Developer: Reload Window`)**
  - لأن Cline لا يُعيد قراءة ملف الإعدادات أثناء التشغيل

**في حالة Claude Code:**
- تغيير النموذج في الخزينة → حدث SSE → يُحدِّث الوكيل حقل `model` في `settings.json`
- يبحث تلقائياً في مسارات WSL وWindows (`~/.claude/`، `/mnt/c/Users/*/.claude/`)

**الاتجاه العكسي (الوكيل → الخزينة):**
- عندما يُرسل الوكيل (Cline، Claude Code، إلخ) طلباً إلى البروكسي، يُضمِّن البروكسي معلومات الخدمة والنموذج الحالية في نبضة القلب
- تُعرض الخدمة/النموذج المستخدَم حالياً في الوقت الفعلي على بطاقة الوكيل في لوحة التحكم

> 💡 **الخلاصة**: يتعرف البروكسي على الوكيل من رمز Authorization في الطلب، ويوجّهه تلقائياً إلى الخدمة/النموذج المُعدَّيْن في الخزينة. حتى لو أرسل Cline أو Claude Code اسم نموذج مختلفاً، يُعيد البروكسي التوجيه وفقاً لإعدادات الخزينة.

### استخدام Cline في VS Code — دليل تفصيلي

#### الخطوة 1: تثبيت Cline

ثبِّت **Cline** (ID: `saoudrizwan.claude-dev`) من سوق إضافات VS Code.

#### الخطوة 2: تسجيل الوكيل في الخزينة

1. افتح لوحة تحكم الخزينة (`http://عنوان-الخزينة:56243`)
2. في قسم **الوكلاء**، انقر **+ إضافة**
3. أدخل البيانات التالية:

| الحقل | القيمة | الوصف |
|-------|--------|-------|
| ID | `my_cline` | معرِّف فريد (بالإنجليزية، بدون مسافات) |
| الاسم | `My Cline` | الاسم الظاهر في لوحة التحكم |
| نوع الوكيل | `cline` | ← يجب اختيار `cline` |
| الخدمة | اختر الخدمة المطلوبة (مثال: `google`) | |
| النموذج | أدخل النموذج المطلوب (مثال: `gemini-2.5-flash`) | |

4. اضغط **حفظ** وسيُنشأ الرمز تلقائياً

#### الخطوة 3: ربط Cline

**الطريقة أ — التطبيق التلقائي (موصى بها)**

1. تأكد أن **بروكسي** wall-vault يعمل على هذا الجهاز (`localhost:56244`)
2. انقر زر **⚡ تطبيق إعدادات Cline** في بطاقة الوكيل بلوحة التحكم
3. إذا ظهر إشعار "تم تطبيق الإعدادات!" فقد نجحت العملية
4. أعد تحميل VS Code (`Ctrl+Alt+R`)

**الطريقة ب — الإعداد اليدوي**

افتح إعدادات Cline (⚙️) من الشريط الجانبي وأدخل:
- **API Provider**: `OpenAI Compatible`
- **Base URL**: `http://عنوان-البروكسي:56244/v1`
  - إذا كان على نفس الجهاز: `http://localhost:56244/v1`
  - إذا كان على جهاز آخر (مثل Mac Mini): `http://192.168.1.20:56244/v1`
- **API Key**: الرمز الصادر من الخزينة (انسخه من بطاقة الوكيل)
- **Model ID**: النموذج المُعدّ في الخزينة (مثال: `gemini-2.5-flash`)

#### الخطوة 4: التحقق

أرسل أي رسالة في نافذة دردشة Cline. إذا كان كل شيء سليماً:
- تظهر **نقطة خضراء (● يعمل)** على بطاقة الوكيل في لوحة التحكم
- يُعرض الخدمة/النموذج الحالي على البطاقة (مثال: `google / gemini-2.5-flash`)

#### تغيير النموذج

لتغيير نموذج Cline، غيِّره من **لوحة تحكم الخزينة**:

1. غيِّر قائمة الخدمة/النموذج المنسدلة في بطاقة الوكيل
2. انقر **تطبيق**
3. أعد تحميل VS Code (`Ctrl+Alt+R`) — يتحدث اسم النموذج في تذييل Cline
4. من الطلب التالي فصاعداً سيُستخدم النموذج الجديد

> 💡 في الواقع، يُعرِّف البروكسي طلب Cline من الرمز ويوجّهه إلى نموذج الخزينة. حتى بدون إعادة تحميل VS Code، **النموذج المستخدَم فعلياً يتغير فوراً** — إعادة التحميل فقط لتحديث اسم النموذج المعروض في واجهة Cline.

#### كشف انقطاع الاتصال

عند إغلاق VS Code، تتحول بطاقة الوكيل في لوحة التحكم إلى اللون الأصفر (تأخر) بعد حوالي **90 ثانية**، وإلى اللون الأحمر (غير متصل) بعد **3 دقائق**. (منذ الإصدار v0.1.18، أصبح فحص الحالة كل 15 ثانية يتيح اكتشاف عدم الاتصال بشكل أسرع.)

#### استكشاف الأخطاء

| العَرَض | السبب | الحل |
|--------|-------|------|
| خطأ "فشل الاتصال" في Cline | البروكسي لا يعمل أو العنوان خاطئ | تحقق بـ `curl http://localhost:56244/health` |
| النقطة الخضراء لا تظهر في الخزينة | مفتاح API (الرمز) غير مُعدّ | اضغط زر **⚡ تطبيق إعدادات Cline** مرة أخرى |
| اسم النموذج في تذييل Cline لا يتغير | Cline يحتفظ بالإعدادات في الذاكرة المؤقتة | أعد تحميل VS Code (`Ctrl+Alt+R`) |
| يظهر اسم نموذج خاطئ | خطأ قديم (تم إصلاحه في v0.1.16) | حدِّث البروكسي إلى الإصدار v0.1.16 أو أحدث |

#### 🟣 زر نسخ أمر النشر — لتثبيت wall-vault على جهاز جديد

يُستخدم عند تثبيت وكيل wall-vault لأول مرة على حاسب جديد وربطه بالخزينة. انقر الزر لنسخ سكريبت التثبيت كاملاً، ثم الصقه في طرفية الجهاز الجديد وشغِّله. سيتم دفعةً واحدة:

1. تثبيت الملف التنفيذي wall-vault (يتخطى هذه الخطوة إن كان مثبَّتاً مسبقاً)
2. تسجيل خدمة systemd للمستخدم تلقائياً
3. تشغيل الخدمة والاتصال بالخزينة تلقائياً

> 💡 يتضمن السكريبت رمز هذا الوكيل وعنوان الخزينة معبَّأَيْن مسبقاً، لذا لا تحتاج إلى أي تعديل — فقط الصق وشغِّل.

---

### بطاقة الخدمات

بطاقة لتشغيل خدمات الذكاء الاصطناعي وإيقافها وإعدادها:

- مفاتيح تشغيل/إيقاف (Toggle) لكل خدمة
- أدخل عنوان خادم الذكاء الاصطناعي المحلي (Ollama أو LM Studio أو vLLM المثبَّت على جهازك) لاكتشاف النماذج المتاحة تلقائياً.
- **مؤشر حالة الاتصال بالخدمة المحلية**: النقطة ● بجانب اسم الخدمة **خضراء** إن كانت متصلة، **رمادية** إن كانت غير متصلة.
- **مزامنة تلقائية لمربعات الاختيار**: إن كانت خدمة محلية (مثل Ollama) تعمل عند فتح الصفحة، يُفعَّل مربع اختيارها تلقائياً.

> 💡 **إن كانت الخدمة المحلية تعمل على جهاز آخر**: أدخل عنوان IP ذلك الجهاز في حقل URL الخدمة. مثال: `http://192.168.1.20:11434` للـ Ollama، أو `http://192.168.1.20:1234` للـ LM Studio.

### إدخال رمز المدير

عند محاولة تنفيذ إجراءات مهمة كإضافة المفاتيح أو حذفها من لوحة التحكم، تظهر نافذة منبثقة لإدخال رمز المدير. أدخل الرمز الذي أعددته في معالج الإعداد. يبقى الرمز محفوظاً حتى تغلق المتصفح.

> ⚠️ **إن تجاوزت 10 محاولات فاشلة للمصادقة خلال 15 دقيقة، سيُحظر عنوان IP مؤقتاً.** إن نسيت رمزك، ابحث عن حقل `admin_token` في ملف `wall-vault.yaml`.

## الوضع الموزع (متعدد البوتات)

عند تشغيل OpenClaw على عدة أجهزة في آنٍ واحد، يمكنك **مشاركة خزينة مفاتيح واحدة** بينها. هكذا تدير المفاتيح من مكان واحد فقط.

### مثال على التكوين

```
[خادم الخزينة]
  wall-vault vault    (خزينة :56243، لوحة التحكم)

[WSL ألفا]            [Raspberry Pi غاما]    [Mac Mini محلي]
  wall-vault proxy      wall-vault proxy        wall-vault proxy
  openclaw TUI          openclaw TUI            openclaw TUI
  ↕ مزامنة SSE          ↕ مزامنة SSE            ↕ مزامنة SSE
```

جميع البوتات تتطلع إلى خادم الخزينة المركزي، لذا أي تغيير في النموذج أو إضافة مفتاح يُطبَّق على الجميع فوراً.

### الخطوة 1: تشغيل خادم الخزينة

على الجهاز المخصص للخزينة:

```bash
wall-vault vault
```

### الخطوة 2: تسجيل كل بوت (عميل)

سجِّل معلومات كل بوت سيتصل بخادم الخزينة مسبقاً:

```bash
curl -X POST http://localhost:56243/admin/clients \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "id": "botA",
    "name": "Bot A",
    "token": "bota-secret",
    "default_service": "google",
    "default_model": "gemini-2.5-flash"
  }'
```

### الخطوة 3: تشغيل الوكيل على كل جهاز بوت

على كل جهاز بوت، شغِّل الوكيل مع تحديد عنوان خادم الخزينة والرمز:

```bash
WV_VAULT_URL=http://192.168.x.x:56243 \
WV_VAULT_TOKEN=bota-secret \
WV_VAULT_CLIENT_ID=botA \
wall-vault proxy
```

> 💡 استبدل **`192.168.x.x`** بعنوان IP الداخلي الفعلي لجهاز الخزينة. يمكنك معرفته من إعدادات الراوتر أو بأمر `ip addr`.

## إعداد التشغيل التلقائي

إن أردت تجنب تشغيل wall-vault يدوياً في كل مرة، سجِّله كخدمة نظام. مرةً واحدة تكفي ليبدأ تلقائياً مع كل تشغيل للجهاز.

### Linux — systemd (معظم توزيعات Linux)

systemd هو نظام إدارة الخدمات في Linux الذي يتولى تشغيلها تلقائياً:

```bash
wall-vault doctor deploy
systemctl --user daemon-reload
systemctl --user enable --now wall-vault
```

لعرض السجلات:

```bash
journalctl --user -u wall-vault -f
```

### macOS — launchd

launchd هو النظام المسؤول عن تشغيل البرامج تلقائياً في macOS:

```bash
wall-vault doctor deploy launchd
launchctl load ~/Library/LaunchAgents/com.wall-vault.plist
```

### Windows — NSSM

1. نزِّل NSSM من [nssm.cc](https://nssm.cc/download) وأضفه إلى PATH.
2. في PowerShell بصلاحيات المسؤول:

```powershell
wall-vault doctor deploy windows
```

## أداة Doctor للتشخيص

أمر `doctor` هو **أداة تشخيص ذاتي وإصلاح تلقائي** تتحقق من صحة إعداد wall-vault:

```bash
wall-vault doctor check   # تشخيص الحالة الراهنة (قراءة فقط، لا يُغيِّر شيئاً)
wall-vault doctor fix     # إصلاح المشكلات تلقائياً
wall-vault doctor all     # تشخيص + إصلاح في خطوة واحدة
```

> 💡 إن شعرت بأن شيئاً ما خطأ، جرِّب `wall-vault doctor all` أولاً. يحل كثيراً من المشكلات تلقائياً.

## مرجع متغيرات البيئة

متغيرات البيئة طريقة لتمرير الإعدادات إلى البرنامج. اكتبها بصيغة `export VARIABLE=value` في الطرفية، أو ضعها في ملف خدمة التشغيل التلقائي لتطبيقها دائماً.

| المتغير | الوصف | مثال على القيمة |
|------|------|---------|
| `WV_LANG` | لغة لوحة التحكم | `ko`، `en`، `ja` |
| `WV_THEME` | مظهر لوحة التحكم | `light`، `dark`، `gold` |
| `WV_KEY_GOOGLE` | مفتاح Google API (عدة مفاتيح تفصل بفاصلة) | `AIza...,AIza...` |
| `WV_KEY_OPENROUTER` | مفتاح OpenRouter API | `sk-or-v1-...` |
| `WV_VAULT_URL` | عنوان خادم الخزينة في الوضع الموزع | `http://192.168.x.x:56243` |
| `WV_VAULT_TOKEN` | رمز مصادقة العميل (البوت) | `my-secret-token` |
| `WV_ADMIN_TOKEN` | رمز المدير | `admin-token-here` |
| `WV_MASTER_PASS` | كلمة مرور تشفير مفاتيح API | `my-password` |
| `WV_AVATAR` | مسار صورة الأفاتار (نسبي من `~/.openclaw/`) | `workspace/avatars/avatar.png` |
| `OLLAMA_URL` | عنوان خادم Ollama المحلي | `http://192.168.x.x:11434` |

## حل المشكلات الشائعة

### الوكيل لا يبدأ

غالباً يكون المنفذ مستخدَماً بالفعل من برنامج آخر:

```bash
ss -tlnp | grep 56244   # معرفة من يستخدم المنفذ 56244
wall-vault proxy --port 8080   # التشغيل على منفذ آخر
```

### أخطاء مفاتيح API (429، 402، 401، 403، 582)

| كود الخطأ | المعنى | الحل |
|----------|------|----------|
| **429** | طلبات كثيرة جداً (تجاوز الحد) | انتظر قليلاً أو أضف مفتاحاً آخر |
| **402** | مطلوب دفع أو رصيد غير كافٍ | اشحن رصيدك في الخدمة المعنية |
| **401 / 403** | المفتاح خاطئ أو لا صلاحية له | تحقق من قيمة المفتاح وأعد تسجيله |
| **582** | ضغط زائد على البوابة (تبريد 5 دقائق) | يُرفع تلقائياً بعد 5 دقائق |

```bash
# عرض قائمة المفاتيح المسجَّلة وحالتها
curl -H "Authorization: Bearer YOUR_ADMIN_TOKEN" http://localhost:56243/admin/keys

# إعادة تعيين عداد استخدام المفاتيح
curl -X POST -H "Authorization: Bearer YOUR_ADMIN_TOKEN" http://localhost:56243/admin/keys/reset
```

### الوكيل يظهر بحالة "غير مُوصَّل"

"غير مُوصَّل" يعني أن عملية الوكيل لا ترسل إشارات نبضة (Heartbeat) إلى الخزينة. **لا يعني ذلك أن الإعدادات ضاعت.** يجب أن يعمل الوكيل مع تحديد عنوان الخزينة ورمزها لتتغير الحالة إلى متصل.

```bash
# تشغيل الوكيل مع تحديد عنوان الخزينة والرمز ومعرِّف العميل
WV_VAULT_URL=http://VAULT-SERVER-ADDRESS:56243 \
WV_VAULT_TOKEN=CLIENT-TOKEN \
WV_VAULT_CLIENT_ID=CLIENT-ID \
wall-vault proxy
```

عند نجاح الاتصال، تتغير الحالة في لوحة التحكم إلى 🟢 يعمل خلال نحو 20 ثانية.

### مشكلة في الاتصال بـ Ollama

Ollama برنامج يُشغِّل الذكاء الاصطناعي مباشرةً على جهازك. تأكد أولاً أن Ollama يعمل:

```bash
curl http://localhost:11434/api/tags   # إن ظهرت قائمة النماذج فكل شيء سليم
export OLLAMA_URL=http://192.168.x.x:11434   # إن كان يعمل على جهاز آخر
```

> ⚠️ إن لم يستجب Ollama، شغِّله أولاً بأمر `ollama serve`.

> ⚠️ **النماذج الكبيرة بطيئة**: نماذج من قبيل `qwen3.5:35b` و`deepseek-r1` قد تستغرق دقائق لتوليد الاستجابة. إن بدا أنها لا تستجيب، فقد تكون لا تزال تعالج الطلب — انتظر بصبر.
