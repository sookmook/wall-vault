## wall-vault के हो?

**wall-vault = OpenClaw का लागि AI मध्यस्थ (प्रोक्सी) + API कुञ्जी तिजोरी**

AI सेवाहरू प्रयोग गर्न **API कुञ्जी** (डिजिटल प्रवेश पास — "यो व्यक्तिलाई यो सेवा प्रयोग गर्ने अधिकार छ" भनेर प्रमाणित गर्ने विशेष कोड) आवश्यक पर्दछ। यो प्रवेश पासमा दैनिक प्रयोगको सीमा हुन्छ र राम्रोसँग सुरक्षित नगरे चोरी हुने जोखिम पनि हुन्छ।

wall-vault यी प्रवेश पासहरूलाई सुरक्षित तिजोरीमा राख्छ र OpenClaw तथा AI सेवाहरूबीच **मध्यस्थ (प्रोक्सी)** को काम गर्दछ। सजिलो भाषामा भन्दा, OpenClaw लाई केवल wall-vault सँग जोड्नुपर्छ, बाँकी जटिल काम wall-vault ले आफैँ सम्हाल्छ।

wall-vault ले समाधान गर्ने समस्याहरू:

- **API कुञ्जी स्वचालित घुमाउने**: एउटा कुञ्जीको सीमा सकियो वा अस्थायी रूपमा अवरुद्ध भयो (कुलडाउन) भने, चुपचाप अर्को कुञ्जीमा सर्छ। OpenClaw बिना अवरोध चलिरहन्छ।
- **सेवा स्वचालित बदल्ने (फलब्याक)**: Google ले जवाफ नदिए OpenRouter मा, त्यो पनि नभए मेरो कम्प्युटरमा स्थापित Ollama (स्थानीय AI) मा स्वचालित रूपमा सर्छ। सत्र टुटँदैन।
- **रियल-टाइम समन्वय (SSE)**: तिजोरी ड्यासबोर्डमा मोडेल बदल्दा १–३ सेकेन्डभित्र OpenClaw स्क्रिनमा देखिन्छ। SSE (Server-Sent Events) भनेको सर्भरले परिवर्तनहरू तुरुन्तै क्लाइन्टलाई पठाउने प्रविधि हो।
- **रियल-टाइम सूचना**: कुञ्जी सकिनु वा सेवा बन्द हुनु जस्ता घटनाहरू OpenClaw TUI (टर्मिनल स्क्रिन) को तल्लो भागमा तुरुन्तै देखिन्छन्।

> 💡 **Claude Code, Cursor, VS Code** पनि जोड्न सकिन्छ, तर wall-vault को मूल उद्देश्य OpenClaw सँग मिलेर प्रयोग गर्नु हो।

```
OpenClaw (TUI टर्मिनल स्क्रिन)
        │
        ▼
  wall-vault प्रोक्सी (:56244)   ← कुञ्जी व्यवस्थापन, राउटिङ, फलब्याक, घटना
        │
        ├─ Google Gemini API
        ├─ OpenRouter API (३४०+ मोडेल)
        └─ Ollama (मेरो कम्प्युटर, अन्तिम विकल्प)
```

## स्थापना

### Linux / macOS

टर्मिनल खोल्नुहोस् र तलको आदेश सिधै टाँस्नुहोस्।

```bash
# Linux (सामान्य PC, सर्भर — amd64)
curl -L https://github.com/sookmook/wall-vault/releases/latest/download/wall-vault-linux-amd64 \
  -o ~/.local/bin/wall-vault && chmod +x ~/.local/bin/wall-vault

# macOS Apple Silicon (M1/M2/M3 Mac)
curl -L https://github.com/sookmook/wall-vault/releases/latest/download/wall-vault-darwin-arm64 \
  -o /usr/local/bin/wall-vault && chmod +x /usr/local/bin/wall-vault
```

- `curl -L ...` — इन्टरनेटबाट फाइल डाउनलोड गर्दछ।
- `chmod +x` — डाउनलोड गरेको फाइललाई "चलाउन मिल्ने" बनाउँछ। यो चरण छोड्दा "अनुमति छैन" भन्ने त्रुटि आउँछ।

### Windows

PowerShell (प्रशासक अधिकार सहित) खोल्नुहोस् र तलको आदेश चलाउनुहोस्।

```powershell
# डाउनलोड
Invoke-WebRequest -Uri `
  "https://github.com/sookmook/wall-vault/releases/latest/download/wall-vault-windows-amd64.exe" `
  -OutFile "$env:LOCALAPPDATA\Programs\wall-vault\wall-vault.exe"

# PATH मा थप्नुहोस् (PowerShell पुनःसुरु गरेपछि लागू हुन्छ)
$env:PATH += ";$env:LOCALAPPDATA\Programs\wall-vault"
```

> 💡 **PATH के हो?** कम्प्युटरले आदेश खोज्ने फोल्डरहरूको सूची हो। PATH मा थपेपछि मात्र जुनसुकै फोल्डरबाट `wall-vault` टाइप गरेर चलाउन सकिन्छ।

### स्रोतबाट आफैँ बनाउनुहोस् (विकासकर्ताका लागि)

Go भाषाको विकास वातावरण स्थापित भएको अवस्थामा मात्र लागू हुन्छ।

```bash
git clone https://github.com/sookmook/wall-vault
cd wall-vault
make build       # bin/wall-vault (संस्करण: v0.1.6.YYYYMMDD.HHmmss)
make install     # ~/.local/bin/wall-vault
```

> 💡 **बिल्ड टाइमस्ट्याम्प संस्करण**: `make build` ले बनाउँदा संस्करण `v0.1.6.20260314.231308` जस्तो मिति र समय समावेश भएको ढाँचामा स्वचालित रूपमा बन्छ। `go build ./...` बाट सिधै बनाउँदा संस्करण केवल `"dev"` देखिन्छ।

## पहिलो पटक सुरु गर्नुहोस्

### setup जादुगर चलाउनुहोस्

स्थापना पछि पहिलो पटक अनिवार्य रूपमा तलको आदेशले **सेटअप जादुगर** चलाउनुहोस्। जादुगरले आवश्यक विवरणहरू एक एक गरेर सोध्दै मार्गदर्शन गर्छ।

```bash
wall-vault setup
```

जादुगरले गर्ने चरणहरू यस्ता छन्:

```
१. भाषा छनोट (नेपाली सहित १० भाषाहरू)
२. थिम छनोट (light / dark / gold / cherry / ocean)
३. सञ्चालन मोड — एक्लै प्रयोग (standalone) वा धेरै कम्प्युटरमा सँगै (distributed) छनोट
४. बटको नाम — ड्यासबोर्डमा देखिने नाम
५. पोर्ट सेटिङ — पूर्वनिर्धारित: प्रोक्सी ५६२४४, तिजोरी ५६२४३ (नबदल्ने हो भने Enter थिच्नुहोस्)
६. AI सेवा छनोट — Google / OpenRouter / Ollama मध्ये प्रयोग गर्ने सेवा
७. उपकरण सुरक्षा फिल्टर सेटिङ
८. प्रशासक टोकन सेटिङ — ड्यासबोर्ड व्यवस्थापन कार्यलाई लक गर्ने पासवर्ड, स्वचालित उत्पन्न पनि गर्न सकिन्छ
९. API कुञ्जी इन्क्रिप्सन पासवर्ड — कुञ्जीलाई थप सुरक्षित राख्न चाहनुभयो भने (ऐच्छिक)
१०. कन्फिगरेसन फाइल सुरक्षित गर्ने स्थान
```

> ⚠️ **प्रशासक टोकन जरुर याद राख्नुहोस्।** पछि ड्यासबोर्डमा कुञ्जी थप्न वा सेटिङ बदल्न आवश्यक पर्छ। बिर्सनुभयो भने कन्फिगरेसन फाइल सिधै सम्पादन गर्नुपर्ने हुन्छ।

जादुगर सम्पन्न भएपछि `wall-vault.yaml` कन्फिगरेसन फाइल स्वचालित रूपमा बन्छ।

### चलाउनुहोस्

```bash
wall-vault start
```

तलका दुई सर्भर एकसाथ सुरु हुन्छन्:

- **प्रोक्सी** (`http://localhost:56244`) — OpenClaw र AI सेवाबीच जोड्ने मध्यस्थ
- **कुञ्जी तिजोरी** (`http://localhost:56243`) — API कुञ्जी व्यवस्थापन तथा वेब ड्यासबोर्ड

ब्राउजरमा `http://localhost:56243` खोल्नुहोस् र ड्यासबोर्ड तुरुन्तै हेर्न सकिन्छ।

## API कुञ्जी दर्ता

API कुञ्जी दर्ता गर्ने चार तरिका छन्। **पहिलो पटक सुरु गर्नेहरूलाई तरिका १ (वातावरण चर) सिफारिस गरिन्छ।**

### तरिका १: वातावरण चर (सिफारिस — सबैभन्दा सजिलो)

वातावरण चर (environment variable) भनेको प्रोग्राम सुरु हुँदा पढिने **अगाडि नै सेट गरेको मान** हो। टर्मिनलमा तल जस्तो टाइप गर्नुहोस्:

```bash
# Google Gemini कुञ्जी दर्ता
export WV_KEY_GOOGLE=AIzaSy...

# OpenRouter कुञ्जी दर्ता
export WV_KEY_OPENROUTER=sk-or-v1-...

# दर्ता पछि चलाउनुहोस्
wall-vault start
```

धेरै कुञ्जी भएमा अल्पविराम (,) ले जोड्नुहोस्। wall-vault ले कुञ्जीहरू पालैपालो स्वचालित रूपमा प्रयोग गर्छ (राउन्ड रोबिन):

```bash
export WV_KEY_GOOGLE=AIzaSy...,AIzaSy...,AIzaSy...
```

> 💡 **सुझाव**: `export` आदेश केवल हालको टर्मिनल सत्रमा मात्र लागू हुन्छ। कम्प्युटर पुनःसुरु गरेपछि पनि कायम राख्न `~/.bashrc` वा `~/.zshrc` फाइलमा माथिको लाइन थप्नुहोस्।

### तरिका २: ड्यासबोर्ड UI (माउसले क्लिक गरेर)

१. ब्राउजरमा `http://localhost:56243` खोल्नुहोस्
२. माथिको **🔑 API कुञ्जी** कार्डमा `[+ थप्नुहोस्]` बटन थिच्नुहोस्
३. सेवाको प्रकार, कुञ्जी मान, लेबल (टिप्पणी नाम), र दैनिक सीमा भरेर सुरक्षित गर्नुहोस्

### तरिका ३: REST API (स्वचालन र स्क्रिप्टका लागि)

REST API भनेको प्रोग्रामहरूबीच HTTP मार्फत डेटा आदानप्रदान गर्ने तरिका हो। स्क्रिप्टबाट स्वचालित दर्ता गर्दा उपयोगी छ।

```bash
curl -X POST http://localhost:56243/admin/keys \
  -H "Authorization: Bearer 관리자토큰" \
  -H "Content-Type: application/json" \
  -d '{
    "service": "google",
    "key": "AIzaSy...",
    "label": "메인 키",
    "daily_limit": 1000
  }'
```

### तरिका ४: proxy फ्ल्याग (छोटो परीक्षणका लागि)

औपचारिक दर्ता नगरी अस्थायी रूपमा कुञ्जी राखेर परीक्षण गर्न प्रयोग गरिन्छ। प्रोग्राम बन्द गरेपछि हराउँछ।

```bash
wall-vault proxy --key-google=AIzaSy... --key-openrouter=sk-or-...
```

## प्रोक्सी प्रयोग विधि

### OpenClaw मा प्रयोग (मुख्य उद्देश्य)

OpenClaw लाई wall-vault मार्फत AI सेवामा जोड्न सेटअप गर्ने तरिका:

`~/.openclaw/openclaw.json` फाइल खोल्नुहोस् र तलको सामग्री थप्नुहोस्:

```json5
// ~/.openclaw/openclaw.json
{
  models: {
    providers: {
      "wall-vault": {
        baseUrl: "http://localhost:56244/v1",
        apiKey: "your-agent-token",   // vault एजेन्ट टोकन
        api: "openai-completions",
        models: [
          { id: "wall-vault/gemini-2.5-flash" },
          { id: "wall-vault/gemini-2.5-pro" },
          { id: "wall-vault/hunter-alpha" },    // निःशुल्क १M सन्दर्भ
          { id: "wall-vault/claude-opus-4-6" }
        ]
      }
    }
  }
}
```

> 💡 **सजिलो तरिका**: ड्यासबोर्डको एजेन्ट कार्डमा **🦞 OpenClaw सेटिङ प्रतिलिपि** बटन थिच्दा टोकन र ठेगाना भरिएको स्निपेट क्लिपबोर्डमा प्रतिलिपि हुन्छ। टाँस्ने मात्र काम बाँकी रहन्छ।

**मोडेल नामको अगाडिको `wall-vault/` ले कहाँ जोड्छ?**

मोडेल नाम हेरेर wall-vault ले कुन AI सेवामा अनुरोध पठाउने स्वचालित रूपमा निर्णय गर्छ:

| मोडेल ढाँचा | जोडिने सेवा |
|----------|--------------|
| `wall-vault/gemini-*` | Google Gemini सिधा जडान |
| `wall-vault/gpt-*`, `wall-vault/o3`, `wall-vault/o4*` | OpenAI सिधा जडान |
| `wall-vault/claude-*` | OpenRouter मार्फत Anthropic जडान |
| `wall-vault/hunter-alpha`, `wall-vault/healer-alpha` | OpenRouter (निःशुल्क १ लाख टोकन सन्दर्भ) |
| `wall-vault/kimi-*`, `wall-vault/glm-*`, `wall-vault/deepseek-*` | OpenRouter जडान |
| `google/मोडेलनाम`, `openai/मोडेलनाम`, `anthropic/मोडेलनाम` आदि | सम्बन्धित सेवा सिधा जडान |
| `custom/google/मोडेलनाम`, `custom/openai/मोडेलनाम` आदि | `custom/` भाग हटाएर पुनः राउट गर्छ |
| `मोडेलनाम:cloud` | `:cloud` भाग हटाएर OpenRouter जडान |

> 💡 **सन्दर्भ (context) के हो?** AI ले एकपटकमा सम्झन सक्ने कुराकानीको मात्रा हो। १M (दश लाख टोकन) भए धेरै लामो कुराकानी वा लामो कागजात पनि एकसाथ प्रशोधन गर्न सकिन्छ।

### Gemini API ढाँचामा सिधा जडान (पुराना उपकरणसँग मिलाउन)

Google Gemini API सिधै प्रयोग गर्ने उपकरण भएमा, ठेगाना मात्र wall-vault मा बदल्नुहोस्:

```bash
export ANTHROPIC_BASE_URL=http://localhost:56244/google
```

वा URL सिधै निर्दिष्ट गर्ने उपकरणमा:

```
http://localhost:56244/google/v1beta/models/gemini-2.5-flash:generateContent
```

### OpenAI SDK (Python) मा प्रयोग

Python मा AI प्रयोग गर्ने कोडमा पनि wall-vault जोड्न सकिन्छ। केवल `base_url` बदल्नुहोस्:

```python
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:56244/v1",
    api_key="not-needed"  # API कुञ्जी wall-vault ले आफैँ व्यवस्थापन गर्छ
)

response = client.chat.completions.create(
    model="google/gemini-2.5-flash",   # provider/model ढाँचामा राख्नुहोस्
    messages=[{"role": "user", "content": "नमस्ते"}]
)
```

### चलिरहेको अवस्थामा मोडेल बदल्नुहोस्

wall-vault चलिरहेको अवस्थामा AI मोडेल बदल्न:

```bash
# प्रोक्सीमा सिधै अनुरोध गरेर मोडेल बदल्नुहोस्
curl -X PUT http://localhost:56244/api/config/model \
  -H "Content-Type: application/json" \
  -d '{"service": "openrouter", "model": "anthropic/claude-3.5-sonnet"}'

# वितरित मोड (बहु-बट) मा तिजोरी सर्भरबाट बदल्नुहोस् → SSE मार्फत तुरुन्तै लागू हुन्छ
curl -X PUT http://localhost:56243/admin/clients/내-봇-id \
  -H "Authorization: Bearer 관리자토큰" \
  -H "Content-Type: application/json" \
  -d '{"default_service": "google", "default_model": "gemini-2.5-pro"}'
```

### API कुञ्जी कार्ड

दर्ता भएका API कुञ्जीहरू एकै नजरमा व्यवस्थापन गर्न मिल्ने कार्ड।

- सेवा अनुसार छुट्याएर कुञ्जीको सूची देखाउँछ।
- `today_usage`: आज सफलतापूर्वक प्रशोधन भएका टोकन (AI ले पढेका र लेखेका अक्षरको गणना) संख्या
- `today_attempts`: आजको कुल कल गणना (सफल + असफल सहित)
- `[+ थप्नुहोस्]` बटनले नयाँ कुञ्जी दर्ता गर्छ, `✕` ले कुञ्जी मेटाउँछ।

> 💡 **टोकन (token) के हो?** AI ले पाठ प्रशोधन गर्दा प्रयोग गर्ने एकाइ हो। लगभग एउटा अंग्रेजी शब्द, वा नेपाली/हिन्दी को १–२ अक्षर बराबर हुन्छ। API शुल्क सामान्यतः यही टोकन संख्यामा आधारित हुन्छ।

#### 🔄 दुई-दिशा मोडेल समन्वय (v0.1.16)

तिजोरी ड्यासबोर्डमा एजेन्टको मोडेल बदल्दा, सो एजेन्टको स्थानीय सेटिङ स्वचालित रूपमा अद्यावधिक हुन्छ।

**Cline को हकमा:**
- तिजोरीमा मोडेल बदल्दा → SSE घटना → प्रोक्सीले `globalState.json` को मोडेल फिल्ड अद्यावधिक गर्छ

- `openAiBaseUrl` र API कुञ्जी परिवर्तन हुँदैन
- **VS Code रिलोड (`Ctrl+Alt+R` वा `Ctrl+Shift+P` → `Developer: Reload Window`) आवश्यक छ**
  - किनभने Cline ले चलिरहेको अवस्थामा सेटिङ फाइल पुनः पढ्दैन

**Claude Code को हकमा:**
- तिजोरीमा मोडेल बदल्दा → SSE घटना → प्रोक्सीले `settings.json` को `model` फिल्ड अद्यावधिक गर्छ
- WSL र Windows दुवै पथ स्वचालित रूपमा खोज्छ (`~/.claude/`, `/mnt/c/Users/*/.claude/`)

**विपरीत दिशा (एजेन्ट → तिजोरी):**
- एजेन्ट (Cline, Claude Code आदि) ले प्रोक्सी मार्फत अनुरोध पठाउँदा, प्रोक्सीले heartbeat मा सो क्लाइन्टको सेवा·मोडेल जानकारी समावेश गर्छ
- तिजोरी ड्यासबोर्डको एजेन्ट कार्डमा हाल प्रयोग भइरहेको सेवा/मोडेल रियल-टाइममा देखिन्छ

> 💡 **मुख्य कुरा**: प्रोक्सीले अनुरोधमा भएको Authorization टोकनबाट एजेन्ट पहिचान गर्छ र तिजोरीमा सेट गरेको सेवा/मोडेलमा स्वचालित राउट गर्छ। Cline वा Claude Code ले अर्को मोडेल नाम पठाए पनि, प्रोक्सीले तिजोरीको सेटिङद्वारा ओभरराइड गर्छ।

नयाँ कम्प्युटरमा wall-vault प्रोक्सी पहिलो पटक स्थापना गर्दा र तिजोरीमा जोड्न प्रयोग गरिन्छ। बटन थिच्दा स्थापना स्क्रिप्ट पूरा प्रतिलिपि हुन्छ। नयाँ कम्प्युटरको टर्मिनलमा टाँसेर चलाउँदा एकसाथ तलका काम हुन्छन्:

१. wall-vault बाइनरी स्थापना (पहिले नै स्थापित भएको भए छोड्छ)
२. systemd प्रयोगकर्ता सेवा स्वचालित दर्ता
३. सेवा सुरु र तिजोरी स्वचालित जडान

> 💡 स्क्रिप्ट भित्र यस एजेन्टको टोकन र तिजोरी सर्भर ठेगाना पहिले नै भरिएको हुन्छ, त्यसैले टाँसेर तुरुन्तै चलाउन सकिन्छ, अलग्गै सम्पादन गर्नुपर्दैन।

### सेवा कार्ड

प्रयोग गर्ने AI सेवा सक्षम/अक्षम गर्ने वा सेटअप गर्ने कार्ड।

- सेवाअनुसार सक्षम/अक्षम टगल स्विच
- स्थानीय AI सर्भर (आफ्नै कम्प्युटरमा चलाउने Ollama, LM Studio, vLLM आदि) को ठेगाना राख्दा उपलब्ध मोडेल स्वचालित रूपमा खोज्छ।
- **स्थानीय सेवा जडान अवस्था प्रदर्शन**: सेवाको नाम छेउको ● बिन्दु **हरियो** भए जोडिएको, **खैरो** भए जोडिएको छैन
- **चेकबक्स स्वचालित समन्वय**: पृष्ठ खोल्दा स्थानीय सेवा (Ollama आदि) चलिरहेको भए स्वचालित रूपमा चेक अवस्थामा रहन्छ।

> 💡 **स्थानीय सेवा अर्को कम्प्युटरमा चलिरहेको भएमा**: सेवा URL प्रविष्टि क्षेत्रमा त्यो कम्प्युटरको IP राख्नुहोस्। उदाहरण: `http://192.168.0.6:11434` (Ollama), `http://192.168.0.6:1234` (LM Studio)

### प्रशासक टोकन प्रविष्टि

ड्यासबोर्डमा कुञ्जी थप्ने/मेटाउने जस्ता महत्त्वपूर्ण कार्य गर्न खोज्दा प्रशासक टोकन प्रविष्टि पप-अप देखिन्छ। setup जादुगरमा सेट गरेको टोकन राख्नुहोस्। एकपटक राखेपछि ब्राउजर बन्द नगरेसम्म कायम रहन्छ।

> ⚠️ **१५ मिनेटभित्र १० पटकभन्दा बढी प्रमाणीकरण असफल भएमा सो IP अस्थायी रूपमा अवरुद्ध हुन्छ।** टोकन बिर्सनुभयो भने `wall-vault.yaml` फाइलमा `admin_token` प्रविष्टि हेर्नुहोस्।

## वितरित मोड (बहु-बट)

धेरै कम्प्युटरमा OpenClaw एकसाथ सञ्चालन गर्दा, **एउटा कुञ्जी तिजोरी साझा** गर्ने व्यवस्था हो। कुञ्जी व्यवस्थापन एक ठाउँबाट मात्र गर्न मिल्ने भएकाले सुविधाजनक छ।

### व्यवस्था उदाहरण

```
[कुञ्जी तिजोरी सर्भर]
  wall-vault vault    (कुञ्जी तिजोरी :56243, ड्यासबोर्ड)

[WSL अल्फा]            [Raspberry Pi गामा]    [Mac Mini स्थानीय]
  wall-vault proxy      wall-vault proxy        wall-vault proxy
  openclaw TUI          openclaw TUI            openclaw TUI
  ↕ SSE समन्वय          ↕ SSE समन्वय            ↕ SSE समन्वय
```

सबै बटहरू बीचको तिजोरी सर्भरतर्फ हेरिरहेका हुन्छन्, त्यसैले तिजोरीमा मोडेल बदल्दा वा कुञ्जी थप्दा सबै बटमा तुरुन्तै लागू हुन्छ।

### चरण १: कुञ्जी तिजोरी सर्भर सुरु गर्नुहोस्

तिजोरी सर्भर बन्ने कम्प्युटरमा चलाउनुहोस्:

```bash
wall-vault vault
```

### चरण २: प्रत्येक बट (क्लाइन्ट) दर्ता गर्नुहोस्

तिजोरी सर्भरमा जोड्ने प्रत्येक बटको जानकारी अगाडि नै दर्ता गर्नुहोस्:

```bash
curl -X POST http://localhost:56243/admin/clients \
  -H "Authorization: Bearer 관리자토큰" \
  -H "Content-Type: application/json" \
  -d '{
    "id": "botA",
    "name": "봇A",
    "token": "bota-secret",
    "default_service": "google",
    "default_model": "gemini-2.5-flash"
  }'
```

### चरण ३: प्रत्येक बट कम्प्युटरमा प्रोक्सी सुरु गर्नुहोस्

बट स्थापित प्रत्येक कम्प्युटरमा तिजोरी सर्भर ठेगाना र टोकन निर्दिष्ट गरेर प्रोक्सी चलाउनुहोस्:

```bash
WV_VAULT_URL=http://192.168.x.x:56243 \
WV_VAULT_TOKEN=bota-secret \
WV_VAULT_CLIENT_ID=botA \
wall-vault proxy
```

> 💡 **`192.168.x.x`** भागलाई तिजोरी सर्भर कम्प्युटरको वास्तविक आन्तरिक IP ठेगानाले बदल्नुहोस्। राउटरको सेटिङ वा `ip addr` आदेशले पत्ता लगाउन सकिन्छ।

## स्वचालित सुरुआत सेटअप

कम्प्युटर पुनःसुरु गर्दा पटक-पटक wall-vault म्यानुअल रूपमा खोल्न झन्झटिलो लागेमा, सिस्टम सेवाको रूपमा दर्ता गर्नुहोस्। एकपटक दर्ता गरेपछि बुट हुँदा स्वचालित रूपमा सुरु हुन्छ।

### Linux — systemd (अधिकांश Linux)

systemd भनेको Linux मा प्रोग्रामहरू स्वचालित रूपमा सुरु गर्ने र व्यवस्थापन गर्ने प्रणाली हो:

```bash
wall-vault doctor deploy
systemctl --user daemon-reload
systemctl --user enable --now wall-vault
```

लग हेर्न:

```bash
journalctl --user -u wall-vault -f
```

### macOS — launchd

macOS मा प्रोग्राम स्वचालित सञ्चालन गर्ने प्रणाली:

```bash
wall-vault doctor deploy launchd
launchctl load ~/Library/LaunchAgents/com.wall-vault.plist
```

### Windows — NSSM

१. [nssm.cc](https://nssm.cc/download) बाट NSSM डाउनलोड गरेर PATH मा थप्नुहोस्।
२. प्रशासक अधिकार सहित PowerShell मा:

```powershell
wall-vault doctor deploy windows
```

## Doctor निदान उपकरण

`doctor` आदेश wall-vault सही तरिकाले सेट भएको छ कि छैन भनेर **आफैँ निदान गरेर सुधार गर्ने उपकरण** हो।

```bash
wall-vault doctor check   # हालको अवस्था निदान (पढ्ने मात्र, केही परिवर्तन गर्दैन)
wall-vault doctor fix     # समस्या स्वचालित रूपमा ठीक गर्छ
wall-vault doctor all     # निदान + स्वचालित सुधार एकसाथ
```

> 💡 केही गडबड जस्तो लागेमा पहिले `wall-vault doctor all` चलाउनुहोस्। धेरै समस्या स्वचालित रूपमा समाधान हुन्छन्।

## वातावरण चर सन्दर्भ

वातावरण चर भनेको प्रोग्राममा सेटिङ मान पठाउने तरिका हो। `export चरनाम=मान` ढाँचामा टर्मिनलमा राख्नुहोस् वा स्वचालित सुरुआत सेवा फाइलमा राख्दा सधैँ लागू हुन्छ।

| चर | विवरण | उदाहरण मान |
|------|------|---------|
| `WV_LANG` | ड्यासबोर्ड भाषा | `ko`, `en`, `ja` |
| `WV_THEME` | ड्यासबोर्ड थिम | `light`, `dark`, `gold` |
| `WV_KEY_GOOGLE` | Google API कुञ्जी (अल्पविरामले धेरै) | `AIza...,AIza...` |
| `WV_KEY_OPENROUTER` | OpenRouter API कुञ्जी | `sk-or-v1-...` |
| `WV_VAULT_URL` | वितरित मोडमा तिजोरी सर्भर ठेगाना | `http://192.168.x.x:56243` |
| `WV_VAULT_TOKEN` | क्लाइन्ट (बट) प्रमाणीकरण टोकन | `my-secret-token` |
| `WV_ADMIN_TOKEN` | प्रशासक टोकन | `admin-token-here` |
| `WV_MASTER_PASS` | API कुञ्जी इन्क्रिप्सन पासवर्ड | `my-password` |
| `WV_AVATAR` | अवतार छवि फाइल पथ (`~/.openclaw/` आधारित सापेक्ष पथ) | `workspace/avatars/avatar.png` |
| `OLLAMA_URL` | Ollama स्थानीय सर्भर ठेगाना | `http://192.168.x.x:11434` |

## समस्या समाधान

### प्रोक्सी सुरु नभएमा

पोर्ट पहिले नै अर्को प्रोग्रामले प्रयोग गरिरहेको हुन सक्छ।

```bash
ss -tlnp | grep 56244   # ५६२४४ पोर्ट कसले प्रयोग गरिरहेको छ भनेर जाँच्नुहोस्
wall-vault proxy --port 8080   # अर्को पोर्ट नम्बरमा सुरु गर्नुहोस्
```

### API कुञ्जी त्रुटि भएमा (429, 402, 401, 403, 582)

| त्रुटि कोड | अर्थ | समाधान |
|----------|------|----------|
| **429** | अनुरोध धेरै भयो (सीमा नाघ्यो) | केही पर्खनुहोस् वा अर्को कुञ्जी थप्नुहोस् |
| **402** | भुक्तानी आवश्यक वा क्रेडिट कम छ | सम्बन्धित सेवामा क्रेडिट थप्नुहोस् |
| **401 / 403** | कुञ्जी गलत छ वा अनुमति छैन | कुञ्जी मान पुनः जाँचेर फेरि दर्ता गर्नुहोस् |
| **582** | गेटवे अत्यधिक भार (५ मिनेट कुलडाउन) | ५ मिनेट पछि स्वचालित रूपमा हट्छ |

```bash
# दर्ता भएका कुञ्जी सूची र अवस्था जाँच्नुहोस्
curl -H "Authorization: Bearer 관리자토큰" http://localhost:56243/admin/keys

# कुञ्जी प्रयोग काउन्टर रिसेट गर्नुहोस्
curl -X POST -H "Authorization: Bearer 관리자토큰" http://localhost:56243/admin/keys/reset
```

### एजेन्ट "नजोडिएको" देखिएमा

"नजोडिएको" भनेको प्रोक्सी प्रक्रियाले तिजोरीमा संकेत (heartbeat) नपठाएको अवस्था हो। **यसको अर्थ सेटिङ सुरक्षित नभएको होइन।** प्रोक्सीले तिजोरी सर्भर ठेगाना र टोकन थाहा पाएर चलेपछि मात्र जोडिएको अवस्थामा बदलिन्छ।

```bash
# तिजोरी सर्भर ठेगाना, टोकन, र क्लाइन्ट ID निर्दिष्ट गरेर प्रोक्सी सुरु गर्नुहोस्
WV_VAULT_URL=http://तिजोरीसर्भरठेगाना:56243 \
WV_VAULT_TOKEN=क्लाइन्टटोकन \
WV_VAULT_CLIENT_ID=क्लाइन्टID \
wall-vault proxy
```

जडान सफल भएपछि लगभग २० सेकेन्डभित्र ड्यासबोर्डमा 🟢 चलिरहेको देखिन्छ।

### Ollama जडान नभएमा

Ollama भनेको आफ्नै कम्प्युटरमा सिधै AI चलाउने प्रोग्राम हो। पहिले Ollama खुलेको छ कि छैन जाँच्नुहोस्।

```bash
curl http://localhost:11434/api/tags   # मोडेल सूची देखिएमा सामान्य छ
export OLLAMA_URL=http://192.168.x.x:11434   # अर्को कम्प्युटरमा चलिरहेको भएमा
```

> ⚠️ Ollama ले जवाफ नदिएमा `ollama serve` आदेशले पहिले Ollama सुरु गर्नुहोस्।

> ⚠️ **ठूला मोडेलको जवाफ ढिलो हुन्छ**: `qwen3.5:35b`, `deepseek-r1` जस्ता ठूला मोडेलहरूले जवाफ दिन धेरै मिनेट लाग्न सक्छ। जवाफ नआएको जस्तो देखिए पनि सामान्य प्रशोधन भइरहेको हुन सक्छ, पर्खनुहोस्।
