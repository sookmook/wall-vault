## wall-vault क्या है?

**wall-vault = OpenClaw के लिए AI प्रॉक्सी (दूत) + API की तिजोरी**

AI सेवाओं का उपयोग करने के लिए **API की** (डिजिटल प्रवेश पास) की जरूरत होती है। यह पास यह साबित करता है कि "इस व्यक्ति को इस सेवा का उपयोग करने का अधिकार है।" लेकिन इस पास की एक दिन में इस्तेमाल की एक सीमा होती है, और अगर इसे ठीक से न संभाला जाए तो इसके उजागर होने का खतरा भी रहता है।

wall-vault इन डिजिटल प्रवेश पासों को एक सुरक्षित तिजोरी में रखता है और OpenClaw तथा AI सेवाओं के बीच **दूत (प्रॉक्सी)** की भूमिका निभाता है। सरल शब्दों में — OpenClaw को केवल wall-vault से जुड़ना होता है, और बाकी सारी जटिल प्रक्रिया wall-vault खुद संभाल लेता है।

wall-vault जो समस्याएं हल करता है:

- **API की ऑटो-रोटेशन**: जब किसी की की सीमा समाप्त हो जाए या वह अस्थायी रूप से ब्लॉक हो जाए (कूलडाउन), तो यह चुपचाप अगली की पर स्विच कर देता है। OpenClaw बिना रुके काम करता रहता है।
- **सर्विस ऑटो-फॉलबैक**: अगर Google जवाब नहीं दे, तो OpenRouter पर; वह भी काम न करे, तो आपके कंप्यूटर पर इंस्टॉल Ollama (लोकल AI) पर स्वचालित रूप से स्विच हो जाता है। सेशन नहीं टूटता।
- **रियल-टाइम सिंक्रोनाइज़ेशन (SSE)**: वॉल्ट डैशबोर्ड पर मॉडल बदलने के 1–3 सेकंड के भीतर OpenClaw की स्क्रीन पर दिखाई देता है। SSE (Server-Sent Events) एक तकनीक है जिससे सर्वर बदलाव को तुरंत क्लाइंट तक पहुंचाता है।
- **रियल-टाइम नोटिफिकेशन**: की समाप्त होने या सर्विस में खराबी जैसी घटनाएं OpenClaw के TUI (टर्मिनल स्क्रीन) के नीचे तुरंत दिखाई देती हैं।

> 💡 **Claude Code, Cursor, VS Code** को भी जोड़कर उपयोग किया जा सकता है, लेकिन wall-vault का मूल उद्देश्य OpenClaw के साथ काम करना है।

```
OpenClaw (TUI टर्मिनल स्क्रीन)
        │
        ▼
  wall-vault प्रॉक्सी (:56244)   ← की प्रबंधन, रूटिंग, फॉलबैक, इवेंट
        │
        ├─ Google Gemini API
        ├─ OpenRouter API (340+ मॉडल)
        └─ Ollama (आपका कंप्यूटर, अंतिम विकल्प)
```

## इंस्टॉलेशन

### Linux / macOS

टर्मिनल खोलें और नीचे दिए गए कमांड को वैसे ही पेस्ट करें।

```bash
# Linux (सामान्य PC, सर्वर — amd64)
curl -L https://github.com/sookmook/wall-vault/releases/latest/download/wall-vault-linux-amd64 \
  -o ~/.local/bin/wall-vault && chmod +x ~/.local/bin/wall-vault

# macOS Apple Silicon (M1/M2/M3 Mac)
curl -L https://github.com/sookmook/wall-vault/releases/latest/download/wall-vault-darwin-arm64 \
  -o /usr/local/bin/wall-vault && chmod +x /usr/local/bin/wall-vault
```

- `curl -L ...` — इंटरनेट से फ़ाइल डाउनलोड करता है।
- `chmod +x` — डाउनलोड की गई फ़ाइल को "चलाने योग्य" बनाता है। यह चरण छोड़ने पर "अनुमति नहीं" की त्रुटि आएगी।

### Windows

PowerShell (एडमिनिस्ट्रेटर अधिकार के साथ) खोलें और नीचे दिए गए कमांड चलाएं।

```powershell
# डाउनलोड
Invoke-WebRequest -Uri `
  "https://github.com/sookmook/wall-vault/releases/latest/download/wall-vault-windows-amd64.exe" `
  -OutFile "$env:LOCALAPPDATA\Programs\wall-vault\wall-vault.exe"

# PATH में जोड़ें (PowerShell रीस्टार्ट के बाद लागू होगा)
$env:PATH += ";$env:LOCALAPPDATA\Programs\wall-vault"
```

> 💡 **PATH क्या है?** यह उन फ़ोल्डरों की सूची है जहाँ कंप्यूटर कमांड खोजता है। PATH में जोड़ने के बाद ही आप किसी भी फ़ोल्डर से `wall-vault` टाइप करके इसे चला सकते हैं।

### सोर्स से बिल्ड करना (डेवलपर्स के लिए)

यह केवल उनके लिए है जिनके पास Go भाषा का डेवलपमेंट वातावरण इंस्टॉल है।

```bash
git clone https://github.com/sookmook/wall-vault
cd wall-vault
make build       # bin/wall-vault (संस्करण: v0.1.6.YYYYMMDD.HHmmss)
make install     # ~/.local/bin/wall-vault
```

> 💡 **बिल्ड टाइमस्टैम्प संस्करण**: `make build` से बनाने पर संस्करण `v0.1.6.20260314.231308` जैसा तारीख-समय सहित स्वचालित रूप से बनता है। `go build ./...` से सीधे बनाने पर संस्करण केवल `"dev"` दिखाई देगा।

## पहली बार शुरू करना

### setup विज़ार्ड चलाना

इंस्टॉलेशन के बाद पहली बार में नीचे दिए कमांड से **सेटअप विज़ार्ड** अवश्य चलाएं। विज़ार्ड आवश्यक जानकारी एक-एक करके पूछते हुए मार्गदर्शन करेगा।

```bash
wall-vault setup
```

विज़ार्ड निम्नलिखित चरणों में आगे बढ़ता है:

```
1. भाषा चुनें (हिंदी सहित 10 भाषाएं)
2. थीम चुनें (light / dark / gold / cherry / ocean)
3. ऑपरेशन मोड — अकेले (standalone) या कई कंप्यूटरों पर एक साथ (distributed)
4. बॉट का नाम — डैशबोर्ड पर दिखाई देने वाला नाम
5. पोर्ट सेटिंग — डिफ़ॉल्ट: प्रॉक्सी 56244, वॉल्ट 56243 (बदलना न हो तो बस Enter दबाएं)
6. AI सेवा चुनें — Google / OpenRouter / Ollama में से जो चाहिए
7. टूल सुरक्षा फ़िल्टर सेटिंग
8. एडमिन टोकन सेटिंग — डैशबोर्ड को लॉक करने का पासवर्ड। स्वचालित जनरेशन भी संभव है
9. API की एन्क्रिप्शन पासवर्ड — की को अधिक सुरक्षित रखना चाहते हैं तो (वैकल्पिक)
10. कॉन्फ़िग फ़ाइल सेव करने का पाथ
```

> ⚠️ **एडमिन टोकन जरूर याद रखें।** बाद में डैशबोर्ड पर की जोड़ने या सेटिंग बदलने के लिए यह जरूरी है। भूल जाने पर कॉन्फ़िग फ़ाइल को सीधे संपादित करना होगा।

विज़ार्ड पूरा होने पर `wall-vault.yaml` कॉन्फ़िग फ़ाइल स्वचालित रूप से बन जाती है।

### चलाना

```bash
wall-vault start
```

नीचे दिए दो सर्वर एक साथ शुरू होते हैं:

- **प्रॉक्सी** (`http://localhost:56244`) — OpenClaw और AI सेवाओं के बीच का दूत
- **की वॉल्ट** (`http://localhost:56243`) — API की प्रबंधन और वेब डैशबोर्ड

ब्राउज़र में `http://localhost:56243` खोलने पर डैशबोर्ड तुरंत देख सकते हैं।

## API की रजिस्ट्रेशन

API की (डिजिटल प्रवेश पास) रजिस्टर करने के चार तरीके हैं। **पहली बार शुरू करने वालों के लिए तरीका 1 (एनवायरनमेंट वेरिएबल) की सिफारिश की जाती है।**

### तरीका 1: एनवायरनमेंट वेरिएबल (अनुशंसित — सबसे सरल)

एनवायरनमेंट वेरिएबल वे **पहले से सेट मान** हैं जो प्रोग्राम शुरू होने पर पढ़ता है। टर्मिनल में नीचे दिए अनुसार टाइप करें:

```bash
# Google Gemini की रजिस्ट्रेशन
export WV_KEY_GOOGLE=AIzaSy...

# OpenRouter की रजिस्ट्रेशन
export WV_KEY_OPENROUTER=sk-or-v1-...

# रजिस्ट्रेशन के बाद चलाएं
wall-vault start
```

अगर आपके पास कई कीज़ हैं तो उन्हें कॉमा (,) से जोड़ें। wall-vault उन्हें बारी-बारी से स्वचालित रूप से उपयोग करेगा (राउंड-रॉबिन):

```bash
export WV_KEY_GOOGLE=AIzaSy...,AIzaSy...,AIzaSy...
```

> 💡 **टिप**: `export` कमांड केवल वर्तमान टर्मिनल सेशन में लागू होता है। कंप्यूटर रीस्टार्ट करने के बाद भी बनाए रखने के लिए `~/.bashrc` या `~/.zshrc` फ़ाइल में वह लाइन जोड़ें।

### तरीका 2: डैशबोर्ड UI (माउस से क्लिक)

1. ब्राउज़र में `http://localhost:56243` खोलें
2. ऊपर **🔑 API की** कार्ड में `[+ जोड़ें]` बटन क्लिक करें
3. सर्विस का प्रकार, की की वैल्यू, लेबल (मेमो नाम), और दैनिक सीमा भरकर सेव करें

### तरीका 3: REST API (ऑटोमेशन/स्क्रिप्ट के लिए)

REST API वह तरीका है जिससे प्रोग्राम HTTP के ज़रिए एक-दूसरे से डेटा आदान-प्रदान करते हैं। स्क्रिप्ट से स्वचालित रूप से रजिस्टर करने के लिए उपयोगी है।

```bash
curl -X POST http://localhost:56243/admin/keys \
  -H "Authorization: Bearer 관리자토큰" \
  -H "Content-Type: application/json" \
  -d '{
    "service": "google",
    "key": "AIzaSy...",
    "label": "메인 키",
    "daily_limit": 1000
  }'
```

### तरीका 4: proxy फ़्लैग (त्वरित परीक्षण के लिए)

बिना आधिकारिक रजिस्ट्रेशन के अस्थायी रूप से की डालकर परीक्षण करने के लिए। प्रोग्राम बंद करने पर यह हट जाती है।

```bash
wall-vault proxy --key-google=AIzaSy... --key-openrouter=sk-or-...
```

## प्रॉक्सी का उपयोग

### OpenClaw में उपयोग (मुख्य उद्देश्य)

OpenClaw को wall-vault के ज़रिए AI सेवाओं से जोड़ने के लिए:

`~/.openclaw/openclaw.json` फ़ाइल खोलें और नीचे दी गई सामग्री जोड़ें:

```json5
// ~/.openclaw/openclaw.json
{
  models: {
    providers: {
      "wall-vault": {
        baseUrl: "http://localhost:56244/v1",
        apiKey: "your-agent-token",   // vault एजेंट टोकन
        api: "openai-completions",
        models: [
          { id: "wall-vault/gemini-2.5-flash" },
          { id: "wall-vault/gemini-2.5-pro" },
          { id: "wall-vault/hunter-alpha" },    // मुफ्त 1M context
          { id: "wall-vault/claude-opus-4-6" }
        ]
      }
    }
  }
}
```

> 💡 **आसान तरीका**: डैशबोर्ड के एजेंट कार्ड में **🦞 OpenClaw सेटिंग कॉपी** बटन दबाने पर टोकन और पता पहले से भरा हुआ स्निपेट क्लिपबोर्ड में कॉपी हो जाता है। बस पेस्ट करें और हो गया।

**मॉडल नाम के आगे `wall-vault/` — यह किस सेवा से जुड़ता है?**

मॉडल नाम देखकर wall-vault स्वचालित रूप से तय करता है कि रिक्वेस्ट किस AI सेवा को भेजनी है:

| मॉडल फॉर्मेट | जुड़ने वाली सेवा |
|--------------|----------------|
| `wall-vault/gemini-*` | Google Gemini से सीधे कनेक्शन |
| `wall-vault/gpt-*`, `wall-vault/o3`, `wall-vault/o4*` | OpenAI से सीधे कनेक्शन |
| `wall-vault/claude-*` | OpenRouter के ज़रिए Anthropic से कनेक्शन |
| `wall-vault/hunter-alpha`, `wall-vault/healer-alpha` | OpenRouter (मुफ्त 10 लाख टोकन कॉन्टेक्स्ट) |
| `wall-vault/kimi-*`, `wall-vault/glm-*`, `wall-vault/deepseek-*` | OpenRouter से कनेक्शन |
| `google/मॉडलनाम`, `openai/मॉडलनाम`, `anthropic/मॉडलनाम` आदि | संबंधित सेवा से सीधा कनेक्शन |
| `custom/google/मॉडलनाम`, `custom/openai/मॉडलनाम` आदि | `custom/` हटाकर फिर से रूट करता है |
| `मॉडलनाम:cloud` | `:cloud` हटाकर OpenRouter से कनेक्शन |

> 💡 **कॉन्टेक्स्ट (context) क्या है?** AI एक बार में जितनी बातचीत याद रख सकता है उसकी मात्रा। 1M (दस लाख टोकन) का मतलब बहुत लंबी बातचीत या लंबे दस्तावेज़ को एक बार में प्रोसेस कर सकता है।

### Gemini API फॉर्मेट से सीधा कनेक्शन (पुराने टूल्स के साथ)

अगर आप Google Gemini API सीधे उपयोग करने वाले टूल के साथ wall-vault जोड़ना चाहते हैं, तो केवल पता बदल दें:

```bash
export ANTHROPIC_BASE_URL=http://localhost:56244/google
```

या जो टूल URL सीधे देता है उसके लिए:

```
http://localhost:56244/google/v1beta/models/gemini-2.5-flash:generateContent
```

### OpenAI SDK (Python) में उपयोग

Python में AI का उपयोग करने वाले कोड में wall-vault जोड़ा जा सकता है। बस `base_url` बदलें:

```python
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:56244/v1",
    api_key="not-needed"  # API की को wall-vault खुद संभालता है
)

response = client.chat.completions.create(
    model="google/gemini-2.5-flash",   # provider/model फॉर्मेट में डालें
    messages=[{"role": "user", "content": "नमस्ते"}]
)
```

### चलते हुए मॉडल बदलना

wall-vault पहले से चल रहा हो तो AI मॉडल बदलने के लिए:

```bash
# प्रॉक्सी पर सीधे रिक्वेस्ट भेजकर मॉडल बदलें
curl -X PUT http://localhost:56244/api/config/model \
  -H "Content-Type: application/json" \
  -d '{"service": "openrouter", "model": "anthropic/claude-3.5-sonnet"}'

# डिस्ट्रिब्यूटेड मोड (मल्टी-बॉट) में — वॉल्ट सर्वर से बदलें → SSE के ज़रिए तुरंत लागू होगा

  -H "Content-Type: application/json" \

```

## की-वॉल्ट डैशबोर्ड

ब्राउज़र में `http://localhost:56243` खोलने पर डैशबोर्ड दिखाई देता है।

**स्क्रीन का लेआउट:**
- **ऊपर का बार (topbar)**: लोगो, भाषा और थीम सेलेक्टर, SSE कनेक्शन स्टेटस
- **कार्ड ग्रिड**: एजेंट, सर्विस और API की कार्ड टाइल्स में व्यवस्थित

### API की कार्ड

रजिस्टर की गई API कीज़ को एक नज़र में प्रबंधित करने का कार्ड।

- सर्विस के अनुसार अलग-अलग की की सूची दिखाता है।
- `today_usage`: आज सफलतापूर्वक प्रोसेस किए गए टोकन (AI के पढ़े-लिखे अक्षरों की संख्या)
- `today_attempts`: आज के कुल कॉल की संख्या (सफल + असफल सहित)
- `[+ जोड़ें]` बटन से नई की जोड़ें, और `✕` से की हटाएं।

> 💡 **टोकन (token) क्या है?** AI जब टेक्स्ट प्रोसेस करता है तो जो इकाई उपयोग करता है। लगभग एक अंग्रेजी शब्द या एक हिंदी अक्षर के बराबर। API की कीमत आमतौर पर इन्हीं टोकन की संख्या पर आधारित होती है।

### एजेंट कार्ड

wall-vault प्रॉक्सी से जुड़े बॉट्स (एजेंट) की स्थिति दिखाने वाला कार्ड।

**कनेक्शन स्टेटस 4 चरणों में दिखाया जाता है:**

| संकेत | स्थिति | अर्थ |
|-------|--------|------|
| 🟢 | चल रहा है | प्रॉक्सी सामान्य रूप से काम कर रहा है |

| ⚫ | कनेक्ट नहीं/निष्क्रिय | प्रॉक्सी कभी वॉल्ट से जुड़ा नहीं या निष्क्रिय है |

**एजेंट कार्ड के नीचे बटनों की जानकारी:**

एजेंट रजिस्टर करते समय **एजेंट का प्रकार** निर्दिष्ट करने पर उस प्रकार के अनुसार सुविधा बटन स्वचालित रूप से दिखाई देते हैं।

#### 🔄 दोतरफा मॉडल सिंक्रोनाइज़ेशन (v0.1.16)

वॉल्ट डैशबोर्ड से एजेंट का मॉडल बदलने पर, उस एजेंट की लोकल सेटिंग स्वचालित रूप से अपडेट हो जाती है।

**Cline के लिए:**
- वॉल्ट में मॉडल बदलें → SSE इवेंट → प्रॉक्सी `globalState.json` के मॉडल फ़ील्ड को अपडेट करती है
- अपडेट होने वाले फ़ील्ड: `actModeOpenAiModelId`, `planModeOpenAiModelId`, `openAiModelId`
- `openAiBaseUrl` और API की को नहीं छूता
- **VS Code रीलोड (`Ctrl+Alt+R` या `Ctrl+Shift+P` → `Developer: Reload Window`) जरूरी है**
  - क्योंकि Cline चलते समय सेटिंग फ़ाइल दोबारा नहीं पढ़ता

**Claude Code के लिए:**
- वॉल्ट में मॉडल बदलें → SSE इवेंट → प्रॉक्सी `settings.json` के `model` फ़ील्ड को अपडेट करती है
- WSL और Windows दोनों पाथ स्वचालित रूप से खोजता है (`~/.claude/`, `/mnt/c/Users/*/.claude/`)

**उल्टी दिशा (एजेंट → वॉल्ट):**
- जब एजेंट (Cline, Claude Code आदि) प्रॉक्सी को रिक्वेस्ट भेजता है, तो प्रॉक्सी हार्टबीट में उस क्लाइंट की सर्विस और मॉडल की जानकारी शामिल करती है
- वॉल्ट डैशबोर्ड के एजेंट कार्ड पर वर्तमान में उपयोग हो रही सर्विस/मॉडल रियल-टाइम में दिखाई देती है

> 💡 **मुख्य बात**: प्रॉक्सी रिक्वेस्ट के Authorization टोकन से एजेंट की पहचान करती है, और वॉल्ट में सेट सर्विस/मॉडल पर स्वचालित रूप से रूट करती है। भले ही Cline या Claude Code कोई अलग मॉडल नाम भेजे, प्रॉक्सी वॉल्ट सेटिंग के अनुसार ओवरराइड कर देती है।

### VS Code में Cline का उपयोग — विस्तृत गाइड

#### चरण 1: Cline इंस्टॉल करें

VS Code एक्सटेंशन मार्केटप्लेस से **Cline** (ID: `saoudrizwan.claude-dev`) इंस्टॉल करें।

#### चरण 2: वॉल्ट में एजेंट रजिस्टर करें

1. वॉल्ट डैशबोर्ड (`http://वॉल्टIP:56243`) खोलें
2. **एजेंट** सेक्शन में **+ जोड़ें** क्लिक करें
3. निम्नलिखित जानकारी भरें:

| फ़ील्ड | मान | विवरण |
|--------|-----|--------|
| ID | `my_cline` | यूनीक पहचानकर्ता (अंग्रेजी में, बिना स्पेस) |
| नाम | `My Cline` | डैशबोर्ड पर दिखाई देने वाला नाम |
| एजेंट प्रकार | `cline` | ← अवश्य `cline` चुनें |
| सर्विस | उपयोग करने वाली सर्विस चुनें (उदा.: `google`) | |
| मॉडल | उपयोग करने वाला मॉडल डालें (उदा.: `gemini-2.5-flash`) | |

4. **सेव** दबाने पर टोकन स्वचालित रूप से बन जाता है

#### चरण 3: Cline से कनेक्ट करें

**तरीका A — ऑटो-अप्लाई (अनुशंसित)**

1. इस मशीन पर wall-vault **प्रॉक्सी** चल रही है, यह सुनिश्चित करें (`localhost:56244`)
2. डैशबोर्ड के एजेंट कार्ड में **⚡ Cline सेटिंग अप्लाई** बटन क्लिक करें
3. "सेटिंग अप्लाई हो गई!" नोटिफिकेशन आए तो सफल
4. VS Code रीलोड करें (`Ctrl+Alt+R`)

**तरीका B — मैनुअल सेटिंग**

Cline साइडबार में सेटिंग (⚙️) खोलें और:
- **API Provider**: `OpenAI Compatible`
- **Base URL**: `http://प्रॉक्सीपता:56244/v1`
  - अगर उसी मशीन पर है तो `http://localhost:56244/v1`
  - दूसरी मशीन (जैसे Mac Mini) पर है तो `http://192.168.0.6:56244/v1`
- **API Key**: वॉल्ट से जारी टोकन (एजेंट कार्ड से कॉपी करें)
- **Model ID**: वॉल्ट में सेट किया गया मॉडल (उदा.: `gemini-2.5-flash`)

#### चरण 4: पुष्टि करें

Cline चैट विंडो में कोई भी मैसेज भेजकर देखें। सब ठीक हो तो:
- वॉल्ट डैशबोर्ड पर उस एजेंट कार्ड पर **हरी बिंदी (● चल रहा है)** दिखाई देगी
- कार्ड पर वर्तमान सर्विस/मॉडल दिखाई देगा (उदा.: `google / gemini-2.5-flash`)

#### मॉडल बदलना

Cline का मॉडल बदलना हो तो **वॉल्ट डैशबोर्ड** से बदलें:

1. एजेंट कार्ड में सर्विस/मॉडल ड्रॉपडाउन बदलें
2. **अप्लाई** क्लिक करें
3. VS Code रीलोड (`Ctrl+Alt+R`) — Cline फूटर में मॉडल नाम अपडेट हो जाएगा
4. अगली रिक्वेस्ट से नया मॉडल उपयोग होगा

> 💡 वास्तव में प्रॉक्सी Cline की रिक्वेस्ट को टोकन से पहचानकर वॉल्ट सेटिंग के मॉडल पर रूट करती है। VS Code रीलोड न करें तब भी **वास्तव में उपयोग होने वाला मॉडल तुरंत बदल जाता है** — रीलोड केवल Cline UI में दिखने वाले मॉडल नाम को अपडेट करने के लिए है।

#### कनेक्शन डिस्कनेक्ट का पता लगाना

VS Code बंद करने पर वॉल्ट डैशबोर्ड पर एजेंट कार्ड लगभग **2–3 मिनट** बाद पीला (विलंब) और **5 मिनट** बाद लाल (ऑफलाइन) हो जाता है।

- सर्विस के अनुसार चालू/बंद टॉगल स्विच
- लोकल AI सर्वर (आपके कंप्यूटर पर चलने वाले Ollama, LM Studio, vLLM आदि) का पता डालने पर उपलब्ध मॉडल स्वचालित रूप से ढूंढ लेता है।
- **लोकल सर्विस कनेक्शन स्टेटस**: सर्विस नाम के बगल में ● बिंदु **हरे रंग** का हो तो जुड़ा है, **ग्रे** हो तो जुड़ा नहीं।
- **चेकबॉक्स ऑटो-सिंक**: पेज खुलने पर लोकल सर्विस (Ollama आदि) चल रही हो तो चेकबॉक्स स्वचालित रूप से चेक हो जाता है।

> 💡 **लोकल सर्विस किसी दूसरे कंप्यूटर पर चल रही हो तो**: सर्विस URL बॉक्स में उस कंप्यूटर का IP डालें। जैसे: `http://192.168.0.6:11434` (Ollama), `http://192.168.0.6:1234` (LM Studio)

### एडमिन टोकन दर्ज करना

डैशबोर्ड पर की जोड़ने-हटाने जैसे महत्वपूर्ण काम करने पर एडमिन टोकन का पॉपअप आता है। setup विज़ार्ड में सेट किया गया टोकन डालें। एक बार डालने के बाद ब्राउज़र बंद करने तक याद रहता है।

> ⚠️ **15 मिनट में 10 बार से अधिक प्रमाणीकरण विफल होने पर वह IP अस्थायी रूप से ब्लॉक हो जाएगा।** टोकन भूल गए हों तो `wall-vault.yaml` फ़ाइल में `admin_token` देखें।

## डिस्ट्रिब्यूटेड मोड (मल्टी-बॉट)

कई कंप्यूटरों पर एक साथ OpenClaw चलाते समय **एक ही की वॉल्ट साझा** करने की व्यवस्था। की प्रबंधन एक ही जगह से होता है, इसलिए सुविधाजनक है।

### कॉन्फ़िगरेशन उदाहरण

```
[की वॉल्ट सर्वर]
  wall-vault vault    (की वॉल्ट :56243, डैशबोर्ड)

[WSL अल्फा]            [रास्पबेरी पाई गामा]    [मैक मिनी लोकल]
  wall-vault proxy      wall-vault proxy        wall-vault proxy
  openclaw TUI          openclaw TUI            openclaw TUI
  ↕ SSE सिंक            ↕ SSE सिंक              ↕ SSE सिंक
```

सभी बॉट बीच के वॉल्ट सर्वर को देखते हैं, इसलिए वॉल्ट पर मॉडल बदलने या की जोड़ने पर सभी बॉट में तुरंत लागू होता है।

### चरण 1: की वॉल्ट सर्वर शुरू करें

वॉल्ट सर्वर के रूप में उपयोग होने वाले कंप्यूटर पर चलाएं:

```bash
wall-vault vault
```

### चरण 2: प्रत्येक बॉट (क्लाइंट) रजिस्टर करें

वॉल्ट सर्वर से जुड़ने वाले प्रत्येक बॉट की जानकारी पहले से रजिस्टर करें:

```bash
curl -X POST http://localhost:56243/admin/clients \
  -H "Authorization: Bearer 관리자토큰" \
  -H "Content-Type: application/json" \
  -d '{
    "id": "botA",
    "name": "봇A",
    "token": "bota-secret",
    "default_service": "google",
    "default_model": "gemini-2.5-flash"
  }'
```

### चरण 3: प्रत्येक बॉट कंप्यूटर पर प्रॉक्सी शुरू करें

बॉट वाले प्रत्येक कंप्यूटर पर वॉल्ट सर्वर का पता और टोकन देकर प्रॉक्सी चलाएं:

```bash
WV_VAULT_URL=http://192.168.x.x:56243 \
WV_VAULT_TOKEN=bota-secret \
WV_VAULT_CLIENT_ID=botA \
wall-vault proxy
```

> 💡 **`192.168.x.x`** को वॉल्ट सर्वर कंप्यूटर के वास्तविक लोकल IP पते से बदलें। राउटर सेटिंग या `ip addr` कमांड से पता कर सकते हैं।

## ऑटो-स्टार्ट सेटअप

हर बार कंप्यूटर रीस्टार्ट करने पर manually wall-vault चलाना बोझिल लगे, तो इसे सिस्टम सर्विस के रूप में रजिस्टर कर लें। एक बार रजिस्टर करने के बाद बूट होते समय स्वचालित रूप से शुरू होगा।

### Linux — systemd (अधिकांश Linux)

systemd Linux में प्रोग्रामों को स्वचालित रूप से शुरू और प्रबंधित करने की प्रणाली है:

```bash
wall-vault doctor deploy
systemctl --user daemon-reload
systemctl --user enable --now wall-vault
```

लॉग देखने के लिए:

```bash
journalctl --user -u wall-vault -f
```

### macOS — launchd

macOS में प्रोग्रामों की स्वचालित शुरुआत संभालने वाली प्रणाली:

```bash
wall-vault doctor deploy launchd
launchctl load ~/Library/LaunchAgents/com.wall-vault.plist
```

### Windows — NSSM

1. [nssm.cc](https://nssm.cc/download) से NSSM डाउनलोड करें और PATH में जोड़ें।
2. एडमिनिस्ट्रेटर PowerShell में:

```powershell
wall-vault doctor deploy windows
```

## Doctor डायग्नोस्टिक टूल

`doctor` कमांड वह टूल है जो जांचता है कि wall-vault सही तरह से सेट है या नहीं और **खुद ही समस्याएं ठीक** करता है।

```bash
wall-vault doctor check   # वर्तमान स्थिति जांचें (केवल पढ़ता है, कुछ बदलता नहीं)
wall-vault doctor fix     # समस्याएं स्वचालित रूप से ठीक करें
wall-vault doctor all     # जांच + स्वचालित सुधार एक साथ
```

> 💡 कुछ गड़बड़ लगे तो पहले `wall-vault doctor all` चलाएं। यह बहुत सी समस्याएं स्वचालित रूप से ठीक कर देता है।

## एनवायरनमेंट वेरिएबल्स संदर्भ

एनवायरनमेंट वेरिएबल (पर्यावरण चर) प्रोग्राम को सेटिंग मान पास करने का तरीका है। `export वेरिएबलनाम=मान` के रूप में टर्मिनल में टाइप करें, या ऑटो-स्टार्ट सर्विस फ़ाइल में डाल दें ताकि हमेशा लागू रहे।

| वेरिएबल | विवरण | उदाहरण मान |
|---------|-------|-----------|
| `WV_LANG` | डैशबोर्ड भाषा | `ko`, `en`, `ja` |
| `WV_THEME` | डैशबोर्ड थीम | `light`, `dark`, `gold` |
| `WV_KEY_GOOGLE` | Google API की (कॉमा से कई) | `AIza...,AIza...` |
| `WV_KEY_OPENROUTER` | OpenRouter API की | `sk-or-v1-...` |
| `WV_VAULT_URL` | डिस्ट्रिब्यूटेड मोड में वॉल्ट सर्वर का पता | `http://192.168.x.x:56243` |
| `WV_VAULT_TOKEN` | क्लाइंट (बॉट) प्रमाणीकरण टोकन | `my-secret-token` |
| `WV_ADMIN_TOKEN` | एडमिन टोकन | `admin-token-here` |
| `WV_MASTER_PASS` | API की एन्क्रिप्शन पासवर्ड | `my-password` |
| `WV_AVATAR` | अवतार इमेज फ़ाइल का पाथ (`~/.openclaw/` से सापेक्ष) | `workspace/avatars/avatar.png` |
| `OLLAMA_URL` | Ollama लोकल सर्वर का पता | `http://192.168.x.x:11434` |

## समस्या निवारण

### प्रॉक्सी शुरू न हो

पोर्ट पहले से किसी दूसरे प्रोग्राम द्वारा उपयोग में हो सकता है।

```bash
ss -tlnp | grep 56244   # 56244 पोर्ट कौन उपयोग कर रहा है — जांचें
wall-vault proxy --port 8080   # दूसरे पोर्ट नंबर से शुरू करें
```

### API की में त्रुटि (429, 402, 401, 403, 582)

| त्रुटि कोड | अर्थ | समाधान |
|-----------|------|--------|
| **429** | बहुत अधिक रिक्वेस्ट (सीमा से अधिक उपयोग) | थोड़ी देर प्रतीक्षा करें या अन्य की जोड़ें |
| **402** | भुगतान आवश्यक या क्रेडिट खत्म | उस सेवा पर क्रेडिट रिचार्ज करें |
| **401 / 403** | की गलत है या अनुमति नहीं | की की जांच करें और फिर से रजिस्टर करें |
| **582** | गेटवे ओवरलोड (कूलडाउन 5 मिनट) | 5 मिनट बाद स्वचालित रूप से हट जाएगा |

```bash
# रजिस्टर की गई की की सूची और स्टेटस देखें
curl -H "Authorization: Bearer 관리자토큰" http://localhost:56243/admin/keys

# की उपयोग काउंटर रीसेट करें
curl -X POST -H "Authorization: Bearer 관리자토큰" http://localhost:56243/admin/keys/reset
```

### एजेंट "कनेक्ट नहीं" दिखाए

"कनेक्ट नहीं" का मतलब है कि प्रॉक्सी प्रोसेस वॉल्ट को सिग्नल (heartbeat) नहीं भेज रहा। **इसका अर्थ यह नहीं कि सेटिंग सेव नहीं हुई।** प्रॉक्सी को वॉल्ट सर्वर का पता और टोकन जानकर चलाना जरूरी है।

```bash
# वॉल्ट सर्वर का पता, टोकन और क्लाइंट ID देकर प्रॉक्सी शुरू करें
WV_VAULT_URL=http://वॉल्टसर्वरपता:56243 \
WV_VAULT_TOKEN=क्लाइंटटोकन \
WV_VAULT_CLIENT_ID=क्लाइंटID \
wall-vault proxy
```

कनेक्शन सफल होने पर लगभग 20 सेकंड के भीतर डैशबोर्ड पर 🟢 चल रहा है में बदल जाएगा।

### Ollama कनेक्ट न हो

Ollama आपके कंप्यूटर पर सीधे AI चलाने का प्रोग्राम है। पहले जांचें कि Ollama चालू है या नहीं।

```bash
curl http://localhost:11434/api/tags   # मॉडल सूची आए तो सब ठीक है
export OLLAMA_URL=http://192.168.x.x:11434   # दूसरे कंप्यूटर पर चल रहा हो तो
```

> ⚠️ Ollama जवाब न दे तो `ollama serve` कमांड से पहले Ollama शुरू करें।

> ⚠️ **बड़े मॉडल की प्रतिक्रिया धीमी होती है**: `qwen3.5:35b`, `deepseek-r1` जैसे बड़े मॉडलों में जवाब आने में कई मिनट लग सकते हैं। जवाब न आता लगे तब भी सामान्य प्रोसेसिंग हो रही हो सकती है, इसलिए प्रतीक्षा करें।
