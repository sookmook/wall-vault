## wall-vault क्या है?

**wall-vault = OpenClaw के लिए AI प्रॉक्सी + API कुंजी तिजोरी**

AI सेवाओं का उपयोग करने के लिए आपको **API कुंजी** की आवश्यकता होती है। API कुंजी एक **डिजिटल पास** की तरह है जो साबित करती है कि "इस व्यक्ति को इस सेवा का उपयोग करने का अधिकार है"। लेकिन इस पास की दैनिक उपयोग सीमा होती है और खराब प्रबंधन से यह उजागर होने का खतरा भी होता है।

wall-vault इन पासों को एक सुरक्षित तिजोरी में रखता है और OpenClaw तथा AI सेवाओं के बीच **प्रॉक्सी (मध्यस्थ)** की भूमिका निभाता है। सरल शब्दों में, OpenClaw को केवल wall-vault से जुड़ना होता है, और बाकी जटिल काम wall-vault स्वयं संभालता है।

wall-vault द्वारा हल की जाने वाली समस्याएं:

- **API कुंजी स्वचालित रोटेशन**: जब एक कुंजी अपनी उपयोग सीमा तक पहुंच जाती है या अस्थायी रूप से ब्लॉक (कूलडाउन) हो जाती है, तो यह चुपचाप अगली कुंजी पर स्विच कर देता है। OpenClaw बिना रुकावट काम करता रहता है।
- **सेवा स्वचालित प्रतिस्थापन (फॉलबैक)**: यदि Google जवाब नहीं देता, तो OpenRouter पर स्विच; वह भी नहीं चलता तो आपके कंप्यूटर पर इंस्टॉल किए गए लोकल AI (Ollama, LM Studio, vLLM) पर स्वचालित स्विच। सत्र नहीं टूटता। जब मूल सेवा ठीक हो जाती है, तो अगले अनुरोध से स्वचालित वापसी (v0.1.18+, LM Studio/vLLM: v0.1.21+)।
- **रीयल-टाइम सिंक्रनाइज़ेशन (SSE)**: जब आप तिजोरी डैशबोर्ड में मॉडल बदलते हैं, तो 1-3 सेकंड के भीतर OpenClaw स्क्रीन पर प्रतिबिंबित होता है। SSE (Server-Sent Events) एक तकनीक है जो सर्वर को रीयल-टाइम में क्लाइंट को अपडेट भेजने की अनुमति देती है।
- **रीयल-टाइम सूचनाएं**: कुंजी समाप्ति या सेवा विफलता जैसी घटनाएं OpenClaw TUI (टर्मिनल स्क्रीन) के निचले भाग में तुरंत दिखाई देती हैं।

> 💡 **Claude Code, Cursor, VS Code** भी कनेक्ट किए जा सकते हैं, लेकिन wall-vault का मूल उद्देश्य OpenClaw के साथ उपयोग करना है।

```
OpenClaw (TUI टर्मिनल स्क्रीन)
        │
        ▼
  wall-vault proxy (:56244)   ← कुंजी प्रबंधन, रूटिंग, फॉलबैक, इवेंट
        │
        ├─ Google Gemini API
        ├─ OpenRouter API (340 से अधिक मॉडल)
        ├─ Ollama / LM Studio / vLLM (आपका कंप्यूटर, अंतिम विकल्प)
        └─ OpenAI / Anthropic API
```

## इंस्टॉलेशन

### Linux / macOS

टर्मिनल खोलें और नीचे दिए गए कमांड पेस्ट करें:

```bash
# Linux (सामान्य PC, सर्वर — amd64)
curl -L https://github.com/sookmook/wall-vault/releases/latest/download/wall-vault-linux-amd64 \
  -o ~/.local/bin/wall-vault && chmod +x ~/.local/bin/wall-vault

# macOS Apple Silicon (M1/M2/M3 Mac)
curl -L https://github.com/sookmook/wall-vault/releases/latest/download/wall-vault-darwin-arm64 \
  -o /usr/local/bin/wall-vault && chmod +x /usr/local/bin/wall-vault
```

- `curl -L ...` — इंटरनेट से फ़ाइल डाउनलोड करता है।
- `chmod +x` — डाउनलोड की गई फ़ाइल को "एक्ज़ीक्यूटेबल" बनाता है। यदि यह चरण छोड़ दिया जाता है, तो "अनुमति अस्वीकृत" त्रुटि आएगी।

### Windows

PowerShell (एडमिनिस्ट्रेटर के रूप में) खोलें और नीचे दिए गए कमांड चलाएं:

```powershell
# डाउनलोड
Invoke-WebRequest -Uri `
  "https://github.com/sookmook/wall-vault/releases/latest/download/wall-vault-windows-amd64.exe" `
  -OutFile "$env:LOCALAPPDATA\Programs\wall-vault\wall-vault.exe"

# PATH में जोड़ें (PowerShell रीस्टार्ट के बाद लागू)
$env:PATH += ";$env:LOCALAPPDATA\Programs\wall-vault"
```

> 💡 **PATH क्या है?** यह उन फ़ोल्डरों की सूची है जहां कंप्यूटर कमांड खोजता है। PATH में जोड़ने के बाद, आप किसी भी फ़ोल्डर से बस `wall-vault` टाइप करके चला सकते हैं।

### सोर्स कोड से बिल्ड करना (डेवलपर्स के लिए)

केवल तभी लागू होता है जब Go भाषा विकास वातावरण इंस्टॉल हो।

```bash
git clone https://github.com/sookmook/wall-vault
cd wall-vault
make build       # bin/wall-vault (संस्करण: v0.1.23.YYYYMMDD.HHmmss)
make install     # ~/.local/bin/wall-vault
```

> 💡 **बिल्ड टाइमस्टैम्प संस्करण**: `make build` से बिल्ड करने पर संस्करण स्वचालित रूप से `v0.1.23.20260406.211004` जैसे दिनांक-समय सहित प्रारूप में जेनरेट होता है। `go build ./...` से सीधे बिल्ड करने पर संस्करण केवल `"dev"` के रूप में दिखेगा।

## पहली शुरुआत

### setup विज़ार्ड चलाएं

इंस्टॉलेशन के बाद, अवश्य नीचे दिए गए कमांड से **सेटअप विज़ार्ड** चलाएं। विज़ार्ड आपसे प्रत्येक आवश्यक आइटम के बारे में एक-एक करके पूछते हुए मार्गदर्शन करेगा।

```bash
wall-vault setup
```

विज़ार्ड द्वारा पूरे किए जाने वाले चरण:

```
1. भाषा चयन (कोरियाई सहित 10 भाषाएं)
2. थीम चयन (light / dark / gold / cherry / ocean)
3. संचालन मोड — अकेले उपयोग (standalone) या कई मशीनों पर साझा (distributed)
4. बॉट नाम — डैशबोर्ड पर दिखाया जाने वाला नाम
5. पोर्ट सेटिंग — डिफ़ॉल्ट: प्रॉक्सी 56244, तिजोरी 56243 (बदलने की ज़रूरत न हो तो बस Enter दबाएं)
6. AI सेवा चयन — Google / OpenRouter / Ollama / LM Studio / vLLM
7. टूल सुरक्षा फ़िल्टर सेटिंग
8. एडमिन टोकन सेटिंग — डैशबोर्ड प्रबंधन कार्यों को लॉक करने वाला पासवर्ड। स्वचालित जनरेशन भी उपलब्ध
9. API कुंजी एन्क्रिप्शन पासवर्ड सेटिंग — अधिक सुरक्षित भंडारण के लिए (वैकल्पिक)
10. कॉन्फ़िगरेशन फ़ाइल सेव पथ
```

> ⚠️ **एडमिन टोकन अवश्य याद रखें।** बाद में डैशबोर्ड में कुंजी जोड़ने या सेटिंग बदलने के लिए इसकी आवश्यकता होगी। यदि खो जाए, तो कॉन्फ़िगरेशन फ़ाइल को मैन्युअल रूप से संपादित करना होगा।

विज़ार्ड पूरा होने पर `wall-vault.yaml` कॉन्फ़िगरेशन फ़ाइल स्वचालित रूप से बनाई जाएगी।

### चलाएं

```bash
wall-vault start
```

नीचे दिए गए दो सर्वर एक साथ शुरू होते हैं:

- **प्रॉक्सी** (`http://localhost:56244`) — OpenClaw और AI सेवाओं के बीच मध्यस्थ
- **कुंजी तिजोरी** (`http://localhost:56243`) — API कुंजी प्रबंधन और वेब डैशबोर्ड

ब्राउज़र में `http://localhost:56243` खोलें और तुरंत डैशबोर्ड देखें।

## API कुंजी पंजीकरण

API कुंजी पंजीकरण के चार तरीके हैं। **शुरुआत करने वालों के लिए विधि 1 (पर्यावरण चर) की सिफारिश की जाती है।**

### विधि 1: पर्यावरण चर (अनुशंसित — सबसे सरल)

पर्यावरण चर **पूर्व-कॉन्फ़िगर मान** होते हैं जो प्रोग्राम शुरू होने पर पढ़ता है। टर्मिनल में नीचे दिए अनुसार दर्ज करें:

```bash
# Google Gemini कुंजी पंजीकरण
export WV_KEY_GOOGLE=AIzaSy...

# OpenRouter कुंजी पंजीकरण
export WV_KEY_OPENROUTER=sk-or-v1-...

# पंजीकरण के बाद चलाएं
wall-vault start
```

यदि आपके पास कई कुंजियां हैं, तो उन्हें कॉमा (,) से जोड़ें। wall-vault उन्हें स्वचालित रूप से बारी-बारी (राउंड रॉबिन) उपयोग करेगा:

```bash
export WV_KEY_GOOGLE=AIzaSy...,AIzaSy...,AIzaSy...
```

> 💡 **टिप**: `export` कमांड केवल वर्तमान टर्मिनल सत्र पर लागू होता है। कंप्यूटर रीस्टार्ट के बाद भी बने रहने के लिए, उपरोक्त पंक्ति को `~/.bashrc` या `~/.zshrc` फ़ाइल में जोड़ें।

### विधि 2: डैशबोर्ड UI (माउस से क्लिक)

1. ब्राउज़र में `http://localhost:56243` पर जाएं
2. शीर्ष पर **🔑 API कुंजी** कार्ड में `[+ जोड़ें]` बटन पर क्लिक करें
3. सेवा प्रकार, कुंजी मान, लेबल (संदर्भ नाम) और दैनिक सीमा दर्ज करें, फिर सेव करें

### विधि 3: REST API (स्वचालन/स्क्रिप्ट के लिए)

REST API प्रोग्राम के बीच HTTP के माध्यम से डेटा का आदान-प्रदान करने का एक तरीका है। स्क्रिप्ट से स्वचालित पंजीकरण के लिए उपयोगी।

```bash
curl -X POST http://localhost:56243/admin/keys \
  -H "Authorization: Bearer एडमिन-टोकन" \
  -H "Content-Type: application/json" \
  -d '{
    "service": "google",
    "key": "AIzaSy...",
    "label": "मुख्य कुंजी",
    "daily_limit": 1000
  }'
```

### विधि 4: प्रॉक्सी फ़्लैग (त्वरित परीक्षण के लिए)

औपचारिक पंजीकरण के बिना अस्थायी रूप से कुंजी डालकर परीक्षण के लिए उपयोग करें। प्रोग्राम बंद करने पर कुंजी गायब हो जाती है।

```bash
wall-vault proxy --key-google=AIzaSy... --key-openrouter=sk-or-...
```

## प्रॉक्सी का उपयोग

### OpenClaw के साथ उपयोग (मुख्य उद्देश्य)

wall-vault के माध्यम से OpenClaw को AI सेवाओं से कनेक्ट करने का तरीका।

`~/.openclaw/openclaw.json` फ़ाइल खोलें और निम्नलिखित सामग्री जोड़ें:

```json5
// ~/.openclaw/openclaw.json
{
  models: {
    providers: {
      "wall-vault": {
        baseUrl: "http://localhost:56244/v1",
        apiKey: "your-agent-token",   // तिजोरी एजेंट टोकन
        api: "openai-completions",
        models: [
          { id: "wall-vault/gemini-2.5-flash" },
          { id: "wall-vault/gemini-2.5-pro" },
          { id: "wall-vault/hunter-alpha" },    // मुफ्त 1M context
          { id: "wall-vault/claude-opus-4-6" }
        ]
      }
    }
  }
}
```

> 💡 **आसान तरीका**: डैशबोर्ड एजेंट कार्ड में **🦞 OpenClaw कॉन्फ़िग कॉपी** बटन दबाएं और टोकन व पता पहले से भरा हुआ स्निपेट क्लिपबोर्ड में कॉपी हो जाएगा। बस पेस्ट करें।

**मॉडल नाम की शुरुआत में `wall-vault/` कहां ले जाता है?**

wall-vault मॉडल नाम के आधार पर स्वचालित रूप से निर्धारित करता है कि किस AI सेवा को अनुरोध भेजना है:

| मॉडल प्रारूप | कनेक्ट होने वाली सेवा |
|----------|--------------|
| `wall-vault/gemini-*` | Google Gemini से सीधा कनेक्शन |
| `wall-vault/gpt-*`, `wall-vault/o3`, `wall-vault/o4*` | OpenAI से सीधा कनेक्शन |
| `wall-vault/claude-*` | OpenRouter के माध्यम से Anthropic कनेक्शन |
| `wall-vault/hunter-alpha`, `wall-vault/healer-alpha` | OpenRouter (मुफ्त 10 लाख टोकन context) |
| `wall-vault/kimi-*`, `wall-vault/glm-*`, `wall-vault/deepseek-*` | OpenRouter कनेक्शन |
| `google/मॉडल-नाम`, `openai/मॉडल-नाम`, `anthropic/मॉडल-नाम` आदि। | संबंधित सेवा से सीधा कनेक्शन |
| `custom/google/मॉडल-नाम`, `custom/openai/मॉडल-नाम` आदि। | `custom/` भाग हटाकर रीरूट |
| `मॉडल-नाम:cloud` | `:cloud` भाग हटाकर OpenRouter कनेक्शन |

> 💡 **Context क्या है?** यह वह बातचीत की मात्रा है जो AI एक बार में "याद" रख सकता है। 1M (दस लाख टोकन) का मतलब बहुत लंबी बातचीत या लंबे दस्तावेज़ भी एक बार में संसाधित किए जा सकते हैं।

### Gemini API प्रारूप में सीधा कनेक्शन (मौजूदा टूल्स के साथ संगतता)

यदि आपके पास पहले से Google Gemini API का सीधा उपयोग करने वाले टूल्स हैं, तो बस पता wall-vault में बदल दें:

```bash
export ANTHROPIC_BASE_URL=http://localhost:56244/google
```

या URL सीधे निर्दिष्ट करने वाले टूल्स के लिए:

```
http://localhost:56244/google/v1beta/models/gemini-2.5-flash:generateContent
```

### OpenAI SDK (Python) के साथ उपयोग

AI का उपयोग करने वाले Python कोड में भी wall-vault कनेक्ट किया जा सकता है। बस `base_url` बदलें:

```python
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:56244/v1",
    api_key="not-needed"  # API कुंजियां wall-vault स्वचालित रूप से प्रबंधित करता है
)

response = client.chat.completions.create(
    model="google/gemini-2.5-flash",   # provider/model प्रारूप में दर्ज करें
    messages=[{"role": "user", "content": "नमस्ते"}]
)
```

### चलते समय मॉडल बदलना

wall-vault पहले से चालू होने पर AI मॉडल बदलने के लिए:

```bash
# प्रॉक्सी को सीधे अनुरोध भेजकर मॉडल बदलें
curl -X PUT http://localhost:56244/api/config/model \
  -H "Content-Type: application/json" \
  -d '{"service": "openrouter", "model": "anthropic/claude-3.5-sonnet"}'

# वितरित मोड (मल्टी बॉट) में, तिजोरी सर्वर से बदलें → SSE के माध्यम से तुरंत प्रतिबिंबित
curl -X PUT http://localhost:56243/admin/clients/मेरा-बॉट-id \
  -H "Authorization: Bearer एडमिन-टोकन" \
  -H "Content-Type: application/json" \
  -d '{"default_service": "google", "default_model": "gemini-2.5-pro"}'
```

### उपलब्ध मॉडल सूची देखना

# पूरी सूची देखें

# नाम से खोजें (उदाहरण: "claude" वाले मॉडल)

## कुंजी तिजोरी डैशबोर्ड

ब्राउज़र में `http://localhost:56243` पर जाकर डैशबोर्ड देखें।

**स्क्रीन संरचना:**
- **शीर्ष फिक्स्ड बार (topbar)**: लोगो, भाषा और थीम चयनकर्ता, SSE कनेक्शन स्थिति संकेतक
- **कार्ड ग्रिड**: एजेंट, सेवा और API कुंजी कार्ड टाइल के रूप में व्यवस्थित

### API कुंजी कार्ड

सभी पंजीकृत API कुंजियों को एक नज़र में प्रबंधित करने के लिए कार्ड।

- सेवा के अनुसार कुंजी सूची दिखाता है।
- `today_usage`: आज सफलतापूर्वक संसाधित टोकन (AI द्वारा पढ़े और लिखे गए अक्षरों की संख्या)
- `today_attempts`: आज कुल कॉल संख्या (सफल + विफल सहित)
- नई कुंजी पंजीकृत करने के लिए `[+ जोड़ें]` बटन और हटाने के लिए `✕`।

> 💡 **टोकन क्या है?** यह AI द्वारा टेक्स्ट प्रोसेस करने की इकाई है। लगभग एक अंग्रेज़ी शब्द या 1-2 कोरियाई अक्षरों के बराबर। API शुल्क आमतौर पर इस टोकन संख्या के आधार पर गणना किया जाता है।

### एजेंट कार्ड

wall-vault प्रॉक्सी से जुड़े बॉट (एजेंट) की स्थिति दिखाने वाला कार्ड।

**कनेक्शन स्थिति 4 स्तरों में दिखाई जाती है:**

| संकेतक | स्थिति | अर्थ |
|------|------|------|
| 🟢 | चालू | प्रॉक्सी सामान्य रूप से काम कर रहा है |
| 🟡 | विलंबित | जवाब आ रहा है लेकिन धीमा |
| 🔴 | ऑफ़लाइन | प्रॉक्सी जवाब नहीं दे रहा |
| ⚫ | अकनेक्टेड/निष्क्रिय | प्रॉक्सी कभी तिजोरी से नहीं जुड़ा या निष्क्रिय है |

**एजेंट कार्ड के निचले भाग में बटन गाइड:**

एजेंट पंजीकृत करते समय, यदि आप **एजेंट प्रकार** निर्दिष्ट करते हैं, तो उस प्रकार के अनुरूप सुविधा बटन स्वचालित रूप से दिखाई देते हैं।

#### 🔘 कॉन्फ़िग कॉपी बटन — कनेक्शन सेटिंग स्वचालित रूप से बनाता है

बटन पर क्लिक करने पर, टोकन, प्रॉक्सी पता और मॉडल जानकारी पहले से भरा हुआ कॉन्फ़िगरेशन स्निपेट क्लिपबोर्ड में कॉपी हो जाता है। कॉपी की गई सामग्री को नीचे दी गई तालिका में बताए गए स्थान पर पेस्ट करने मात्र से कनेक्शन सेटअप पूरा हो जाता है।

| बटन | एजेंट प्रकार | पेस्ट करने का स्थान |
|------|-------------|-------------|
| 🦞 OpenClaw कॉन्फ़िग कॉपी | `openclaw` | `~/.openclaw/openclaw.json` |
| 🦀 NanoClaw कॉन्फ़िग कॉपी | `nanoclaw` | `~/.openclaw/openclaw.json` |
| 🟠 Claude Code कॉन्फ़िग कॉपी | `claude-code` | `~/.claude/settings.json` |
| ⌨ Cursor कॉन्फ़िग कॉपी | `cursor` | Cursor → Settings → AI |
| 💻 VSCode कॉन्फ़िग कॉपी | `vscode` | `~/.continue/config.json` |

**उदाहरण — यदि प्रकार Claude Code है, तो यह सामग्री कॉपी होती है:**

```json
// ~/.claude/settings.json
{
  "apiProvider": "openai",
  "baseUrl": "http://192.168.0.6:56244/v1",
  "apiKey": "इस-एजेंट-का-टोकन"
}
```

**उदाहरण — यदि प्रकार VSCode (Continue) है:**

```yaml
# ~/.continue/config.yaml  ← config.json नहीं बल्कि config.yaml में पेस्ट करें
name: My Config
version: 0.0.1
schema: v1

models:
  - name: wall-vault proxy
    provider: openai
    model: gemini-2.5-flash
    apiBase: http://192.168.0.6:56244/v1
    apiKey: इस-एजेंट-का-टोकन
    roles:
      - chat
      - edit
      - apply
```

> ⚠️ **Continue का नवीनतम संस्करण `config.yaml` का उपयोग करता है।** यदि `config.yaml` मौजूद है, तो `config.json` पूरी तरह से अनदेखा किया जाता है। सुनिश्चित करें कि `config.yaml` में ही पेस्ट करें।

**उदाहरण — यदि प्रकार Cursor है:**

```
Base URL : http://192.168.0.6:56244/v1
API Key  : इस-एजेंट-का-टोकन

// या पर्यावरण चर के माध्यम से:
OPENAI_BASE_URL=http://192.168.0.6:56244/v1
OPENAI_API_KEY=इस-एजेंट-का-टोकन
```

> ⚠️ **जब क्लिपबोर्ड कॉपी काम न करे**: ब्राउज़र सुरक्षा नीति कॉपी को ब्लॉक कर सकती है। यदि पॉपअप में टेक्स्टबॉक्स खुलता है, तो Ctrl+A से सब चुनें और Ctrl+C से कॉपी करें।

### VS Code में Cline का उपयोग — विस्तृत गाइड

#### चरण 1: Cline इंस्टॉल करें

VS Code एक्सटेंशन मार्केटप्लेस से **Cline** (ID: `saoudrizwan.claude-dev`) इंस्टॉल करें।

#### चरण 2: तिजोरी में एजेंट पंजीकृत करें

1. तिजोरी डैशबोर्ड (`http://तिजोरी-IP:56243`) खोलें
2. **एजेंट** सेक्शन में **+ जोड़ें** पर क्लिक करें
3. निम्नलिखित दर्ज करें:

| फ़ील्ड | मान | विवरण |
|------|----|------|
| ID | `my_cline` | अद्वितीय पहचानकर्ता (अंग्रेज़ी में, बिना स्पेस) |
| नाम | `My Cline` | डैशबोर्ड पर दिखाया जाने वाला नाम |
| एजेंट प्रकार | `cline` | ← अवश्य `cline` चुनें |
| सेवा | वांछित सेवा चुनें (उदाहरण: `google`) | |
| मॉडल | वांछित मॉडल दर्ज करें (उदाहरण: `gemini-2.5-flash`) | |

4. **सेव** पर क्लिक करने पर टोकन स्वचालित रूप से जनरेट होता है

#### चरण 3: Cline से कनेक्ट करें

**विधि A — स्वचालित लागू (अनुशंसित)**

1. सुनिश्चित करें कि wall-vault **प्रॉक्सी** उस मशीन पर चालू है (`localhost:56244`)
2. डैशबोर्ड के एजेंट कार्ड में **⚡ Cline कॉन्फ़िग लागू** बटन पर क्लिक करें
3. "कॉन्फ़िगरेशन लागू सफल!" अधिसूचना आने पर सफल
4. VS Code रीलोड करें (`Ctrl+Alt+R`)

**विधि B — मैन्युअल सेटिंग**

Cline साइडबार में सेटिंग (⚙️) खोलें:
- **API Provider**: `OpenAI Compatible`
- **Base URL**: `http://प्रॉक्सी-पता:56244/v1`
  - उसी मशीन पर: `http://localhost:56244/v1`
  - अन्य मशीन जैसे मिनी सर्वर पर: `http://192.168.0.6:56244/v1`
- **API Key**: तिजोरी से जारी टोकन (एजेंट कार्ड से कॉपी करें)
- **Model ID**: तिजोरी में सेट किया गया मॉडल (उदाहरण: `gemini-2.5-flash`)

#### चरण 4: सत्यापन

Cline चैट विंडो में कोई भी संदेश भेजें। यदि सामान्य है:
- तिजोरी डैशबोर्ड में संबंधित एजेंट कार्ड पर **हरा बिंदु (● चालू)** दिखेगा
- कार्ड में वर्तमान सेवा/मॉडल दिखेगा (उदाहरण: `google / gemini-2.5-flash`)

#### मॉडल बदलना

जब Cline का मॉडल बदलना हो, **तिजोरी डैशबोर्ड** से बदलें:

1. एजेंट कार्ड में सेवा/मॉडल ड्रॉपडाउन बदलें
2. **लागू करें** पर क्लिक करें
3. VS Code रीलोड करें (`Ctrl+Alt+R`) — Cline फुटर में मॉडल नाम अपडेट होगा
4. अगले अनुरोध से नया मॉडल उपयोग होगा

> 💡 वास्तव में, प्रॉक्सी Cline के अनुरोध को टोकन से पहचानकर तिजोरी सेटिंग के मॉडल पर रूट करता है। VS Code रीलोड न करने पर भी **वास्तव में उपयोग होने वाला मॉडल तुरंत बदल जाता है** — रीलोड केवल Cline UI में मॉडल प्रदर्शन अपडेट करने के लिए है।

#### कनेक्शन विच्छेद पता लगाना

| Cline फुटर मॉडल नहीं बदलता | Cline सेटिंग कैश करता है | VS Code रीलोड करें (`Ctrl+Alt+R`) |
| गलत मॉडल नाम दिखता है | पुराना बग (v0.1.16 में ठीक किया गया) | प्रॉक्सी को v0.1.16 या उससे ऊपर अपडेट करें |

#### 🟣 डिप्लॉय कमांड कॉपी बटन — नई मशीन पर इंस्टॉल करते समय उपयोग करें

नए कंप्यूटर पर पहली बार wall-vault प्रॉक्सी इंस्टॉल करके तिजोरी से कनेक्ट करते समय उपयोग करें। बटन पर क्लिक करने पर पूरा इंस्टॉलेशन स्क्रिप्ट कॉपी हो जाता है। नए कंप्यूटर के टर्मिनल में पेस्ट करें और चलाएं, सब एक साथ प्रोसेस होगा:

1. wall-vault बाइनरी इंस्टॉलेशन (पहले से इंस्टॉल होने पर स्किप)
2. systemd यूज़र सर्विस स्वचालित पंजीकरण
3. सर्विस शुरू और तिजोरी से स्वचालित कनेक्शन

> 💡 स्क्रिप्ट में इस एजेंट का टोकन और तिजोरी सर्वर पता पहले से भरा होता है, इसलिए पेस्ट करने के बाद बिना अतिरिक्त संशोधन के तुरंत चला सकते हैं।

### सर्विस कार्ड

AI सेवाओं को चालू/बंद करने और कॉन्फ़िगर करने का कार्ड।

- प्रत्येक सेवा के लिए सक्रिय/निष्क्रिय टॉगल स्विच
- लोकल AI सर्वर (आपके कंप्यूटर पर चलने वाले Ollama, LM Studio, vLLM आदि) का पता दर्ज करने पर उपलब्ध मॉडल स्वचालित रूप से पता लग जाते हैं।
- **लोकल सर्विस कनेक्शन स्टेटस इंडिकेटर**: सर्विस नाम के बगल में ● बिंदु कनेक्ट होने पर **हरा** और डिस्कनेक्ट होने पर **ग्रे** होता है
- **लोकल सर्विस ऑटो ट्रैफिक लाइट** (v0.1.23+): लोकल सर्विसेज (Ollama, LM Studio, vLLM) कनेक्शन उपलब्धता के अनुसार स्वचालित रूप से सक्रिय/निष्क्रिय होती हैं। सर्विस चालू करने पर 15 सेकंड के भीतर ● हरा हो जाता है और चेकबॉक्स चालू होता है; बंद करने पर स्वचालित रूप से बंद हो जाता है। क्लाउड सर्विसेज (Google, OpenRouter आदि) के API कुंजी आधारित ऑटो टॉगल के समान काम करता है।

> 💡 **यदि लोकल सर्विस किसी अन्य कंप्यूटर पर चल रही है**: सर्विस URL फ़ील्ड में उस कंप्यूटर का IP दर्ज करें। उदाहरण: `http://192.168.0.6:11434` (Ollama), `http://192.168.0.6:1234` (LM Studio)। यदि सर्विस `0.0.0.0` की बजाय `127.0.0.1` पर ही बाउंड है, तो बाहरी IP से एक्सेस नहीं होगा — सर्विस सेटिंग में बाइंडिंग पता जांचें।

### एडमिन टोकन इनपुट

डैशबोर्ड में कुंजी जोड़ने/हटाने जैसे महत्वपूर्ण कार्यों का उपयोग करने पर एडमिन टोकन इनपुट पॉपअप दिखाई देता है। setup विज़ार्ड में सेट किया गया टोकन दर्ज करें। एक बार दर्ज करने पर ब्राउज़र बंद होने तक मान्य रहता है।

> ⚠️ **यदि 15 मिनट में 10 से अधिक विफल प्रमाणीकरण प्रयास होते हैं, तो उस IP को अस्थायी रूप से ब्लॉक कर दिया जाएगा।** यदि टोकन भूल गए, तो `wall-vault.yaml` फ़ाइल में `admin_token` आइटम जांचें।

## वितरित मोड (मल्टी बॉट)

जब आप कई मशीनों पर एक साथ OpenClaw चलाते हैं, तो यह **एक कुंजी तिजोरी साझा** करने की कॉन्फ़िगरेशन है। सुविधाजनक क्योंकि कुंजी प्रबंधन एक ही जगह से होता है।

### कॉन्फ़िगरेशन उदाहरण

```
[कुंजी तिजोरी सर्वर]
  wall-vault vault    (कुंजी तिजोरी :56243, डैशबोर्ड)

[WSL अल्फा]          [Raspberry Pi गामा]    [Mac Mini लोकल]
  wall-vault proxy      wall-vault proxy        wall-vault proxy
  openclaw TUI          openclaw TUI            openclaw TUI
  ↕ SSE सिंक            ↕ SSE सिंक              ↕ SSE सिंक
```

सभी बॉट केंद्रीय तिजोरी सर्वर की ओर इंगित करते हैं, इसलिए तिजोरी में मॉडल बदलने या कुंजी जोड़ने पर सभी बॉट्स पर तुरंत प्रतिबिंबित होता है।

### चरण 1: कुंजी तिजोरी सर्वर शुरू करें

तिजोरी सर्वर के रूप में उपयोग होने वाले कंप्यूटर पर चलाएं:

```bash
wall-vault vault
```

### चरण 2: प्रत्येक बॉट (क्लाइंट) पंजीकृत करें

तिजोरी सर्वर से कनेक्ट होने वाले प्रत्येक बॉट की जानकारी पहले से पंजीकृत करें:

```bash
curl -X POST http://localhost:56243/admin/clients \
  -H "Authorization: Bearer एडमिन-टोकन" \
  -H "Content-Type: application/json" \
  -d '{
    "id": "botA",
    "name": "बॉट A",
    "token": "bota-secret",
    "default_service": "google",
    "default_model": "gemini-2.5-flash"
  }'
```

### चरण 3: प्रत्येक बॉट कंप्यूटर पर प्रॉक्सी शुरू करें

प्रत्येक बॉट कंप्यूटर पर तिजोरी सर्वर पता और टोकन निर्दिष्ट करके प्रॉक्सी चलाएं:

```bash
WV_VAULT_URL=http://192.168.x.x:56243 \
WV_VAULT_TOKEN=bota-secret \
WV_VAULT_CLIENT_ID=botA \
wall-vault proxy
```

> 💡 **`192.168.x.x`** को तिजोरी सर्वर कंप्यूटर के वास्तविक आंतरिक IP पते से बदलें। राउटर सेटिंग या `ip addr` कमांड से जांच सकते हैं।

## स्वचालित स्टार्ट सेटअप

यदि कंप्यूटर रीस्टार्ट करने पर हर बार मैन्युअल रूप से wall-vault शुरू करना असुविधाजनक है, तो इसे सिस्टम सर्विस के रूप में पंजीकृत करें। एक बार पंजीकृत होने पर, बूट पर स्वचालित रूप से शुरू होता है।

### Linux — systemd (अधिकांश Linux)

systemd Linux में प्रोग्राम को स्वचालित रूप से शुरू और प्रबंधित करने वाला सिस्टम है:

```bash
wall-vault doctor deploy
systemctl --user daemon-reload
systemctl --user enable --now wall-vault
```

लॉग देखें:

```bash
journalctl --user -u wall-vault -f
```

### macOS — launchd

macOS में प्रोग्राम के स्वचालित निष्पादन का प्रभारी सिस्टम:

```bash
wall-vault doctor deploy launchd
launchctl load ~/Library/LaunchAgents/com.wall-vault.plist
```

### Windows — NSSM

1. [nssm.cc](https://nssm.cc/download) से NSSM डाउनलोड करें और PATH में जोड़ें।
2. एडमिनिस्ट्रेटर PowerShell में:

```powershell
wall-vault doctor deploy windows
```

## Doctor (निदान)

`doctor` कमांड एक उपकरण है जो wall-vault सही ढंग से कॉन्फ़िगर है या नहीं इसका **स्वचालित रूप से निदान और मरम्मत** करता है।

```bash
wall-vault doctor check   # वर्तमान स्थिति का निदान (केवल पढ़ना, कुछ नहीं बदलता)
wall-vault doctor fix     # समस्याओं को स्वचालित रूप से ठीक करें
wall-vault doctor all     # निदान + स्वचालित मरम्मत एक साथ
```

> 💡 यदि कुछ अजीब लगे, तो पहले `wall-vault doctor all` चलाएं। यह कई समस्याओं को स्वचालित रूप से पकड़ता और ठीक करता है।

## पर्यावरण चर संदर्भ

पर्यावरण चर प्रोग्राम को कॉन्फ़िगरेशन मान पास करने का एक तरीका है। टर्मिनल में `export चर_नाम=मान` प्रारूप में दर्ज करें या स्थायी रूप से लागू करने के लिए ऑटो-स्टार्ट सर्विस फ़ाइल में रखें।

| चर | विवरण | उदाहरण मान |
|------|------|---------|
| `WV_LANG` | डैशबोर्ड भाषा | `ko`, `en`, `ja` |
| `WV_THEME` | डैशबोर्ड थीम | `light`, `dark`, `gold` |
| `WV_KEY_GOOGLE` | Google API कुंजी (कॉमा से अलग कई) | `AIza...,AIza...` |
| `WV_KEY_OPENROUTER` | OpenRouter API कुंजी | `sk-or-v1-...` |
| `WV_VAULT_URL` | वितरित मोड में तिजोरी सर्वर पता | `http://192.168.x.x:56243` |
| `WV_VAULT_TOKEN` | क्लाइंट (बॉट) प्रमाणीकरण टोकन | `my-secret-token` |
| `WV_ADMIN_TOKEN` | एडमिन टोकन | `admin-token-here` |
| `WV_MASTER_PASS` | API कुंजी एन्क्रिप्शन पासवर्ड | `my-password` |
| `WV_AVATAR` | अवतार छवि फ़ाइल पथ (`~/.openclaw/` से सापेक्ष पथ) | `workspace/avatars/avatar.png` |
| `OLLAMA_URL` | Ollama लोकल सर्वर पता | `http://192.168.x.x:11434` |

## समस्या निवारण

### जब प्रॉक्सी शुरू नहीं होता

आमतौर पर पोर्ट पहले से किसी अन्य प्रोग्राम द्वारा उपयोग में है।

```bash
ss -tlnp | grep 56244   # जांचें कि पोर्ट 56244 कौन उपयोग कर रहा है
wall-vault proxy --port 8080   # अलग पोर्ट नंबर से शुरू करें
```

### जब API कुंजी त्रुटियां आती हैं (429, 402, 401, 403, 582)

| त्रुटि कोड | अर्थ | कार्रवाई |
|----------|------|----------|
| **429** | बहुत अधिक अनुरोध (उपयोग सीमा पार) | कुछ देर प्रतीक्षा करें या अन्य कुंजी जोड़ें |
| **402** | भुगतान आवश्यक या क्रेडिट अपर्याप्त | संबंधित सेवा में क्रेडिट रीचार्ज करें |
| **401 / 403** | कुंजी गलत या अनुमति नहीं | कुंजी मान पुनः जांचें और पुनः पंजीकरण करें |
| **582** | गेटवे ओवरलोड (5 मिनट कूलडाउन) | 5 मिनट बाद स्वचालित रूप से हट जाता है |

```bash
# पंजीकृत कुंजियों की सूची और स्थिति जांचें
curl -H "Authorization: Bearer एडमिन-टोकन" http://localhost:56243/admin/keys

# कुंजी उपयोग काउंटर रीसेट करें
curl -X POST -H "Authorization: Bearer एडमिन-टोकन" http://localhost:56243/admin/keys/reset
```

### जब एजेंट "अकनेक्टेड" दिखता है

"अकनेक्टेड" का मतलब है कि प्रॉक्सी प्रक्रिया तिजोरी को सिग्नल (heartbeat) नहीं भेज रही। **इसका मतलब यह नहीं है कि सेटिंग सेव नहीं हुई।** प्रॉक्सी को तिजोरी सर्वर पता और टोकन के साथ चालू होना चाहिए तभी कनेक्टेड स्थिति में आएगा।

```bash
# तिजोरी सर्वर पता, टोकन और क्लाइंट ID निर्दिष्ट करके प्रॉक्सी शुरू करें
WV_VAULT_URL=http://तिजोरी-सर्वर-पता:56243 \
WV_VAULT_TOKEN=क्लाइंट-टोकन \
WV_VAULT_CLIENT_ID=क्लाइंट-ID \
wall-vault proxy
```

कनेक्शन सफल होने पर लगभग 20 सेकंड में डैशबोर्ड में 🟢 चालू हो जाएगा।

### जब Ollama कनेक्शन काम नहीं करता

Ollama एक प्रोग्राम है जो आपके कंप्यूटर पर सीधे AI चलाता है। पहले जांचें कि Ollama चालू है।

```bash
curl http://localhost:11434/api/tags   # यदि मॉडल सूची दिखती है तो सामान्य है
export OLLAMA_URL=http://192.168.x.x:11434   # यदि किसी अन्य कंप्यूटर पर चल रहा है
```

> ⚠️ यदि Ollama जवाब नहीं देता, तो पहले `ollama serve` कमांड से Ollama शुरू करें।

> ⚠️ **बड़े मॉडल की प्रतिक्रिया धीमी होती है**: `qwen3.5:35b`, `deepseek-r1` जैसे बड़े मॉडल को प्रतिक्रिया उत्पन्न करने में कई मिनट लग सकते हैं। भले ही ऐसा लगे कि कोई प्रतिक्रिया नहीं आ रही, यह सामान्य रूप से प्रोसेस हो रहा हो सकता है — कृपया प्रतीक्षा करें।

- **डैशबोर्ड सर्विस डिस्प्ले सुधार**: फॉलबैक होने पर भी डैशबोर्ड हमेशा उपयोगकर्ता द्वारा सेट की गई सर्विस दिखाता है।
- **लोकल सर्विस स्टेटस इंडिकेटर**: डैशबोर्ड लोड होने पर लोकल सर्विसेज (Ollama, LM Studio, vLLM आदि) की कनेक्शन स्थिति ● बिंदु के रंग से दिखाई जाती है।
- **टूल फ़िल्टर पर्यावरण चर**: `WV_TOOL_FILTER=passthrough` पर्यावरण चर से टूल्स पास-थ्रू मोड सेट किया जा सकता है।
