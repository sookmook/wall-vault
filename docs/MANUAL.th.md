## wall-vault คืออะไร?

**wall-vault = พร็อกซี AI + ตู้เซฟคีย์ API สำหรับ OpenClaw**

ในการใช้บริการ AI คุณต้องมี **คีย์ API** คีย์ API เปรียบเสมือน **บัตรผ่านดิจิทัล** ที่พิสูจน์ว่า "บุคคลนี้มีสิทธิ์ใช้บริการนี้" แต่บัตรผ่านนี้มีจำนวนครั้งการใช้งานจำกัดต่อวัน และอาจถูกเปิดเผยหากจัดการไม่ดี

wall-vault เก็บบัตรผ่านเหล่านี้ไว้ในตู้เซฟที่ปลอดภัย และทำหน้าที่เป็น **พร็อกซี (ตัวกลาง)** ระหว่าง OpenClaw และบริการ AI พูดง่ายๆ คือ OpenClaw ต้องเชื่อมต่อกับ wall-vault เท่านั้น และ wall-vault จะจัดการส่วนที่ซับซ้อนที่เหลือทั้งหมด

ปัญหาที่ wall-vault แก้ไขได้:

- **การหมุนเวียนคีย์ API อัตโนมัติ**: เมื่อคีย์หนึ่งถึงขีดจำกัดการใช้งานหรือถูกบล็อกชั่วคราว (cooldown) จะสลับไปใช้คีย์ถัดไปอย่างเงียบๆ OpenClaw ทำงานต่อเนื่องโดยไม่หยุดชะงัก
- **การเปลี่ยนบริการอัตโนมัติ (fallback)**: หาก Google ไม่ตอบสนอง จะสลับไปใช้ OpenRouter; หากนั่นก็ไม่ได้ จะสลับไปใช้ AI ในเครื่องที่ติดตั้งบนคอมพิวเตอร์ของคุณ (Ollama, LM Studio, vLLM) โดยอัตโนมัติ เซสชันไม่ขาดหาย เมื่อบริการเดิมกลับมาทำงาน จะกลับมาโดยอัตโนมัติตั้งแต่คำขอถัดไป (v0.1.18+, LM Studio/vLLM: v0.1.21+)
- **การซิงค์แบบเรียลไทม์ (SSE)**: เมื่อคุณเปลี่ยนโมเดลที่แดชบอร์ดตู้เซฟ การเปลี่ยนแปลงจะสะท้อนบนหน้าจอ OpenClaw ภายใน 1-3 วินาที SSE (Server-Sent Events) คือเทคโนโลยีที่เซิร์ฟเวอร์ส่งอัปเดตให้ไคลเอนต์แบบเรียลไทม์
- **การแจ้งเตือนแบบเรียลไทม์**: เหตุการณ์ต่างๆ เช่น คีย์หมดหรือบริการขัดข้อง จะแสดงทันทีที่ด้านล่างของ TUI (หน้าจอเทอร์มินัล) ของ OpenClaw

> 💡 **Claude Code, Cursor, VS Code** ก็สามารถเชื่อมต่อได้เช่นกัน แต่จุดประสงค์หลักของ wall-vault คือใช้ร่วมกับ OpenClaw

```
OpenClaw (อินเทอร์เฟซ TUI ในเทอร์มินัล)
        │
        ▼
  wall-vault proxy (:56244)   ← จัดการคีย์, เราท์ติ้ง, fallback, อีเวนต์
        │
        ├─ Google Gemini API
        ├─ OpenRouter API (มากกว่า 340 โมเดล)
        ├─ Ollama / LM Studio / vLLM (คอมพิวเตอร์ของคุณ, ทางเลือกสุดท้าย)
        └─ OpenAI / Anthropic API
```

## การติดตั้ง

### Linux / macOS

เปิดเทอร์มินัลแล้ววางคำสั่งด้านล่าง:

```bash
# Linux (PC ทั่วไป, เซิร์ฟเวอร์ — amd64)
curl -L https://github.com/sookmook/wall-vault/releases/latest/download/wall-vault-linux-amd64 \
  -o ~/.local/bin/wall-vault && chmod +x ~/.local/bin/wall-vault

# macOS Apple Silicon (M1/M2/M3 Mac)
curl -L https://github.com/sookmook/wall-vault/releases/latest/download/wall-vault-darwin-arm64 \
  -o /usr/local/bin/wall-vault && chmod +x /usr/local/bin/wall-vault
```

- `curl -L ...` — ดาวน์โหลดไฟล์จากอินเทอร์เน็ต
- `chmod +x` — ทำให้ไฟล์ที่ดาวน์โหลด "สามารถรันได้" หากข้ามขั้นตอนนี้ จะเกิดข้อผิดพลาด "ไม่อนุญาต"

### Windows

เปิด PowerShell (ในฐานะผู้ดูแลระบบ) แล้วรันคำสั่งต่อไปนี้:

```powershell
# ดาวน์โหลด
Invoke-WebRequest -Uri `
  "https://github.com/sookmook/wall-vault/releases/latest/download/wall-vault-windows-amd64.exe" `
  -OutFile "$env:LOCALAPPDATA\Programs\wall-vault\wall-vault.exe"

# เพิ่มใน PATH (มีผลหลังรีสตาร์ท PowerShell)
$env:PATH += ";$env:LOCALAPPDATA\Programs\wall-vault"
```

> 💡 **PATH คืออะไร?** คือรายการโฟลเดอร์ที่คอมพิวเตอร์ค้นหาคำสั่ง เมื่อเพิ่มใน PATH แล้ว คุณสามารถรัน `wall-vault` จากโฟลเดอร์ใดก็ได้โดยพิมพ์ชื่อเท่านั้น

### บิลด์จากซอร์สโค้ด (สำหรับนักพัฒนา)

ใช้ได้เฉพาะเมื่อติดตั้งสภาพแวดล้อมการพัฒนาภาษา Go แล้ว

```bash
git clone https://github.com/sookmook/wall-vault
cd wall-vault
make build       # bin/wall-vault (เวอร์ชัน: v0.1.24.YYYYMMDD.HHmmss)
make install     # ~/.local/bin/wall-vault
```

> 💡 **เวอร์ชันพร้อม timestamp**: เมื่อบิลด์ด้วย `make build` เวอร์ชันจะถูกสร้างโดยอัตโนมัติในรูปแบบที่รวมวันที่และเวลา เช่น `v0.1.24.20260406.225957` หากบิลด์โดยตรงด้วย `go build ./...` เวอร์ชันจะแสดงเป็น `"dev"` เท่านั้น

## เริ่มต้นใช้งานครั้งแรก

### รันวิซาร์ดตั้งค่า

หลังจากติดตั้ง ต้องรัน **วิซาร์ดตั้งค่า** ด้วยคำสั่งด้านล่าง วิซาร์ดจะนำทางคุณโดยถามทีละรายการที่จำเป็น

```bash
wall-vault setup
```

ขั้นตอนที่วิซาร์ดดำเนินการ:

```
1. เลือกภาษา (10 ภาษา รวมถึงภาษาเกาหลี)
2. เลือกธีม (light / dark / gold / cherry / ocean)
3. โหมดใช้งาน — ใช้คนเดียว (standalone) หรือใช้ร่วมกันหลายเครื่อง (distributed)
4. ชื่อบอท — ชื่อที่แสดงบนแดชบอร์ด
5. ตั้งค่าพอร์ต — ค่าเริ่มต้น: พร็อกซี 56244, ตู้เซฟ 56243 (กด Enter หากไม่ต้องเปลี่ยน)
6. เลือกบริการ AI — Google / OpenRouter / Ollama / LM Studio / vLLM
7. ตั้งค่าตัวกรองความปลอดภัยของเครื่องมือ
8. ตั้งค่าโทเค็นแอดมิน — รหัสผ่านที่ล็อกฟังก์ชันจัดการแดชบอร์ด สามารถสร้างอัตโนมัติได้
9. ตั้งค่ารหัสผ่านเข้ารหัสคีย์ API — สำหรับการจัดเก็บที่ปลอดภัยยิ่งขึ้น (ไม่บังคับ)
10. พาธบันทึกไฟล์คอนฟิก
```

> ⚠️ **อย่าลืมจำโทเค็นแอดมิน** จะต้องใช้ในภายหลังเพื่อเพิ่มคีย์หรือเปลี่ยนการตั้งค่าในแดชบอร์ด หากทำหาย จะต้องแก้ไขไฟล์คอนฟิกด้วยตนเอง

เมื่อวิซาร์ดเสร็จสิ้น ไฟล์คอนฟิก `wall-vault.yaml` จะถูกสร้างโดยอัตโนมัติ

### รัน

```bash
wall-vault start
```

เซิร์ฟเวอร์สองตัวด้านล่างจะเริ่มต้นพร้อมกัน:

- **พร็อกซี** (`http://localhost:56244`) — ตัวกลางที่เชื่อมต่อ OpenClaw กับบริการ AI
- **ตู้เซฟคีย์** (`http://localhost:56243`) — จัดการคีย์ API และแดชบอร์ดเว็บ

เปิด `http://localhost:56243` ในเบราว์เซอร์เพื่อเข้าถึงแดชบอร์ดทันที

## การลงทะเบียนคีย์ API

มีสี่วิธีในการลงทะเบียนคีย์ API **สำหรับผู้เริ่มต้น แนะนำวิธีที่ 1 (ตัวแปรสภาพแวดล้อม)**

### วิธีที่ 1: ตัวแปรสภาพแวดล้อม (แนะนำ — ง่ายที่สุด)

ตัวแปรสภาพแวดล้อมคือ **ค่าที่ตั้งไว้ล่วงหน้า** ที่โปรแกรมอ่านเมื่อเริ่มต้น ป้อนในเทอร์มินัลดังนี้:

```bash
# ลงทะเบียนคีย์ Google Gemini
export WV_KEY_GOOGLE=AIzaSy...

# ลงทะเบียนคีย์ OpenRouter
export WV_KEY_OPENROUTER=sk-or-v1-...

# รันหลังจากลงทะเบียน
wall-vault start
```

หากคุณมีหลายคีย์ ให้เชื่อมด้วยเครื่องหมายจุลภาค (,) wall-vault จะใช้สลับกันอัตโนมัติ (round robin):

```bash
export WV_KEY_GOOGLE=AIzaSy...,AIzaSy...,AIzaSy...
```

> 💡 **เคล็ดลับ**: คำสั่ง `export` ใช้ได้เฉพาะเซสชันเทอร์มินัลปัจจุบัน หากต้องการให้คงอยู่หลังรีสตาร์ทคอมพิวเตอร์ ให้เพิ่มบรรทัดข้างต้นในไฟล์ `~/.bashrc` หรือ `~/.zshrc`

### วิธีที่ 2: UI แดชบอร์ด (คลิกด้วยเมาส์)

1. เข้าถึง `http://localhost:56243` ในเบราว์เซอร์
2. คลิกปุ่ม `[+ เพิ่ม]` ในการ์ด **🔑 คีย์ API** ด้านบน
3. ป้อนประเภทบริการ ค่าคีย์ ป้ายกำกับ (ชื่ออ้างอิง) และขีดจำกัดรายวัน แล้วบันทึก

### วิธีที่ 3: REST API (สำหรับอัตโนมัติ/สคริปต์)

REST API คือวิธีที่โปรแกรมแลกเปลี่ยนข้อมูลผ่าน HTTP มีประโยชน์สำหรับการลงทะเบียนอัตโนมัติผ่านสคริปต์

```bash
curl -X POST http://localhost:56243/admin/keys \
  -H "Authorization: Bearer โทเค็น-แอดมิน" \
  -H "Content-Type: application/json" \
  -d '{
    "service": "google",
    "key": "AIzaSy...",
    "label": "คีย์หลัก",
    "daily_limit": 1000
  }'
```

### วิธีที่ 4: แฟล็กพร็อกซี (สำหรับทดสอบด่วน)

ใช้เพื่อทดสอบชั่วคราวโดยใส่คีย์โดยไม่ลงทะเบียนอย่างเป็นทางการ คีย์จะหายไปเมื่อปิดโปรแกรม

```bash
wall-vault proxy --key-google=AIzaSy... --key-openrouter=sk-or-...
```

## วิธีใช้พร็อกซี

### ใช้กับ OpenClaw (จุดประสงค์หลัก)

วิธีตั้งค่า OpenClaw เพื่อเชื่อมต่อกับบริการ AI ผ่าน wall-vault

เปิดไฟล์ `~/.openclaw/openclaw.json` แล้วเพิ่มเนื้อหาต่อไปนี้:

```json5
// ~/.openclaw/openclaw.json
{
  models: {
    providers: {
      "wall-vault": {
        baseUrl: "http://localhost:56244/v1",
        apiKey: "your-agent-token",   // โทเค็นเอเจนต์ของตู้เซฟ
        api: "openai-completions",
        models: [
          { id: "wall-vault/gemini-2.5-flash" },
          { id: "wall-vault/gemini-2.5-pro" },
          { id: "wall-vault/hunter-alpha" },    // context 1M ฟรี
          { id: "wall-vault/claude-opus-4-6" }
        ]
      }
    }
  }
}
```

> 💡 **วิธีที่ง่ายกว่า**: กดปุ่ม **🦞 คัดลอกคอนฟิก OpenClaw** ที่การ์ดเอเจนต์บนแดชบอร์ด แล้วสนิปเพ็ตที่มีโทเค็นและที่อยู่กรอกไว้แล้วจะถูกคัดลอกไปยังคลิปบอร์ด แค่วางก็เสร็จ

**`wall-vault/` ที่ต้นชื่อโมเดลนำไปที่ไหน?**

wall-vault กำหนดโดยอัตโนมัติว่าจะส่งคำขอไปยังบริการ AI ใดโดยอิงจากชื่อโมเดล:

| รูปแบบโมเดล | บริการที่เชื่อมต่อ |
|----------|--------------|
| `wall-vault/gemini-*` | เชื่อมต่อ Google Gemini โดยตรง |
| `wall-vault/gpt-*`, `wall-vault/o3`, `wall-vault/o4*` | เชื่อมต่อ OpenAI โดยตรง |
| `wall-vault/claude-*` | เชื่อมต่อ Anthropic ผ่าน OpenRouter |
| `wall-vault/hunter-alpha`, `wall-vault/healer-alpha` | OpenRouter (context 1 ล้านโทเค็นฟรี) |
| `wall-vault/kimi-*`, `wall-vault/glm-*`, `wall-vault/deepseek-*` | เชื่อมต่อผ่าน OpenRouter |
| `google/ชื่อโมเดล`, `openai/ชื่อโมเดล`, `anthropic/ชื่อโมเดล` ฯลฯ | เชื่อมต่อบริการที่เกี่ยวข้องโดยตรง |
| `custom/google/ชื่อโมเดล`, `custom/openai/ชื่อโมเดล` ฯลฯ | ลบส่วน `custom/` แล้วเราท์ใหม่ |
| `ชื่อโมเดล:cloud` | ลบส่วน `:cloud` แล้วเชื่อมต่อผ่าน OpenRouter |

> 💡 **Context คืออะไร?** คือปริมาณบทสนทนาที่ AI สามารถ "จำ" ได้ในคราวเดียว 1M (หนึ่งล้านโทเค็น) หมายความว่าสามารถประมวลผลบทสนทนายาวมากหรือเอกสารยาวได้ในคราวเดียว

### เชื่อมต่อโดยตรงในรูปแบบ Gemini API (ความเข้ากันได้กับเครื่องมือที่มีอยู่)

หากคุณมีเครื่องมือที่ใช้ Google Gemini API โดยตรง เพียงเปลี่ยนที่อยู่เป็น wall-vault:

```bash
export ANTHROPIC_BASE_URL=http://localhost:56244/google
```

หรือสำหรับเครื่องมือที่ระบุ URL โดยตรง:

```
http://localhost:56244/google/v1beta/models/gemini-2.5-flash:generateContent
```

### ใช้กับ OpenAI SDK (Python)

คุณสามารถเชื่อมต่อ wall-vault ในโค้ด Python ที่ใช้ AI ได้เช่นกัน เพียงเปลี่ยน `base_url`:

```python
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:56244/v1",
    api_key="not-needed"  # wall-vault จัดการคีย์ API โดยอัตโนมัติ
)

response = client.chat.completions.create(
    model="google/gemini-2.5-flash",   # ป้อนในรูปแบบ provider/model
    messages=[{"role": "user", "content": "สวัสดี"}]
)
```

### เปลี่ยนโมเดลระหว่างรัน

เพื่อเปลี่ยนโมเดล AI ขณะที่ wall-vault กำลังทำงาน:

```bash
# เปลี่ยนโมเดลโดยส่งคำขอตรงไปยังพร็อกซี
curl -X PUT http://localhost:56244/api/config/model \
  -H "Content-Type: application/json" \
  -d '{"service": "openrouter", "model": "anthropic/claude-3.5-sonnet"}'

## แดชบอร์ดตู้เซฟคีย์

เข้าถึง `http://localhost:56243` ในเบราว์เซอร์เพื่อดูแดชบอร์ด

**โครงสร้างหน้าจอ:**
- **แถบด้านบนคงที่ (topbar)**: โลโก้, ตัวเลือกภาษาและธีม, ตัวบ่งชี้สถานะการเชื่อมต่อ SSE
- **กริดการ์ด**: การ์ดเอเจนต์, บริการ และคีย์ API จัดเรียงในรูปแบบไทล์

### การ์ดคีย์ API

การ์ดสำหรับจัดการคีย์ API ที่ลงทะเบียนทั้งหมดในมุมมองเดียว

- แสดงรายการคีย์แยกตามบริการ
- `today_usage`: โทเค็น (จำนวนอักขระที่ AI อ่านและเขียน) ที่ประมวลผลสำเร็จวันนี้
- `today_attempts`: จำนวนการเรียกทั้งหมดวันนี้ (รวมสำเร็จ + ล้มเหลว)
- ปุ่ม `[+ เพิ่ม]` สำหรับลงทะเบียนคีย์ใหม่ และ `✕` สำหรับลบ

> 💡 **โทเค็นคืออะไร?** คือหน่วยที่ AI ใช้ประมวลผลข้อความ เทียบได้ประมาณหนึ่งคำในภาษาอังกฤษ หรือ 1-2 ตัวอักษรภาษาเกาหลี ค่าบริการ API มักคำนวณตามจำนวนโทเค็นนี้

### การ์ดเอเจนต์

การ์ดที่แสดงสถานะของบอท (เอเจนต์) ที่เชื่อมต่อกับพร็อกซี wall-vault

**สถานะการเชื่อมต่อแสดงใน 4 ระดับ:**

| ตัวบ่งชี้ | สถานะ | ความหมาย |
|------|------|------|
| 🟢 | กำลังทำงาน | พร็อกซีทำงานปกติ |
| 🟡 | ล่าช้า | ตอบสนองแต่ช้า |
| 🔴 | ออฟไลน์ | พร็อกซีไม่ตอบสนอง |
| ⚫ | ยังไม่เชื่อมต่อ/ปิดใช้งาน | พร็อกซีไม่เคยเชื่อมต่อตู้เซฟ หรือปิดใช้งาน |

**คู่มือปุ่มด้านล่างการ์ดเอเจนต์:**

เมื่อลงทะเบียนเอเจนต์ หากคุณระบุ **ประเภทเอเจนต์** ปุ่มอำนวยความสะดวกที่เหมาะกับประเภทนั้นจะปรากฏโดยอัตโนมัติ

#### 🔘 ปุ่มคัดลอกคอนฟิก — สร้างการตั้งค่าการเชื่อมต่อโดยอัตโนมัติ

| ปุ่ม | ประเภทเอเจนต์ | ตำแหน่งที่จะวาง |
|------|-------------|-------------|
| 🦞 คัดลอกคอนฟิก OpenClaw | `openclaw` | `~/.openclaw/openclaw.json` |

| ⌨ คัดลอกคอนฟิก Cursor | `cursor` | Cursor → Settings → AI |
| 💻 คัดลอกคอนฟิก VSCode | `vscode` | `~/.continue/config.json` |

**ตัวอย่าง — หากประเภทเป็น Claude Code เนื้อหาต่อไปนี้จะถูกคัดลอก:**

version: 0.0.1
schema: v1

    provider: openai
    model: gemini-2.5-flash

      - chat
      - edit
      - apply
```

> ⚠️ **Continue เวอร์ชันล่าสุดใช้ `config.yaml`** หาก `config.yaml` มีอยู่ `config.json` จะถูกเพิกเฉยโดยสมบูรณ์ ตรวจสอบให้แน่ใจว่าวางใน `config.yaml`

**ตัวอย่าง — หากประเภทเป็น Cursor:**

**กรณี Cline:**
- เปลี่ยนโมเดลที่ตู้เซฟ → อีเวนต์ SSE → พร็อกซีอัปเดตฟิลด์โมเดลใน `globalState.json`

- `openAiBaseUrl` และคีย์ API ไม่ถูกเปลี่ยน
- **ต้องรีโหลด VS Code (`Ctrl+Alt+R` หรือ `Ctrl+Shift+P` → `Developer: Reload Window`)**
  - เพราะ Cline ไม่อ่านไฟล์คอนฟิกซ้ำระหว่างทำงาน

**กรณี Claude Code:**
- เปลี่ยนโมเดลที่ตู้เซฟ → อีเวนต์ SSE → พร็อกซีอัปเดตฟิลด์ `model` ใน `settings.json`
- ค้นหาอัตโนมัติทั้งพาธ WSL และ Windows (`~/.claude/`, `/mnt/c/Users/*/.claude/`)

**ทิศทางตรงข้าม (เอเจนต์ → ตู้เซฟ):**
- เมื่อเอเจนต์ (Cline, Claude Code ฯลฯ) ส่งคำขอไปยังพร็อกซี พร็อกซีจะรวมข้อมูลบริการ/โมเดลของไคลเอนต์ใน heartbeat
- บริการ/โมเดลที่กำลังใช้จะแสดงแบบเรียลไทม์ในการ์ดเอเจนต์บนแดชบอร์ดตู้เซฟ

> 💡 **จุดสำคัญ**: พร็อกซีระบุเอเจนต์จากโทเค็น Authorization ในคำขอและเราท์ไปยังบริการ/โมเดลที่ตั้งค่าในตู้เซฟโดยอัตโนมัติ แม้ Cline หรือ Claude Code จะส่งชื่อโมเดลอื่น พร็อกซีก็จะแทนที่ด้วยการตั้งค่าของตู้เซฟ

### ใช้ Cline ใน VS Code — คู่มือโดยละเอียด

#### ขั้นตอนที่ 1: ติดตั้ง Cline

ติดตั้ง **Cline** (ID: `saoudrizwan.claude-dev`) จากมาร์เก็ตเพลสส่วนขยาย VS Code

#### ขั้นตอนที่ 2: ลงทะเบียนเอเจนต์ในตู้เซฟ

1. เปิดแดชบอร์ดตู้เซฟ (`http://IP-ตู้เซฟ:56243`)
2. ในส่วน **เอเจนต์** คลิก **+ เพิ่ม**
3. ป้อนดังนี้:

| ฟิลด์ | ค่า | คำอธิบาย |
|------|----|------|
| ID | `my_cline` | ตัวระบุเฉพาะ (ภาษาอังกฤษ ไม่มีช่องว่าง) |
| ชื่อ | `My Cline` | ชื่อที่แสดงบนแดชบอร์ด |
| ประเภทเอเจนต์ | `cline` | ← ต้องเลือก `cline` |
| บริการ | เลือกบริการที่ต้องการ (ตัวอย่าง: `google`) | |
| โมเดล | ป้อนโมเดลที่ต้องการ (ตัวอย่าง: `gemini-2.5-flash`) | |

4. คลิก **บันทึก** แล้วโทเค็นจะถูกสร้างโดยอัตโนมัติ

#### ขั้นตอนที่ 3: เชื่อมต่อกับ Cline

**วิธี A — ใช้งานอัตโนมัติ (แนะนำ)**

1. ตรวจสอบว่า **พร็อกซี** wall-vault กำลังทำงานบนเครื่องนั้น (`localhost:56244`)
2. คลิกปุ่ม **⚡ ใช้คอนฟิก Cline** ที่การ์ดเอเจนต์บนแดชบอร์ด
3. หากแสดงการแจ้งเตือน "ใช้คอนฟิกสำเร็จ!" แสดงว่าสำเร็จ
4. รีโหลด VS Code (`Ctrl+Alt+R`)

**วิธี B — ตั้งค่าด้วยตนเอง**

เปิดการตั้งค่า (⚙️) ในแถบข้างของ Cline:
- **API Provider**: `OpenAI Compatible`
- **Base URL**: `http://ที่อยู่-พร็อกซี:56244/v1`
  - เครื่องเดียวกัน: `http://localhost:56244/v1`
  - เครื่องอื่นเช่นเซิร์ฟเวอร์มินิ: `http://192.168.1.20:56244/v1`
- **API Key**: โทเค็นที่ออกจากตู้เซฟ (คัดลอกจากการ์ดเอเจนต์)
- **Model ID**: โมเดลที่ตั้งค่าในตู้เซฟ (ตัวอย่าง: `gemini-2.5-flash`)

#### ขั้นตอนที่ 4: ตรวจสอบ

ส่งข้อความใดก็ได้ในหน้าต่างแชทของ Cline หากปกติ:
- การ์ดเอเจนต์ที่เกี่ยวข้องบนแดชบอร์ดตู้เซฟจะแสดง **จุดเขียว (● กำลังทำงาน)**
- บริการ/โมเดลปัจจุบันจะแสดงบนการ์ด (ตัวอย่าง: `google / gemini-2.5-flash`)

#### เปลี่ยนโมเดล

เมื่อต้องการเปลี่ยนโมเดลของ Cline ให้เปลี่ยนจาก **แดชบอร์ดตู้เซฟ**:

1. เปลี่ยนดรอปดาวน์บริการ/โมเดลในการ์ดเอเจนต์
2. คลิก **ใช้**
3. รีโหลด VS Code (`Ctrl+Alt+R`) — ชื่อโมเดลที่ฟุตเตอร์ของ Cline จะอัปเดต
4. โมเดลใหม่จะถูกใช้ตั้งแต่คำขอถัดไป

> 💡 จริงๆ แล้ว พร็อกซีระบุคำขอของ Cline ด้วยโทเค็นและเราท์ไปยังโมเดลที่ตั้งค่าในตู้เซฟ แม้ไม่รีโหลด VS Code **โมเดลที่ใช้จริงก็เปลี่ยนทันที** — การรีโหลดเป็นเพียงเพื่ออัปเดตการแสดงโมเดลใน UI ของ Cline

ใช้เมื่อติดตั้งพร็อกซี wall-vault ครั้งแรกบนคอมพิวเตอร์ใหม่และเชื่อมต่อกับตู้เซฟ เมื่อคลิกปุ่ม สคริปต์ติดตั้งทั้งหมดจะถูกคัดลอก วางในเทอร์มินัลของคอมพิวเตอร์ใหม่แล้วรัน ทุกอย่างจะถูกประมวลผลพร้อมกัน:

1. ติดตั้งไบนารี wall-vault (ข้ามหากติดตั้งแล้ว)
2. ลงทะเบียนบริการผู้ใช้ systemd อัตโนมัติ
3. เริ่มบริการและเชื่อมต่อตู้เซฟอัตโนมัติ

> 💡 สคริปต์มีโทเค็นของเอเจนต์นี้และที่อยู่เซิร์ฟเวอร์ตู้เซฟกรอกไว้แล้ว จึงสามารถรันได้ทันทีหลังวาง โดยไม่ต้องแก้ไขเพิ่มเติม

### การ์ดบริการ

การ์ดสำหรับเปิด/ปิดใช้งานและตั้งค่าบริการ AI

- สวิตช์เปิด/ปิดใช้งานต่อบริการ
- เมื่อป้อนที่อยู่เซิร์ฟเวอร์ AI ในเครื่อง (Ollama, LM Studio, vLLM ฯลฯ ที่ทำงานบนคอมพิวเตอร์ของคุณ) จะตรวจจับโมเดลที่ใช้ได้โดยอัตโนมัติ
- **ตัวบ่งชี้สถานะการเชื่อมต่อบริการในเครื่อง**: จุด ● ข้างชื่อบริการจะเป็นสี **เขียว** เมื่อเชื่อมต่อแล้ว และ **เทา** เมื่อไม่ได้เชื่อมต่อ
- **ไฟจราจรอัตโนมัติบริการในเครื่อง** (v0.1.23+): บริการในเครื่อง (Ollama, LM Studio, vLLM) จะเปิด/ปิดใช้งานอัตโนมัติตามความพร้อมของการเชื่อมต่อ เมื่อเปิดบริการ จะเปลี่ยนเป็น ● เขียวภายใน 15 วินาที และเช็คบ็อกซ์จะถูกเปิด; เมื่อปิด จะปิดอัตโนมัติ ทำงานในลักษณะเดียวกับบริการคลาวด์ (Google, OpenRouter ฯลฯ) ที่สลับอัตโนมัติตามการมีคีย์ API

> 💡 **หากบริการในเครื่องทำงานบนคอมพิวเตอร์อื่น**: ป้อน IP ของคอมพิวเตอร์นั้นในช่อง URL บริการ ตัวอย่าง: `http://192.168.1.20:11434` (Ollama), `http://192.168.1.20:1234` (LM Studio) หากบริการผูกกับ `127.0.0.1` เท่านั้น ไม่ใช่ `0.0.0.0` จะไม่สามารถเข้าถึงผ่าน IP ภายนอกได้ — ตรวจสอบที่อยู่ binding ในการตั้งค่าบริการ

### ป้อนโทเค็นแอดมิน

เมื่อพยายามใช้ฟังก์ชันสำคัญในแดชบอร์ด เช่น เพิ่ม/ลบคีย์ ป็อปอัปป้อนโทเค็นแอดมินจะปรากฏ ป้อนโทเค็นที่ตั้งค่าในวิซาร์ดตั้งค่า เมื่อป้อนแล้วจะมีผลจนกว่าจะปิดเบราว์เซอร์

> ⚠️ **หากการพิสูจน์ตัวตนล้มเหลวเกิน 10 ครั้งใน 15 นาที IP นั้นจะถูกบล็อกชั่วคราว** หากลืมโทเค็น ตรวจสอบรายการ `admin_token` ในไฟล์ `wall-vault.yaml`

## โหมดกระจาย (หลายบอท)

เมื่อใช้งาน OpenClaw พร้อมกันบนหลายเครื่อง นี่คือการตั้งค่าสำหรับ **แชร์ตู้เซฟคีย์เดียว** สะดวกเพราะจัดการคีย์ในที่เดียว

### ตัวอย่างการตั้งค่า

```
[เซิร์ฟเวอร์ตู้เซฟคีย์]
  wall-vault vault    (ตู้เซฟคีย์ :56243, แดชบอร์ด)

[WSL อัลฟ่า]          [Raspberry Pi แกมม่า]    [Mac Mini ในเครื่อง]
  wall-vault proxy      wall-vault proxy        wall-vault proxy
  openclaw TUI          openclaw TUI            openclaw TUI
  ↕ ซิงค์ SSE           ↕ ซิงค์ SSE             ↕ ซิงค์ SSE
```

บอททุกตัวชี้ไปที่เซิร์ฟเวอร์ตู้เซฟกลาง ดังนั้นเมื่อเปลี่ยนโมเดลหรือเพิ่มคีย์ที่ตู้เซฟ จะสะท้อนทันทีบนบอททุกตัว

### ขั้นตอนที่ 1: เริ่มเซิร์ฟเวอร์ตู้เซฟคีย์

รันบนคอมพิวเตอร์ที่จะใช้เป็นเซิร์ฟเวอร์ตู้เซฟ:

```bash
wall-vault vault
```

### ขั้นตอนที่ 2: ลงทะเบียนแต่ละบอท (ไคลเอนต์)

ลงทะเบียนข้อมูลของแต่ละบอทที่จะเชื่อมต่อกับเซิร์ฟเวอร์ตู้เซฟล่วงหน้า:

```bash
curl -X POST http://localhost:56243/admin/clients \
  -H "Authorization: Bearer โทเค็น-แอดมิน" \
  -H "Content-Type: application/json" \
  -d '{
    "id": "botA",
    "name": "บอท A",
    "token": "bota-secret",
    "default_service": "google",
    "default_model": "gemini-2.5-flash"
  }'
```

### ขั้นตอนที่ 3: เริ่มพร็อกซีบนคอมพิวเตอร์แต่ละบอท

บนคอมพิวเตอร์แต่ละตัวที่ติดตั้งบอท รันพร็อกซีโดยระบุที่อยู่และโทเค็นของเซิร์ฟเวอร์ตู้เซฟ:

```bash
WV_VAULT_URL=http://192.168.x.x:56243 \
WV_VAULT_TOKEN=bota-secret \
WV_VAULT_CLIENT_ID=botA \
wall-vault proxy
```

> 💡 **`192.168.x.x`** ต้องเปลี่ยนเป็นที่อยู่ IP ภายในจริงของคอมพิวเตอร์เซิร์ฟเวอร์ตู้เซฟ สามารถตรวจสอบได้จากการตั้งค่าเราเตอร์หรือคำสั่ง `ip addr`

## การตั้งค่าเริ่มต้นอัตโนมัติ

หากไม่สะดวกที่จะเริ่ม wall-vault ด้วยตนเองทุกครั้งที่รีสตาร์ทคอมพิวเตอร์ ให้ลงทะเบียนเป็นบริการระบบ เมื่อลงทะเบียนแล้ว จะเริ่มต้นอัตโนมัติเมื่อบูต

### Linux — systemd (Linux ส่วนใหญ่)

systemd คือระบบที่เริ่มต้นและจัดการโปรแกรมอัตโนมัติใน Linux:

```bash
wall-vault doctor deploy
systemctl --user daemon-reload
systemctl --user enable --now wall-vault
```

ดูล็อก:

```bash
journalctl --user -u wall-vault -f
```

### macOS — launchd

ระบบที่รับผิดชอบการรันโปรแกรมอัตโนมัติใน macOS:

```bash
wall-vault doctor deploy launchd
launchctl load ~/Library/LaunchAgents/com.wall-vault.plist
```

### Windows — NSSM

1. ดาวน์โหลด NSSM จาก [nssm.cc](https://nssm.cc/download) แล้วเพิ่มใน PATH
2. ใน PowerShell ในฐานะผู้ดูแลระบบ:

```powershell
wall-vault doctor deploy windows
```

> 💡 หากรู้สึกว่ามีอะไรผิดปกติ ให้รัน `wall-vault doctor all` ก่อน จะตรวจจับและแก้ไขปัญหาหลายอย่างอัตโนมัติ

## RTK ประหยัดโทเค็น

*(v0.1.24+)*

**RTK (เครื่องมือประหยัดโทเค็น)** บีบอัดเอาต์พุตของคำสั่ง shell ที่รันโดยเอเจนต์เขียนโค้ด AI (เช่น Claude Code) โดยอัตโนมัติ ช่วยลดการใช้โทเค็น ตัวอย่างเช่น เอาต์พุต 15 บรรทัดของ `git status` จะถูกย่อเหลือสรุป 2 บรรทัด

### การใช้งานพื้นฐาน

```bash
# ครอบคำสั่งด้วย wall-vault rtk แล้วเอาต์พุตจะถูกกรองอัตโนมัติ
wall-vault rtk git status          # แสดงเฉพาะรายการไฟล์ที่เปลี่ยน
wall-vault rtk git diff HEAD~1     # เฉพาะบรรทัดที่เปลี่ยน + บริบทขั้นต่ำ
wall-vault rtk git log -10         # แฮช + ข้อความหนึ่งบรรทัด
wall-vault rtk go test ./...       # แสดงเฉพาะเทสต์ที่ล้มเหลว
wall-vault rtk ls -la              # คำสั่งที่ไม่รองรับจะถูกตัดอัตโนมัติ
```

### คำสั่งที่รองรับและการประหยัด

| คำสั่ง | วิธีกรอง | การประหยัด |
|------|----------|--------|
| `git status` | เฉพาะสรุปไฟล์ที่เปลี่ยน | ~87% |
| `git diff` | บรรทัดที่เปลี่ยน + 3 บรรทัดบริบท | ~60-94% |
| `git log` | แฮช + บรรทัดแรกของข้อความ | ~90% |
| `git push/pull/fetch` | ลบความคืบหน้า เฉพาะสรุป | ~80% |
| `go test` | แสดงเฉพาะที่ล้มเหลว นับที่ผ่าน | ~88-99% |
| `go build/vet` | แสดงเฉพาะข้อผิดพลาด | ~90% |
| คำสั่งอื่นทั้งหมด | 50 บรรทัดแรก + 50 บรรทัดท้าย สูงสุด 32KB | แปรผัน |

### ไปป์ไลน์กรอง 3 ขั้นตอน

1. **ตัวกรองโครงสร้างตามคำสั่ง** — เข้าใจรูปแบบเอาต์พุตของ git, go ฯลฯ และดึงเฉพาะส่วนที่มีความหมาย
2. **โพสต์โปรเซสด้วย regex** — ลบรหัสสี ANSI ลดบรรทัดว่าง รวมบรรทัดซ้ำ
3. **Passthrough + ตัด** — คำสั่งที่ไม่รองรับเก็บเฉพาะ 50 บรรทัดแรก/ท้าย

### การเชื่อมต่อ Claude Code

คุณสามารถตั้งค่า hook `PreToolUse` ของ Claude Code เพื่อให้คำสั่ง shell ทั้งหมดผ่าน RTK โดยอัตโนมัติ

```bash
# ติดตั้ง hook (เพิ่มอัตโนมัติใน settings.json ของ Claude Code)
wall-vault rtk hook install
```

หรือเพิ่มด้วยตนเองใน `~/.claude/settings.json`:

```json
{
  "hooks": {
    "PreToolUse": [{
      "matcher": "Bash",
      "command": "wall-vault rtk rewrite"
    }]
  }
}
```

> 💡 **รักษา exit code**: RTK คืน exit code ดั้งเดิมของคำสั่ง หากคำสั่งล้มเหลว (exit code ≠ 0) AI ก็จะตรวจจับความล้มเหลวได้อย่างถูกต้อง

> 💡 **บังคับเอาต์พุตภาษาอังกฤษ**: RTK รันคำสั่งด้วย `LC_ALL=C` เพื่อสร้างเอาต์พุตภาษาอังกฤษเสมอ โดยไม่คำนึงถึงการตั้งค่าภาษาของระบบ สิ่งนี้จำเป็นเพื่อให้ตัวกรองทำงานอย่างถูกต้อง

## ตัวแปรสภาพแวดล้อม

ตัวแปรสภาพแวดล้อมคือวิธีส่งค่าคอนฟิกไปยังโปรแกรม ป้อนในเทอร์มินัลในรูปแบบ `export ชื่อตัวแปร=ค่า` หรือใส่ในไฟล์บริการเริ่มต้นอัตโนมัติเพื่อการใช้งานถาวร

| ตัวแปร | คำอธิบาย | ค่าตัวอย่าง |
|------|------|---------|
| `WV_LANG` | ภาษาแดชบอร์ด | `ko`, `en`, `ja` |
| `WV_THEME` | ธีมแดชบอร์ด | `light`, `dark`, `gold` |
| `WV_KEY_GOOGLE` | คีย์ Google API (หลายคีย์คั่นด้วยจุลภาค) | `AIza...,AIza...` |
| `WV_KEY_OPENROUTER` | คีย์ OpenRouter API | `sk-or-v1-...` |
| `WV_VAULT_URL` | ที่อยู่เซิร์ฟเวอร์ตู้เซฟในโหมดกระจาย | `http://192.168.x.x:56243` |
| `WV_VAULT_TOKEN` | โทเค็นพิสูจน์ตัวตนไคลเอนต์ (บอท) | `my-secret-token` |
| `WV_ADMIN_TOKEN` | โทเค็นแอดมิน | `admin-token-here` |
| `WV_MASTER_PASS` | รหัสผ่านเข้ารหัสคีย์ API | `my-password` |
| `WV_AVATAR` | พาธไฟล์รูปอวาตาร์ (พาธสัมพัทธ์จาก `~/.openclaw/`) | `workspace/avatars/avatar.png` |
| `OLLAMA_URL` | ที่อยู่เซิร์ฟเวอร์ Ollama ในเครื่อง | `http://192.168.x.x:11434` |

## การแก้ไขปัญหา

### เมื่อพร็อกซีไม่เริ่ม

มักเป็นเพราะพอร์ตถูกใช้โดยโปรแกรมอื่นอยู่แล้ว

```bash
ss -tlnp | grep 56244   # ตรวจสอบว่าใครใช้พอร์ต 56244
wall-vault proxy --port 8080   # เริ่มด้วยหมายเลขพอร์ตอื่น
```

### เมื่อเกิดข้อผิดพลาดคีย์ API (429, 402, 401, 403, 582)

| รหัสข้อผิดพลาด | ความหมาย | การดำเนินการ |
|----------|------|----------|
| **429** | คำขอมากเกินไป (เกินขีดจำกัดการใช้งาน) | รอสักครู่หรือเพิ่มคีย์อื่น |
| **402** | ต้องชำระเงินหรือเครดิตไม่เพียงพอ | เติมเครดิตในบริการที่เกี่ยวข้อง |
| **401 / 403** | คีย์ผิดหรือไม่มีสิทธิ์ | ตรวจสอบค่าคีย์อีกครั้งแล้วลงทะเบียนใหม่ |
| **582** | เกตเวย์โอเวอร์โหลด (cooldown 5 นาที) | จะปลดล็อกอัตโนมัติหลังจาก 5 นาที |

```bash
# ตรวจสอบรายการและสถานะของคีย์ที่ลงทะเบียน
curl -H "Authorization: Bearer โทเค็น-แอดมิน" http://localhost:56243/admin/keys

# รีเซ็ตตัวนับการใช้งานคีย์
curl -X POST -H "Authorization: Bearer โทเค็น-แอดมิน" http://localhost:56243/admin/keys/reset
```

### เมื่อเอเจนต์แสดงว่า "ยังไม่เชื่อมต่อ"

"ยังไม่เชื่อมต่อ" หมายความว่ากระบวนการพร็อกซีไม่ได้ส่งสัญญาณ (heartbeat) ไปยังตู้เซฟ **ไม่ได้หมายความว่าการตั้งค่าไม่ได้ถูกบันทึก** พร็อกซีต้องทำงานพร้อมที่อยู่และโทเค็นของเซิร์ฟเวอร์ตู้เซฟจึงจะเปลี่ยนเป็นสถานะเชื่อมต่อ

```bash
# เริ่มพร็อกซีโดยระบุที่อยู่เซิร์ฟเวอร์ตู้เซฟ โทเค็น และ ID ไคลเอนต์
WV_VAULT_URL=http://ที่อยู่-เซิร์ฟเวอร์-ตู้เซฟ:56243 \
WV_VAULT_TOKEN=โทเค็น-ไคลเอนต์ \
WV_VAULT_CLIENT_ID=ID-ไคลเอนต์ \
wall-vault proxy
```

หากเชื่อมต่อสำเร็จ สถานะจะเปลี่ยนเป็น 🟢 กำลังทำงาน บนแดชบอร์ดภายในประมาณ 20 วินาที

### เมื่อการเชื่อมต่อ Ollama ไม่ทำงาน

Ollama คือโปรแกรมที่รัน AI บนคอมพิวเตอร์ของคุณโดยตรง ก่อนอื่น ตรวจสอบว่า Ollama ทำงานอยู่

```bash
curl http://localhost:11434/api/tags   # หากรายการโมเดลแสดง แสดงว่าปกติ
export OLLAMA_URL=http://192.168.x.x:11434   # หากทำงานบนคอมพิวเตอร์อื่น
```

> ⚠️ หาก Ollama ไม่ตอบสนอง ให้เริ่มก่อนด้วยคำสั่ง `ollama serve`

> ⚠️ **โมเดลใหญ่ตอบสนองช้า**: โมเดลใหญ่เช่น `qwen3.5:35b`, `deepseek-r1` อาจใช้เวลาหลายนาทีในการสร้างการตอบสนอง แม้ดูเหมือนไม่มีการตอบสนอง อาจกำลังประมวลผลตามปกติ — โปรดรอ

### v0.1.24 (2026-04-06)

### v0.1.23 (2026-04-06)

### v0.1.22 (2026-04-05)
