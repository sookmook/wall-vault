## wall-vault гэж юу вэ?

**wall-vault = OpenClaw-д зориулсан AI зуучлагч (Proxy) + API түлхүүрийн сан**

AI үйлчилгээг ашиглахын тулд **API түлхүүр** хэрэгтэй. API түлхүүр гэдэг нь "энэ хүн энэ үйлчилгээг ашиглах эрхтэй" гэдгийг баталдаг **дижитал нэвтрэх карт** юм. Гэвч энэ нэвтрэх карт нь өдөр тутмын хэрэглээний хязгаартай, зөв удирдахгүй бол задруулах эрсдэлтэй.

wall-vault эдгээр нэвтрэх картуудыг аюулгүй санд хадгалж, OpenClaw болон AI үйлчилгээний хооронд **зуучлагч (proxy)** үүрэг гүйцэтгэдэг. Товчхондоо, OpenClaw нь зөвхөн wall-vault-тай холбогдоход л хангалттай, бусад нарийн ажлуудыг wall-vault өөрөө зохицуулна.

wall-vault шийддэг асуудлууд:

- **API түлхүүрийг автоматаар сэлгэх**: Нэг түлхүүрийн хэрэглээ хязгаарт хүрсэн эсвэл түр хориглогдсон (cooldown) тохиолдолд чимээгүйхэн дараагийн түлхүүр рүү шилждэг. OpenClaw тасалдалгүй үргэлжлэн ажиллана.
- **Үйлчилгээг автоматаар солих (Fallback)**: Google хариу өгөхгүй бол OpenRouter руу, энэ нь ч ажиллахгүй бол таны компьютерт суулгасан Ollama/LM Studio/vLLM (дотоод AI) руу автоматаар шилждэг. Сессион тасдахгүй. Анхны үйлчилгээ сэргэхэд дараагийн хүсэлтээс автоматаар буцаж ирдэг (v0.1.18+, LM Studio/vLLM: v0.1.21+).
- **Бодит цагийн синхрончлол (SSE)**: Түлхүүрийн сангийн хяналтын самбар дээр загвар өөрчлөхөд 1-3 секундын дотор OpenClaw дэлгэцэд тусгагддаг. SSE (Server-Sent Events) гэдэг нь серверээс өөрчлөлтүүдийг бодит цагт клиентүүд рүү илгээдэг технологи юм.
- **Бодит цагийн мэдэгдэл**: Түлхүүр дуусах эсвэл үйлчилгээ амжилтгүй болох зэрэг үйл явдлууд OpenClaw TUI (терминал дэлгэц)-ийн доод талд шууд харагддаг.

> 💡 **Claude Code, Cursor, VS Code**-г бас холбож болно, гэхдээ wall-vault-ийн анхны зорилго нь OpenClaw-тай хамт ашиглах явдал юм.

```
OpenClaw (TUI терминал дэлгэц)
        │
        ▼
  wall-vault proxy (:56244)   ← түлхүүр удирдлага, чиглүүлэлт, fallback, үйл явдлууд
        │
        ├─ Google Gemini API
        ├─ OpenRouter API (340+ загвар)
        ├─ Ollama / LM Studio / vLLM (таны компьютер, сүүлчийн хамгаалалт)
        └─ OpenAI / Anthropic API
```

## Суулгах

### Linux / macOS

Терминал нээгээд доорх тушаалуудыг буулгаарай.

```bash
# Linux (ердийн PC, сервер — amd64)
curl -L https://github.com/sookmook/wall-vault/releases/latest/download/wall-vault-linux-amd64 \
  -o ~/.local/bin/wall-vault && chmod +x ~/.local/bin/wall-vault

# macOS Apple Silicon (M1/M2/M3 Mac)
curl -L https://github.com/sookmook/wall-vault/releases/latest/download/wall-vault-darwin-arm64 \
  -o /usr/local/bin/wall-vault && chmod +x /usr/local/bin/wall-vault
```

- `curl -L ...` — Интернетээс файл татаж авна.
- `chmod +x` — Татаж авсан файлыг "ажиллуулах боломжтой" болгоно. Энэ алхмыг алгасвал "зөвшөөрөл байхгүй" алдаа гарна.

### Windows

PowerShell (администраторын эрхтэй) нээж доорх тушаалуудыг ажиллуулна.

```powershell
# Татах
Invoke-WebRequest -Uri `
  "https://github.com/sookmook/wall-vault/releases/latest/download/wall-vault-windows-amd64.exe" `
  -OutFile "$env:LOCALAPPDATA\Programs\wall-vault\wall-vault.exe"

# PATH-д нэмэх (PowerShell дахин эхлүүлсний дараа хүчин төгөлдөр болно)
$env:PATH += ";$env:LOCALAPPDATA\Programs\wall-vault"
```

> 💡 **PATH гэж юу вэ?** Компьютерийн тушаалуудыг хайдаг хавтасуудын жагсаалт юм. PATH-д нэмсэнээр ямар ч хавтаснаас `wall-vault` гэж бичээд ажиллуулах боломжтой болно.

### Эх кодоос бүтээх (хөгжүүлэгчдэд зориулсан)

Go хэлний хөгжүүлэлтийн орчин суулгасан тохиолдолд л хамаарна.

```bash
git clone https://github.com/sookmook/wall-vault
cd wall-vault
make build       # bin/wall-vault (хувилбар: v0.1.25.YYYYMMDD.HHmmss)
make install     # ~/.local/bin/wall-vault
```

> 💡 **Бүтээх цагийн тэмдэг хувилбар**: `make build`-ээр бүтээхэд хувилбар нь `v0.1.25.20260408.022325` гэх мэт огноо, цагийг агуулсан форматаар автоматаар үүсдэг. `go build ./...`-ээр шууд бүтээвэл хувилбар зөвхөн `"dev"` гэж харагдана.

## Эхний тохиргоо

### Setup шидтэн ажиллуулах

Суулгасны дараа доорх тушаалаар **тохиргооны шидтэн** ажиллуулахаа мартваарай. Шидтэн шаардлагатай зүйлсийг нэг нэгээр нь асууж зааварчилна.

```bash
wall-vault setup
```

Шидтэнгийн дагах алхмууд:

```
1. Хэл сонгох (Монгол зэрэг 10 хэл)
2. Загвар сонгох (light / dark / gold / cherry / ocean)
3. Ажиллах горим — ганцаараа (standalone) эсвэл олон машин дээр (distributed) сонгох
4. Ботын нэр оруулах — хяналтын самбарт харагдах нэр
5. Порт тохиргоо — анхдагч: proxy 56244, сан 56243 (өөрчлөх шаардлагагүй бол Enter дарна)
6. AI үйлчилгээ сонгох — Google / OpenRouter / Ollama / LM Studio / vLLM
7. Хэрэгслийн аюулгүй байдлын шүүлтүүрийн тохиргоо
8. Админ токен тохируулах — хяналтын самбарын удирдлагын функцуудыг түгжих нууц үг. Автоматаар үүсгэх боломжтой
9. API түлхүүрийн шифрлэлтийн нууц үг тохируулах — түлхүүрүүдийг илүү аюулгүй хадгалах (сонголт)
10. Тохиргооны файлын хадгалах зам
```

> ⚠️ **Админ токеноо заавал санаарай.** Дараа нь хяналтын самбарт түлхүүр нэмэх эсвэл тохиргоо өөрчлөхөд хэрэг болно. Мартвал тохиргооны файлыг шууд засварлах хэрэгтэй.

Шидтэн дууссаны дараа `wall-vault.yaml` тохиргооны файл автоматаар үүсдэг.

### Ажиллуулах

```bash
wall-vault start
```

Хоёр сервер нэгэн зэрэг эхэлнэ:

- **Прокси** (`http://localhost:56244`) — OpenClaw болон AI үйлчилгээг холбох зуучлагч
- **Түлхүүрийн сан** (`http://localhost:56243`) — API түлхүүрийн удирдлага болон вэб хяналтын самбар

Хөтөч дээр `http://localhost:56243` нээвэл хяналтын самбарыг шууд харах боломжтой.

## API түлхүүр бүртгэх

API түлхүүр бүртгэх дөрвөн арга бий. **Шинээр эхлэж буй хүмүүст Арга 1 (орчны хувьсагч) санал болгож байна**.

### Арга 1: Орчны хувьсагч (Санал болгох — Хамгийн энгийн)

Орчны хувьсагч гэдэг нь програм эхлэхэд уншдаг **урьдчилан тохируулсан утга** юм. Терминалд дараах байдлаар бичнэ.

```bash
# Google Gemini түлхүүр бүртгэх
export WV_KEY_GOOGLE=AIzaSy...

# OpenRouter түлхүүр бүртгэх
export WV_KEY_OPENROUTER=sk-or-v1-...

# Бүртгэлийн дараа ажиллуулах
wall-vault start
```

Олон түлхүүртэй бол таслалаар (,) холбоно. wall-vault түлхүүрүүдийг ээлжлэн автоматаар ашиглана (round robin):

```bash
export WV_KEY_GOOGLE=AIzaSy...,AIzaSy...,AIzaSy...
```

> 💡 **Зөвлөгөө**: `export` тушаал зөвхөн одоогийн терминал сессэнд хүчинтэй. Компьютер дахин асаасны дараа ч хүчинтэй байхыг хүсвэл `~/.bashrc` эсвэл `~/.zshrc` файлд дээрх мөрийг нэмээрэй.

### Арга 2: Хяналтын самбарын UI (хулганаар дарах)

1. Хөтөч дээр `http://localhost:56243` руу орно
2. Дээд талын **🔑 API түлхүүр** картан дээрх `[+ Нэмэх]` товчийг дарна
3. Үйлчилгээний төрөл, түлхүүрийн утга, шошго (санамж нэр), өдрийн хязгаарыг оруулаад хадгална

### Арга 3: REST API (автоматжуулалт/скриптэд зориулсан)

REST API гэдэг нь програмууд хоорондоо HTTP-ээр өгөгдөл солилцдог арга юм. Скриптээр автоматаар бүртгэхэд тохиромжтой.

```bash
curl -X POST http://localhost:56243/admin/keys \
  -H "Authorization: Bearer админ-токен" \
  -H "Content-Type: application/json" \
  -d '{
    "service": "google",
    "key": "AIzaSy...",
    "label": "Үндсэн түлхүүр",
    "daily_limit": 1000
  }'
```

### Арга 4: Proxy флаг (богино хугацааны туршилтад)

Албан ёсны бүртгэлгүйгээр түр зуур түлхүүр оруулж туршихад ашиглана. Програм унтраахад алга болно.

```bash
wall-vault proxy --key-google=AIzaSy... --key-openrouter=sk-or-...
```

## Прокси ашиглах

### OpenClaw дээр ашиглах (Үндсэн зорилго)

OpenClaw-г wall-vault-аар дамжуулан AI үйлчилгээнд холбох тохиргоо хийх арга.

`~/.openclaw/openclaw.json` файлыг нээж доорх агуулгыг нэмнэ:

```json5
// ~/.openclaw/openclaw.json
{
  models: {
    providers: {
      "wall-vault": {
        baseUrl: "http://localhost:56244/v1",
        apiKey: "your-agent-token",   // vault агент токен
        api: "openai-completions",
        models: [
          { id: "wall-vault/gemini-2.5-flash" },
          { id: "wall-vault/gemini-2.5-pro" },
          { id: "wall-vault/hunter-alpha" },    // үнэгүй 1M context
          { id: "wall-vault/claude-opus-4-6" }
        ]
      }
    }
  }
}
```

> 💡 **Илүү хялбар арга**: Хяналтын самбарын агент картан дээрх **🦞 OpenClaw тохиргоо хуулах** товчийг дарахад токен болон хаяг аль хэдийн бөглөгдсөн хэсэг clipboard-д хуулагдана. Зүгээр л буулгахад л хангалттай.

**Загварын нэрний өмнөх `wall-vault/` хаашаа холбогдох вэ?**

wall-vault нь загварын нэрний дагуу аль AI үйлчилгээ рүү хүсэлт илгээхийг автоматаар шийддэг:

| Загварын формат | Холбогдох үйлчилгээ |
|----------|--------------|
| `wall-vault/gemini-*` | Google Gemini руу шууд холбогдох |
| `wall-vault/gpt-*`, `wall-vault/o3`, `wall-vault/o4*` | OpenAI руу шууд холбогдох |
| `wall-vault/claude-*` | OpenRouter-ээр дамжуулан Anthropic холбогдох |
| `wall-vault/hunter-alpha`, `wall-vault/healer-alpha` | OpenRouter (үнэгүй 1 сая токен context) |
| `wall-vault/kimi-*`, `wall-vault/glm-*`, `wall-vault/deepseek-*` | OpenRouter холбогдох |
| `google/загварын-нэр`, `openai/загварын-нэр`, `anthropic/загварын-нэр` гм. | Тохирох үйлчилгээ рүү шууд холбогдох |
| `custom/google/загварын-нэр`, `custom/openai/загварын-нэр` гм. | `custom/` хэсгийг хасаж дахин чиглүүлэх |
| `загварын-нэр:cloud` | `:cloud` хэсгийг хасаж OpenRouter руу холбогдох |

> 💡 **Context гэж юу вэ?** AI нэг удаад санах чадах харилцан яриаын хэмжээ юм. 1M (1 сая токен) гэдэг нь маш урт харилцан яриа эсвэл урт бичиг баримтыг нэг удаад боловсруулж чадна гэсэн үг.

### Gemini API форматаар шууд холбогдох (одоо байгаа хэрэгслүүдтэй нийцтэй)

Google Gemini API-г шууд ашиглаж байсан хэрэгслүүдтэй бол хаягийг зөвхөн wall-vault руу солино:

```bash
export ANTHROPIC_BASE_URL=http://localhost:56244/google
```

Эсвэл URL-г шууд зааж өгдөг хэрэгслийн хувьд:

```
http://localhost:56244/google/v1beta/models/gemini-2.5-flash:generateContent
```

### OpenAI SDK (Python) дээр ашиглах

Python дээр AI ашигладаг кодонд ч wall-vault-г холбож болно. Зөвхөн `base_url`-г солиход л хангалттай:

```python
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:56244/v1",
    api_key="not-needed"  # API түлхүүрийг wall-vault автоматаар удирдана
)

response = client.chat.completions.create(
    model="google/gemini-2.5-flash",   # нийлүүлэгч/загвар формат
    messages=[{"role": "user", "content": "Сайн байна уу"}]
)
```

### Ажиллаж байх үед загвар солих

wall-vault аль хэдийн ажиллаж байх үед AI загвар солихын тулд:

```bash
# Прокси руу шууд хүсэлт илгээж загвар солих
curl -X PUT http://localhost:56244/api/config/model \
  -H "Content-Type: application/json" \
  -d '{"service": "openrouter", "model": "anthropic/claude-3.5-sonnet"}'

# Тархсан горимд (олон бот) сангийн серверт өөрчлөх → SSE-ээр шууд тусгагдана
curl -X PUT http://localhost:56243/admin/clients/миний-бот-id \
  -H "Authorization: Bearer админ-токен" \
  -H "Content-Type: application/json" \
  -d '{"default_service": "google", "default_model": "gemini-2.5-pro"}'
```

### Боломжит загваруудын жагсаалт шалгах

```bash
# Бүх жагсаалт харах
curl http://localhost:56244/api/models | python3 -m json.tool

# Зөвхөн Google загваруудыг харах
curl "http://localhost:56244/api/models?service=google"

# Нэрээр хайх (жнь: "claude" агуулсан загварууд)
curl "http://localhost:56244/api/models?q=claude"
```

**Үйлчилгээ тус бүрийн гол загваруудын хураангуй:**

| Үйлчилгээ | Гол загварууд |
|--------|----------|
| Google | gemini-2.5-pro, gemini-2.5-flash, gemini-2.5-flash-8b, gemini-2.0-flash |
| OpenAI | gpt-4o, gpt-4o-mini, o3, o1, o1-mini |
| OpenRouter | 346+ (Hunter Alpha 1M context үнэгүй, DeepSeek R1/V3, Qwen 2.5 гм.) |
| Ollama | Таны компьютерт суулгасан дотоод серверийг автоматаар илрүүлнэ |
| LM Studio | Таны компьютерын дотоод сервер (порт 1234) |
| vLLM | Таны компьютерын дотоод сервер (порт 8000) |

## Түлхүүрийн санги хяналтын самбар

Хөтөч дээр `http://localhost:56243` руу орвол хяналтын самбарыг харж болно.

**Дэлгэцийн бүтэц:**
- **Дээд тогтмол мөр (topbar)**: Лого, хэл ба загвар сонголт, SSE холболтын төлөв
- **Картуудын тор**: Агент, үйлчилгээ, API түлхүүрийн картууд хавтан хэлбэрээр байрласан

### API түлхүүрийн карт

Бүртгэгдсэн API түлхүүрүүдийг нэг харцаар удирдах карт.

- Үйлчилгээ тус бүрээр ялган түлхүүрийн жагсаалтыг харуулна.
- `today_usage`: Өнөөдөр амжилттай боловсруулагдсан токеноор (AI-н уншсан, бичсэн тэмдэгтүүдийн тоо) тоо
- `today_attempts`: Өнөөдрийн нийт дуудлагын тоо (амжилттай + амжилтгүй)
- `[+ Нэмэх]` товчоор шинэ түлхүүр бүртгэх, `✕`-ээр түлхүүр устгах.

> 💡 **Токен гэж юу вэ?** AI текст боловсруулахад ашигладаг нэгж юм. Ойролцоогоор нэг англи үг, эсвэл 1-2 солонгос тэмдэгтэд тэнцэнэ. API-н төлбөр ихэвчлэн энэ токеноор тоогоор тооцогддог.

### Агентын карт

wall-vault прокситой холбогдсон ботуудын (агентуудын) төлвийг харуулах карт.

**Холболтын төлвийг 4 түвшинд харуулна:**

| Үзүүлэлт | Төлөв | Утга |
|------|------|------|
| 🟢 | Ажиллаж байна | Прокси хэвийн ажиллаж байна |
| 🟡 | Удааширсан | Хариу ирж байгаа боловч удаан |
| 🔴 | Офлайн | Прокси хариу өгөхгүй байна |
| ⚫ | Холбогдоогүй/Идэвхгүй | Прокси сан руу хэзээ ч холбогдоогүй эсвэл идэвхгүй |

**Агентын картны доод хэсгийн товчлууруудын зааварчилгаа:**

Агент бүртгэхдээ **агентын төрөл** зааж өгвөл, тухайн төрөлд зориулсан тусламжийн товчлуурууд автоматаар гарч ирнэ.

#### 🔘 Тохиргоо хуулах товчлуур — Холболтын тохиргоог автоматаар үүсгэнэ

Товчлуур дарахад тухайн агентын токен, прокси хаяг, загварын мэдээлэл аль хэдийн бөглөгдсөн тохиргооны хэсэг clipboard руу хуулагдана. Хуулагдсан агуулгыг доорх хүснэгтэд заасан газарт буулгахад л холболтын тохиргоо дуусна.

| Товчлуур | Агентын төрөл | Буулгах газар |
|------|-------------|-------------|
| 🦞 OpenClaw тохиргоо хуулах | `openclaw` | `~/.openclaw/openclaw.json` |
| 🦀 NanoClaw тохиргоо хуулах | `nanoclaw` | `~/.openclaw/openclaw.json` |
| 🟠 Claude Code тохиргоо хуулах | `claude-code` | `~/.claude/settings.json` |
| ⌨ Cursor тохиргоо хуулах | `cursor` | Cursor → Settings → AI |
| 💻 VSCode тохиргоо хуулах | `vscode` | `~/.continue/config.json` |

**Жишээ — Claude Code төрөл бол дараах агуулга хуулагдана:**

```json
// ~/.claude/settings.json
{
  "apiProvider": "openai",
  "baseUrl": "http://192.168.0.6:56244/v1",
  "apiKey": "энэ-агентын-токен"
}
```

**Жишээ — VSCode (Continue) төрөл бол:**

```yaml
# ~/.continue/config.yaml  ← config.json биш config.yaml-д буулгана
name: My Config
version: 0.0.1
schema: v1

models:
  - name: wall-vault proxy
    provider: openai
    model: gemini-2.5-flash
    apiBase: http://192.168.0.6:56244/v1
    apiKey: энэ-агентын-токен
    roles:
      - chat
      - edit
      - apply
```

> ⚠️ **Continue-ийн сүүлийн хувилбар `config.yaml` ашигладаг.** `config.yaml` байвал `config.json`-г бүрэн үл тоомсорлодог. Заавал `config.yaml`-д буулгаарай.

**Жишээ — Cursor төрөл бол:**

```
Base URL : http://192.168.0.6:56244/v1
API Key  : энэ-агентын-токен

// Эсвэл орчны хувьсагч:
OPENAI_BASE_URL=http://192.168.0.6:56244/v1
OPENAI_API_KEY=энэ-агентын-токен
```

> ⚠️ **Clipboard-д хуулах ажиллахгүй бол**: Хөтчийн аюулгүй байдлын бодлого хуулахыг хориглож болно. Текст талбар popup хэлбэрээр нээгдвэл Ctrl+A-аар бүгдийг сонгоод Ctrl+C-ээр хуулна.

#### ⚡ Автомат хэрэглэх товчлуур — Нэг удаа дарахад тохиргоо дуусна

Агентын төрөл `cline`, `claude-code`, `openclaw` эсвэл `nanoclaw` бол агентын картанд **⚡ Тохиргоо хэрэглэх** товчлуур гарч ирнэ. Энэ товчлуурыг дарахад тухайн агентын дотоод тохиргооны файл автоматаар шинэчлэгдэнэ.

| Товчлуур | Агентын төрөл | Зорилтот файл |
|------|-------------|-------------|
| ⚡ Cline тохиргоо хэрэглэх | `cline` | `~/.cline/data/globalState.json` + `secrets.json` |
| ⚡ Claude Code тохиргоо хэрэглэх | `claude-code` | `~/.claude/settings.json` |
| ⚡ OpenClaw тохиргоо хэрэглэх | `openclaw` | `~/.openclaw/openclaw.json` |
| ⚡ NanoClaw тохиргоо хэрэглэх | `nanoclaw` | `~/.openclaw/openclaw.json` |

> ⚠️ Энэ товчлуур нь **localhost:56244** (дотоод прокси) руу хүсэлт илгээнэ. Тухайн машин дээр прокси ажиллаж байх шаардлагатай.

#### 🔄 Хоёр чиглэлтэй загварын синхрончлол (v0.1.16)

Түлхүүрийн сангийн хяналтын самбар дээр агентын загварыг өөрчлөхөд тухайн агентын дотоод тохиргоо автоматаар шинэчлэгдэнэ.

**Cline-ийн хувьд:**
- Санд загвар өөрчлөхөд → SSE үйл явдал → Прокси `globalState.json`-ийн загвар талбарыг шинэчилнэ
- Шинэчлэх зорилтууд: `actModeOpenAiModelId`, `planModeOpenAiModelId`, `openAiModelId`
- `openAiBaseUrl` болон API түлхүүрт хүрэхгүй
- **VS Code дахин ачаалах (`Ctrl+Alt+R` эсвэл `Ctrl+Shift+P` → `Developer: Reload Window`) шаардлагатай**
  - Cline ажиллаж байх үедээ тохиргооны файлыг дахин унших уулгүй

**Claude Code-ийн хувьд:**
- Санд загвар өөрчлөхөд → SSE үйл явдал → Прокси `settings.json`-ийн `model` талбарыг шинэчилнэ
- WSL болон Windows хоёр замыг автоматаар хайна (`~/.claude/`, `/mnt/c/Users/*/.claude/`)

**Эсрэг чиглэл (агент → сан):**
- Агент (Cline, Claude Code гм.) прокси руу хүсэлт илгээхэд, прокси heartbeat-д тухайн клиентийн үйлчилгээ/загварын мэдээллийг оруулдаг
- Түлхүүрийн сангийн хяналтын самбарын агентын картанд одоо ашиглаж буй үйлчилгээ/загвар бодит цагт харагддаг

> 💡 **Гол зүйл**: Прокси нь хүсэлтийн Authorization токеноор агентыг таньж, санд тохируулсан үйлчилгээ/загвар руу автоматаар чиглүүлдэг. Cline эсвэл Claude Code өөр загварын нэр илгээсэн ч прокси сангийн тохиргоогоор дарж бичнэ.

### VS Code дээр Cline ашиглах — Дэлгэрэнгүй зааварчилгаа

#### Алхам 1: Cline суулгах

VS Code extension marketplace-ээс **Cline** (ID: `saoudrizwan.claude-dev`) суулгана.

#### Алхам 2: Санд агент бүртгэх

1. Түлхүүрийн сангийн хяналтын самбар (`http://сангийн-IP:56243`) нээнэ
2. **Агент** хэсэгт **+ Нэмэх** дарна
3. Дараах байдлаар оруулна:

| Талбар | Утга | Тайлбар |
|------|----|------|
| ID | `miin_cline` | Давтагдашгүй тэмдэглэгч (англиар, зайгүй) |
| Нэр | `Миний Cline` | Хяналтын самбарт харагдах нэр |
| Агентын төрөл | `cline` | ← Заавал `cline` сонгох |
| Үйлчилгээ | Ашиглах үйлчилгээг сонгоно (жнь: `google`) | |
| Загвар | Ашиглах загварыг оруулна (жнь: `gemini-2.5-flash`) | |

4. **Хадгалах** дарахад токен автоматаар үүснэ

#### Алхам 3: Cline-тай холбогдох

**Арга A — Автомат хэрэглэх (Санал болгох)**

1. Тухайн машин дээр wall-vault **прокси** ажиллаж байгаа эсэхийг шалгана (`localhost:56244`)
2. Хяналтын самбарын агентын картан дээрх **⚡ Cline тохиргоо хэрэглэх** товчлуурыг дарна
3. "Тохиргоо хэрэглэгдлээ!" мэдэгдэл гарвал амжилттай
4. VS Code дахин ачаална (`Ctrl+Alt+R`)

**Арга B — Гар аргаар тохируулах**

Cline-ийн хажуу мөрний тохиргоо (⚙️) нээж:
- **API Provider**: `OpenAI Compatible`
- **Base URL**: `http://проксийн-хаяг:56244/v1`
  - Ижил машин дээр: `http://localhost:56244/v1`
  - Өөр машин дээр жнь Mini сервер: `http://192.168.0.6:56244/v1`
- **API Key**: Сангаас олгосон токен (агентын картнаас хуулна)
- **Model ID**: Санд тохируулсан загвар (жнь: `gemini-2.5-flash`)

#### Алхам 4: Баталгаажуулах

Cline чатад ямар нэг мессеж илгээнэ. Хэвийн бол:
- Түлхүүрийн сангийн хяналтын самбарын агентын картанд **ногоон цэг (● Ажиллаж байна)** харагдана
- Картанд одоогийн үйлчилгээ/загвар харагдана (жнь: `google / gemini-2.5-flash`)

#### Загвар өөрчлөх

Cline-ийн загвар солихыг хүсвэл **түлхүүрийн сангийн хяналтын самбар** дээрээс өөрчилнө:

1. Агентын картын үйлчилгээ/загвар сонголтыг өөрчилнө
2. **Хэрэглэх** дарна
3. VS Code дахин ачаална (`Ctrl+Alt+R`) — Cline footer-ийн загварын нэр шинэчлэгдэнэ
4. Дараагийн хүсэлтээс шинэ загвар ашиглагдана

> 💡 Бодит байдалд прокси Cline-ийн хүсэлтийг токеноор таньж сангийн тохиргооны загвар руу чиглүүлдэг. VS Code дахин ачаалахгүй ч **бодитоор ашиглагдаж буй загвар шууд өөрчлөгдөнө** — дахин ачаалах нь Cline UI дээрх загварын нэрийг шинэчлэхийн л тулд юм.

#### Холболт тасрах илрүүлэлт

VS Code хаахад түлхүүрийн сангийн хяналтын самбарын агентын карт ойролцоогоор **90 секунд** дараа шар (удааширсан), **3 минут** дараа улаан (офлайн) болж хувирна. (v0.1.18-аас 15 секунд тутмын төлвийн шалгалтаар офлайн илрүүлэлт хурдассан.)

#### Алдаа засах

| Шинж тэмдэг | Шалтгаан | Шийдэл |
|------|------|------|

### Үйлчилгээний карт

Ашиглах AI үйлчилгээг асааж/унтрааж эсвэл тохируулдаг карт.

- Үйлчилгээ бүрт идэвхжүүлэх/идэвхгүй болгох шилжүүлэгч
- Дотоод AI серверийн (таны компьютер дээрх Ollama, LM Studio, vLLM гм.) хаягийг оруулахад боломжит загваруудыг автоматаар олдог.
- **Дотоод үйлчилгээний холболтын төлвийн үзүүлэлт**: Үйлчилгээний нэрний хажууд ● цэг **ногоон** бол холбогдсон, **саарал** бол холбогдоогүй
- **Дотоод үйлчилгээний автомат гэрлэн дохио** (v0.1.23+): Дотоод үйлчилгээнүүд (Ollama, LM Studio, vLLM) холбогдох боломжтой бол автоматаар идэвхждэг, тасрахад автоматаар идэвхгүй болдог. Үйлчилгээг асаахад 15 секундын дотор ● ногоон болж чекбокс асдаг, унтраахад автоматаар унтрадаг. Үүлэн үйлчилгээнүүд (Google, OpenRouter гм.) API түлхүүр байгаа эсэхээс хамааран автоматаар шилжих аргатай ижил.

> 💡 **Дотоод үйлчилгээ өөр компьютер дээр ажиллаж байвал**: Үйлчилгээний URL оролтын талбарт тухайн компьютерийн IP-г оруулна. Жнь: `http://192.168.0.6:11434` (Ollama), `http://192.168.0.6:1234` (LM Studio). Үйлчилгээ `0.0.0.0` биш `127.0.0.1`-д зөвхөн холбогдсон бол гадна IP-ээр хандах боломжгүй тул үйлчилгээний тохиргоонд холбох хаягийг шалгаарай.

### Админ токен оруулах

Хяналтын самбарт түлхүүр нэмэх/устгах зэрэг чухал функцуудыг ашиглахыг оролдоход админ токен оруулах popup гарч ирнэ. Setup шидтэнд тохируулсан токенийг оруулна. Нэг оруулахад хөтөчийг хаах хүртэл хадгалагдана.

> ⚠️ **15 минутын дотор баталгаажуулалт 10-аас дээш удаа амжилтгүй болвол тухайн IP түр хаагдана.** Токеноо мартсан бол `wall-vault.yaml` файлын `admin_token` хэсгийг шалгаарай.

## Тархсан горим (олон бот)

Олон компьютер дээр нэгэн зэрэг OpenClaw ажиллуулахад **нэг түлхүүрийн санг хуваалцах** бүтэц. Түлхүүр удирдлагыг нэг газраас л хийхэд л хангалттай тул тохиромжтой.

### Бүтцийн жишээ

```
[Түлхүүрийн сангийн сервер]
  wall-vault vault    (түлхүүрийн сан :56243, хяналтын самбар)

[WSL Альфа]            [Raspberry Pi Гамма]    [Mac Mini Дотоод]
  wall-vault proxy      wall-vault proxy        wall-vault proxy
  openclaw TUI          openclaw TUI            openclaw TUI
  ↕ SSE синхрончлол     ↕ SSE синхрончлол       ↕ SSE синхрончлол
```

Бүх ботууд дунд байгаа сангийн сервер рүү хандаж байгаа тул санд загвар өөрчлөх эсвэл түлхүүр нэмэхэд бүх ботод шууд тусгагдана.

### Алхам 1: Түлхүүрийн сангийн сервер эхлүүлэх

Сангийн серверийн үүрэг гүйцэтгэх компьютер дээр ажиллуулна:

```bash
wall-vault vault
```

### Алхам 2: Бот бүрийг (клиент) бүртгэх

Сангийн серверт холбогдох бот бүрийн мэдээллийг урьдчилан бүртгэнэ:

```bash
curl -X POST http://localhost:56243/admin/clients \
  -H "Authorization: Bearer админ-токен" \
  -H "Content-Type: application/json" \
  -d '{
    "id": "botA",
    "name": "БотA",
    "token": "bota-secret",
    "default_service": "google",
    "default_model": "gemini-2.5-flash"
  }'
```

### Алхам 3: Бот бүрийн компьютер дээр прокси эхлүүлэх

Бот суулгасан компьютер бүр дээр сангийн серверийн хаяг болон токенийг зааж прокси ажиллуулна:

```bash
WV_VAULT_URL=http://192.168.x.x:56243 \
WV_VAULT_TOKEN=bota-secret \
WV_VAULT_CLIENT_ID=botA \
wall-vault proxy
```

> 💡 **`192.168.x.x`** хэсгийг сангийн серверийн компьютерийн бодит дотоод IP хаягаар солино. Router тохиргоо эсвэл `ip addr` тушаалаар шалгаж болно.

## RTK токен хэмнэлт

*(v0.1.24+)*

**RTK (Токен Хэмнэлтийн Хэрэгсэл)** нь AI кодчилолын агент (Claude Code гм.)-ийн ажиллуулдаг shell тушаалуудын гаралтыг автоматаар шахаж токен хэрэглээг бууруулдаг. Жишээ нь, `git status`-ийн 15 мөр гаралт 2 мөр хураангуй болж буурна.

### Үндсэн хэрэглээ

```bash
# Тушаалыг wall-vault rtk-аар ороох ба гаралт автоматаар шүүгдэнэ
wall-vault rtk git status          # Зөвхөн өөрчлөгдсөн файлуудын жагсаалт
wall-vault rtk git diff HEAD~1     # Өөрчлөгдсөн мөрүүд + хамгийн бага context
wall-vault rtk git log -10         # Hash + нэг мөрийн мессеж
wall-vault rtk go test ./...       # Зөвхөн амжилтгүй тестүүд
wall-vault rtk ls -la              # Дэмжигдээгүй тушаалууд автоматаар хасагдана
```

### Дэмжигдсэн тушаалууд ба хэмнэлтийн нөлөө

| Тушаал | Шүүлтүүрийн арга | Хэмнэлтийн хувь |
|------|----------|--------|
| `git status` | Зөвхөн өөрчлөгдсөн файлуудын хураангуй | ~87% |
| `git diff` | Өөрчлөгдсөн мөрүүд + 3 мөрийн context | ~60-94% |
| `git log` | Hash + эхний мөрийн мессеж | ~90% |
| `git push/pull/fetch` | Явцыг хасах, зөвхөн хураангуй | ~80% |
| `go test` | Зөвхөн амжилтгүй харуулах, амжилттайг тоолох | ~88-99% |
| `go build/vet` | Зөвхөн алдааг харуулах | ~90% |
| Бусад бүх тушаалууд | Эхний 50 + сүүлийн 50 мөр, дээд хэмжээ 32KB | Хувьсах |

### 3 алхамт шүүлтүүрийн дамжуулга

1. **Тушаал тус бүрийн бүтцийн шүүлтүүр** — git, go гм.-ийн гаралтын форматыг ойлгож утга учиртай хэсгийг л гаргана
2. **Regex дараа боловсруулалт** — ANSI өнгөний кодуудыг хасах, хоосон мөрүүдийг шахах, давтагдсан мөрүүдийг нэгтгэх
3. **Passthrough + хасалт** — Дэмжигдээгүй тушаалуудад зөвхөн эхний/сүүлийн 50 мөрийг хадгална

### Claude Code-тай холболт

Claude Code-ийн `PreToolUse` hook-оор бүх shell тушаалуудыг автоматаар RTK-аар дамжуулахаар тохируулж болно.

```bash
# Hook суулгах (Claude Code settings.json-д автоматаар нэмэгдэнэ)
wall-vault rtk hook install
```

Эсвэл гараар `~/.claude/settings.json`-д нэмнэ:

```json
{
  "hooks": {
    "PreToolUse": [{
      "matcher": "Bash",
      "command": "wall-vault rtk rewrite"
    }]
  }
}
```

> 💡 **Exit code хадгалалт**: RTK нь анхны тушаалын гарах кодыг яг байгаагаар нь буцаадаг. Тушаал амжилтгүй болвол (exit code ≠ 0) AI мөн амжилтгүй болсныг зөв илрүүлнэ.

> 💡 **Англи хэлийг албадах**: RTK нь тушаалуудыг `LC_ALL=C`-ээр ажиллуулж системийн хэлний тохиргооноос үл хамааран үргэлж англи хэл дээрх гаралт үүсгэдэг. Ингэснээр шүүлтүүр зөв ажиллахыг баталгаажуулна.

## Орчны хувьсагч лавлах

Орчны хувьсагч гэдэг нь програмд тохиргооны утга дамжуулах арга юм. `export хувьсагчийн-нэр=утга` форматаар терминалд оруулах эсвэл автомат эхлүүлэлтийн үйлчилгээний файлд оруулахад үргэлж хүчинтэй байна.

| Хувьсагч | Тайлбар | Жишээ утга |
|------|------|---------|
| `WV_LANG` | Хяналтын самбарын хэл | `ko`, `en`, `ja` |
| `WV_THEME` | Хяналтын самбарын загвар | `light`, `dark`, `gold` |
| `WV_KEY_GOOGLE` | Google API түлхүүр (олон бол таслалаар) | `AIza...,AIza...` |
| `WV_KEY_OPENROUTER` | OpenRouter API түлхүүр | `sk-or-v1-...` |
| `WV_VAULT_URL` | Тархсан горимд сангийн серверийн хаяг | `http://192.168.x.x:56243` |
| `WV_VAULT_TOKEN` | Клиент (бот) баталгаажуулалтын токен | `my-secret-token` |
| `WV_ADMIN_TOKEN` | Админ токен | `admin-token-here` |
| `WV_MASTER_PASS` | API түлхүүрийн шифрлэлтийн нууц үг | `my-password` |
| `WV_AVATAR` | Аватар зургийн файлын зам (`~/.openclaw/`-ээс харьцангуй зам) | `workspace/avatars/avatar.png` |
| `OLLAMA_URL` | Ollama дотоод серверийн хаяг | `http://192.168.x.x:11434` |

## Алдаа засах

### Прокси эхлэхгүй бол

Ихэвчлэн порт аль хэдийн өөр програмд ашиглагдаж байна.

```bash
ss -tlnp | grep 56244   # 56244 портыг хэн ашиглаж байгааг шалгах
wall-vault proxy --port 8080   # Өөр порт дугаараар эхлүүлэх
```

### API түлхүүрийн алдаа гарвал (429, 402, 401, 403, 582)

| Алдааны код | Утга | Шийдэх арга |
|----------|------|----------|
| **429** | Хүсэлт хэт олон (хэрэглээ хэтэрсэн) | Түр хүлээх эсвэл өөр түлхүүр нэмэх |
| **402** | Төлбөр шаардлагатай эсвэл кредит хүрэлцэхгүй | Тохирох үйлчилгээнд кредит цэнэглэх |
| **401 / 403** | Түлхүүр буруу эсвэл зөвшөөрөлгүй | Түлхүүрийн утгыг дахин шалгаж бүртгэх |
| **582** | Gateway-ийн ачаалал (5 минутын cooldown) | 5 минутын дараа автоматаар нээгдэнэ |

```bash
# Бүртгэгдсэн түлхүүрүүдийн жагсаалт ба төлвийг шалгах
curl -H "Authorization: Bearer админ-токен" http://localhost:56243/admin/keys

# Түлхүүрийн хэрэглээний тоолуурыг дахин тохируулах
curl -X POST -H "Authorization: Bearer админ-токен" http://localhost:56243/admin/keys/reset
```

### Агент "холбогдоогүй" гэж харагдвал

"Холбогдоогүй" гэдэг нь прокси процесс сан руу дохио (heartbeat) илгээхгүй байна гэсэн утгатай. **Тохиргоо хадгалагдаагүй гэсэн утга биш.** Прокси сангийн серверийн хаяг болон токенийг мэдэж ажиллаж байх шаардлагатай бөгөөд холболтын төлөв өөрчлөгдөнө.

```bash
# Сангийн серверийн хаяг, токен, клиентийн ID-г зааж прокси эхлүүлэх
WV_VAULT_URL=http://сангийн-серверийн-хаяг:56243 \
WV_VAULT_TOKEN=клиентийн-токен \
WV_VAULT_CLIENT_ID=клиентийн-ID \
wall-vault proxy
```

Холболт амжилттай бол ойролцоогоор 20 секундын дотор хяналтын самбарт 🟢 Ажиллаж байна болж хувирна.

### Ollama холбогдохгүй бол

Ollama гэдэг нь таны компьютер дээр шууд AI ажиллуулдаг програм. Эхлээд Ollama асаалттай байгаа эсэхийг шалгана.

```bash
curl http://localhost:11434/api/tags   # Загварын жагсаалт гарвал хэвийн
export OLLAMA_URL=http://192.168.x.x:11434   # Өөр компьютер дээр ажиллаж байвал
```

> ⚠️ Ollama хариу өгөхгүй бол эхлээд `ollama serve` тушаалаар Ollama-г эхлүүлнэ.

> ⚠️ **Том загварууд удаан**: `qwen3.5:35b`, `deepseek-r1` гэх мэт том загварууд хариу үүсгэхэд хэдэн минут шаардагдаж болно. Хариу ирэхгүй мэт санагдаж болох ч хэвийн боловсруулагдаж байж болзошгүй тул хүлээнэ үү.
